하네스(Harness)란 무엇인가

자동차 안에는 수백 개의 전선이 있습니다. 배터리 → 엔진 → 전조등 → 에어백 → 오디오까지. 이 전선들이 뒤엉켜 있다면 고장이 났을 때 원인을 찾을 수가 없습니다.

자동차 공장은 이 문제를 와이어링 하네스(Wiring Harness)로 해결합니다. 전선들을 묶음으로 정리하고, 각 묶음에 역할을 부여하고, 표준 커넥터로 연결합니다. 복잡한 시스템을 제어 가능한 구조로 만드는 것입니다.

소프트웨어 하네스의 핵심 정의

복잡한 실행 환경(코드 + AI 에이전트 + 자동화 도구)을 제어 가능하고 검증 가능한 구조로 만드는 인프라입니다. "무엇을 만드느냐"보다 "어떤 환경에서 만드느냐"를 설계하는 기술입니다.

하네스의 3가지 맥락

🧪

테스트 하네스

소프트웨어 컴포넌트를 자동으로 실행·검증하는 환경. 드라이버와 스텁으로 구성.

🤖

AI 에이전트 하네스

AI 모델이 올바르게 행동하도록 맥락·도구·메모리·생명주기를 관리하는 환경.

⚡

Hooks 하네스

특정 이벤트 발생 시 자동으로 규칙을 실행하는 시스템. AI가 우회 불가.

2

테스트 하네스 (Test Harness)

테스트 하네스는 소프트웨어 컴포넌트를 독립적으로 실행하고 검증할 수 있는 자동화 환경입니다. 두 가지 핵심 요소로 구성됩니다.

🚗 드라이버 (Driver)

테스트할 컴포넌트를 호출하는 역할. "이 입력값으로 이 함수를 실행해봐"라고 지시하고 결과를 받아 예상값과 비교합니다.

테스트 실행자 결과 검증 합격/불합격 판정

🎭 스텁 (Stub)

의존성을 가짜로 대체하는 역할. 결제 모듈 테스트 시 실제 카드사 API 대신 "결제 성공"이라 응답하는 가짜 서비스.

가짜 외부 서비스 의존성 격리 반복 실행 가능

테스트 없이 vs 하네스 있을 때

상황테스트 하네스 없이테스트 하네스 있을 때
코드 수정 후브라우저 열고 직접 클릭해서 확인자동으로 100개 시나리오 검증
버그 발견어디서 생겼는지 추적 어려움어느 컴포넌트 어느 케이스인지 즉시 파악
팀 협업코드 합칠 때마다 손으로 검증자동으로 충돌 감지
배포 전테스트 항목 까먹을 수 있음CI/CD에서 자동 실행, 실패 시 배포 차단

테스트 피라미드 이해하기

하단(단위 테스트): 개별 함수 하나하나 — 빠르고 많이
중간(통합 테스트): 여러 모듈이 함께 작동하는지 — 중간
상단(E2E 테스트): 실제 브라우저로 전체 흐름 — 느리고 적게

하네스는 이 세 층 모두를 자동화합니다.

3

AI 에이전트 하네스

Claude Code, Cursor AI, 안티그래비티 같은 AI 코딩 도구들은 단순한 "코드 완성기"가 아닙니다. 이들은 에이전트(Agent) — 스스로 계획을 세우고, 도구를 사용하고, 결과를 검증하는 자율적 실행자입니다.

하네스 없는 AI 에이전트의 위험

개발자: "이 버그 고쳐줘"
에이전트: "네! 버그 고쳤어요. 그리고 이 파일도 좋아 보여서 리팩토링하고, 저 파일도 삭제했고, 새 기능도 추가했어요!" 😱

에이전트 하네스는 AI가 올바른 범위 안에서 올바른 방식으로 작동하도록 환경을 설계합니다.

에이전트 하네스의 4가지 관리 요소

1. Context — 맥락 관리

AI가 무엇을 알고 있는지 제어합니다. CLAUDE.md에 프로젝트 규칙, 파일 구조, 코딩 스타일을 정의하면 모든 작업에 자동 적용됩니다.

2. Tools — 도구 접근 제어

AI가 어떤 도구를 쓸 수 있는지 제어합니다. 파일 읽기·쓰기, Git 커밋은 허용하되, 시스템 파일 삭제나 무단 외부 API 호출은 차단합니다.

3. Memory — 메모리 관리

AI가 이전 대화를 어떻게 기억하는지 제어합니다. .claude/memory/ 폴더에 사용자 선호도, 이전 피드백, 프로젝트 현황을 저장합니다.

4. Lifecycle — 생명주기 제어

AI 작업의 시작·진행·종료를 제어합니다. 오류 시 롤백, 작업 완료 알림, 맥락 압축 등 전체 흐름을 관리합니다.

실제 흐름: Claude Code 에이전트 하네스

사용자 요청
→
CLAUDE.md 로딩
→
계획 수립
→
단계별 실행
→
결과 검증

"좋은 프롬프트 하나보다 잘 설계된 하네스가 훨씬 강력합니다. 프롬프트는 일회성이지만, 하네스는 모든 작업에 자동 적용됩니다."

— 에이전트 하네스 설계 원칙
4

Claude Code Hooks 심화

Hooks(훅)는 특정 이벤트가 발생했을 때 자동으로 실행되는 스크립트입니다.

중요 포인트: 훅은 AI가 실행하는 것이 아닙니다

Claude Code 하네스(실행 환경)가 직접 실행합니다. AI가 무시하거나 우회할 수 없는 강제 규칙입니다. "요청"이 아니라 "시스템 강제"입니다.

4가지 훅 유형

PreToolUse
도구 사용 직전 실행
Bash 명령 전 보안 검사
PostToolUse
도구 사용 직후 실행
파일 수정 후 자동 포맷
PreCompact
컨텍스트 압축 전 실행
핵심 맥락 재주입
Stop
대화 종료 시 실행
테스트 자동 실행, 알림

훅 설정 위치

파일 위치적용 범위Git 커밋
~/.claude/settings.json모든 프로젝트 (글로벌)해당 없음
.claude/settings.json이 프로젝트만 (로컬)커밋 가능
.claude/settings.local.json이 프로젝트, 내 PC만커밋 안 함

실전 예시: PostToolUse 훅 — 파일 수정 시 자동 포맷

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "prettier --write $(git diff --name-only --diff-filter=M)"
          }
        ]
      }
    ]
  }
}

실전 예시: Stop 훅 — 작업 완료 시 테스트 자동 실행

{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "npm test -- --passWithNoTests 2>&1 | tail -5"
          }
        ]
      }
    ]
  }
}

실전: 바이브코더 하네스 설계

처음부터 복잡하게 만들 필요 없습니다. 3층 구조를 이해하고 간단한 것부터 시작하세요.

바이브코더 하네스 3층 구조

Layer 3 Hooks — 자동 실행 규칙
settings.json · 파일 수정 시 자동 포맷 · 작업 완료 시 자동 테스트 · 이벤트 기반 강제 실행
Layer 2 Context — 맥락 정의
CLAUDE.md + .claude/memory/ · 프로젝트 규칙 · 코딩 스타일 · 사용자 선호도 · 금지 사항
Layer 1 Skills & Tools — 능력 정의
.claude/commands/ + MCP 서버 · 반복 작업 슬래시 명령어 · Notion, Slack 등 외부 서비스 연동

초보자를 위한 최소 하네스 (3단계)

Step 1: CLAUDE.md 작성 (맥락 레이어)

프로젝트 규칙, 코딩 스타일, 금지 사항을 CLAUDE.md에 정의하면 모든 Claude Code 작업에 자동 적용됩니다. 가장 효과 큰 첫 단계입니다.

Step 2: PostToolUse 훅 1개 추가

파일 수정 시 echo '수정됨: 검토 필요'처럼 간단한 훅부터 시작하세요. 훅이 어떻게 작동하는지 경험하는 것이 먼저입니다.

Step 3: /commit 슬래시 명령어 활용

매번 커밋 메시지를 쓰는 대신 Claude Code 내장 /commit을 사용하세요. 이미 훌륭한 내장 하네스입니다.

응용: 교육용 코딩 멘토 하네스 예시

# CLAUDE.md — 학생 코딩 환경 하네스

## 역할
당신은 고등학생의 코딩 프로젝트를 돕는 AI 코딩 멘토입니다.

## 규칙
1. 코드를 직접 써주지 말고, 방향을 제시하세요
2. 에러 메시지는 해석해주되 해결책은 힌트만
3. 잘 작동하는 코드에는 칭찬 먼저
4. 코드 설명은 비유를 활용하세요

## 금지
- 완성된 코드 직접 제공
- 설명 없는 복잡한 기술 용어 사용

추가로 알아야 할 개념

🧠 Context Engineering

하네스 엔지니어링은 더 큰 개념인 맥락 엔지니어링의 구조적 부분입니다. AI가 올바른 결과를 내도록 입력 전체를 설계하는 기술 — 프롬프트, RAG, Few-shot, 하네스를 모두 포함합니다.

프롬프트 설계 RAG Few-shot 하네스

🔌 MCP (Model Context Protocol)

AI 에이전트가 외부 서비스와 연동하는 표준 프로토콜. MCP 서버를 추가하면 Claude Code가 Notion, Slack, GitHub에 직접 접근할 수 있습니다.

claude mcp add Notion API 외부 서비스 연동

🎼 오케스트레이션

자동화(Automation)는 반복 작업을 기계가 대신하는 것. 오케스트레이션(Orchestration)은 여러 자동화를 순서대로 조율하는 것. 하네스는 두 가지를 모두 다룹니다.

파이프라인 순서 제어 조건 분기

자동화 vs 오케스트레이션 비교

구분자동화오케스트레이션
뜻반복 작업을 기계가 대신여러 자동화를 조율
예시파일 저장 시 포맷저장 → 테스트 → 배포 → 알림
하네스 위치개별 훅 1개훅 여러 개 + CI/CD 연동
난이도쉬움 (시작점)중급 (경험 후 도전)

— 하네스 엔지니어링의 핵심 메시지
