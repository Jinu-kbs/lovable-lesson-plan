스킬이란?

스킬(Skills)은 Claude Code에서 사용할 수 있는 재사용 가능한 AI 워크플로우 템플릿입니다. Markdown 파일로 정의된 지시사항을 /슬래시-명령어 형태로 언제든지 호출할 수 있습니다.

핵심 아이디어: "같은 지시를 매번 타이핑하지 않아도 됩니다." — 복잡한 워크플로우를 한 번 정의해두면, 이후엔 슬래시 명령어 하나로 실행합니다.

스킬이 해결하는 문제

  • 반복 작업: 매번 같은 지시를 타이핑하는 비효율 제거
  • 복잡한 워크플로우 단순화: 다단계 작업을 단일 명령어로 압축
  • 팀 공유: Git으로 스킬 파일을 버전 관리하고 팀원과 공유
  • 일관성 확보: 코드 리뷰, 커밋 메시지 등 팀 표준 자동 적용

스킬 시스템 구조

스킬 파일은 두 위치에 저장됩니다. 전역 스킬은 모든 프로젝트에서 사용 가능하고, 프로젝트 스킬은 현재 프로젝트에서만 작동합니다.

전역 스킬 ~/.claude/skills/*.md 모든 프로젝트에서 사용 가능
프로젝트 스킬 .claude/skills/*.md 현재 프로젝트에서만 적용 우선순위 높음

로딩 순서 및 우선순위

Claude Code는 시작 시 전역 스킬을 먼저 로드하고, 그 다음 프로젝트 스킬을 로드합니다. 동일한 이름의 스킬이 있으면 프로젝트 스킬이 전역 스킬을 덮어씁니다. 이를 활용해 프로젝트별로 스킬을 커스터마이징할 수 있습니다.

~/.claude/
  skills/          # 전역 스킬 디렉토리
    commit.md      # 내장 커밋 스킬
    review-pr.md   # 내장 PR 리뷰 스킬
    my-skill.md    # 사용자 정의 스킬

.claude/            # 프로젝트 스킬 디렉토리
  skills/
    lesson-review.md # 프로젝트 전용 스킬

스킬 파일 형식

모든 스킬 파일은 YAML frontmatter + Markdown 본문으로 구성됩니다. frontmatter에 스킬 이름과 설명을 정의하고, 본문에 Claude에게 전달할 지시사항을 작성합니다.

핵심: description 필드는 Claude가 자동으로 스킬을 트리거할지 판단하는 기준이 됩니다. 자연어로 "언제 이 스킬을 사용해야 하는지" 명확하게 작성하세요.
3

내장 스킬 목록

Claude Code에는 기본적으로 제공되는 내장 스킬이 있습니다. 별도 설정 없이 바로 사용할 수 있으며, 필요에 따라 프로젝트 스킬로 오버라이드할 수 있습니다.

📝 커밋 생성
/commit
변경사항을 분석하고 잘 형식화된 git 커밋 메시지를 자동으로 작성하여 커밋합니다. Co-Authored-By 태그 포함 옵션.
🔍 PR 코드 리뷰
/review-pr
Pull Request의 변경사항을 분석하고 코드 품질, 보안, 성능 관점에서 구조화된 피드백을 생성합니다.
⚙️ 설정 업데이트
/update-config
Claude Code의 settings.json 파일을 업데이트합니다. 훅 설정, 권한 관리, 환경 변수 등을 대화형으로 수정합니다.
🔧 코드 리팩토링
/simplify
코드 품질을 검토하고 가독성, 중복 제거, 성능 개선 관점에서 리팩토링 제안 및 실행합니다.
🔄 반복 실행
/loop
지정한 명령이나 워크플로우를 반복 실행하도록 설정합니다. 일정 간격 자동화나 배치 처리에 활용합니다.
🤖 API 앱 빌드
/claude-api
Claude API 및 Anthropic SDK를 활용한 앱을 빌드합니다. API 호출 패턴, 에러 처리, 스트리밍 응답 구현을 안내합니다.
참고: 내장 스킬 목록은 Claude Code 버전에 따라 달라질 수 있습니다. 현재 사용 가능한 스킬 목록은 /를 입력하면 자동완성으로 확인할 수 있습니다.
4

스킬 사용법

Claude Code 대화창에서 슬래시로 시작하는 명령어를 입력하면 해당 스킬이 실행됩니다. 인수를 함께 전달하거나 인수 없이 호출하면 스킬이 기본 동작을 수행합니다.

기본 호출 방법

# 인수 없이 호출 (기본 동작)
/commit

# 문자열 인수 전달
/commit -m "Fix login bug"

# PR 번호 전달
/review-pr 123

# 여러 인수
/lesson-review lovable-beginner.html --strict

자동 트리거

스킬의 description이 현재 대화 맥락과 잘 맞으면, Claude가 자동으로 해당 스킬을 호출할 수 있습니다. 예를 들어 "커밋해줘"라고 요청하면 /commit 스킬이 자동으로 실행될 수 있습니다.

자동 트리거 조건: 스킬의 description을 "~할 때 사용", "~하는 경우" 등 명확한 트리거 조건으로 작성하면 자동 호출 정확도가 높아집니다.

Skill 도구를 통한 명시적 호출

Claude Code가 내부적으로 스킬을 실행할 때는 Skill 도구를 사용합니다. 이 도구는 스킬 이름과 선택적 인수를 받아 해당 스킬의 지시사항을 Claude에게 전달합니다.

// Skill 도구 호출 예시 (내부 메커니즘)
{
  "skill": "commit",
  "args": "-m 'Fix: 로그인 버그 수정'"
}

슬래시 명령어 자동완성

Claude Code 입력창에서 /를 입력하면 사용 가능한 스킬 목록이 자동완성으로 표시됩니다. 방향키로 선택하고 Enter로 실행합니다.

5

커스텀 스킬 만들기

자신만의 스킬을 만들면 자주 사용하는 워크플로우를 완전히 자동화할 수 있습니다. 아래 5단계를 따라 첫 번째 커스텀 스킬을 만들어보세요.

1

스킬 디렉토리 생성

전역 스킬 디렉토리 또는 프로젝트 스킬 디렉토리를 생성합니다.

# 전역 스킬 (모든 프로젝트에서 사용)
mkdir -p ~/.claude/skills

# 프로젝트 스킬 (현재 프로젝트에서만)
mkdir -p .claude/skills
2

스킬 파일 생성

스킬 이름으로 Markdown 파일을 생성합니다. 파일명이 슬래시 명령어가 됩니다.

# my-skill.md → /my-skill 로 호출됨
touch ~/.claude/skills/my-skill.md
3

YAML Frontmatter 작성

파일 상단에 스킬의 이름과 설명을 YAML 형식으로 작성합니다. description은 자동 트리거 판단에 사용됩니다.

4

스킬 본문(프롬프트) 작성

Frontmatter 아래에 Claude에게 전달할 지시사항을 Markdown으로 작성합니다. {args} 변수로 호출 시 전달된 인수를 참조할 수 있습니다.

5

스킬 테스트

Claude Code를 재시작하거나 새 대화를 시작한 후 슬래시 명령어로 호출합니다.

/my-skill
/my-skill some arguments here
6

파일 구조

스킬 파일은 YAML frontmatter와 Markdown 본문으로 구성됩니다. 아래 템플릿을 기반으로 작성하세요.

기본 템플릿

---
name: my-skill-name
description: 언제 이 스킬을 사용해야 하는지 설명 (자동 트리거에 중요)
---

# 스킬 내용

[Claude에게 전달할 지시사항을 여기에 작성합니다]

## 인수 처리

인수가 있으면: {args}
없으면 기본 동작을 수행합니다.

## 출력 형식

- 출력 결과의 형식을 지정합니다
- 예: 마크다운, 코드 블록, 테이블 등

실전 예시 파일들

📋 code-review.md — 코드 리뷰 스킬 
---
name: code-review
description: 특정 파일이나 변경사항의 코드 품질을 리뷰하고 개선 제안이 필요할 때 사용
---

# 코드 리뷰

## 검토 대상

{args}가 있으면 해당 파일/경로를 검토합니다.
없으면 최근 변경된 파일을 검토합니다.

## 검토 기준

다음 항목을 체계적으로 검토해주세요:

### 1. 코드 품질
- 명명 규칙 일관성
- 함수/변수 이름의 명확성
- 코드 중복 여부

### 2. 잠재적 버그
- 엣지 케이스 처리
- 에러 핸들링
- null/undefined 처리

### 3. 성능
- 불필요한 연산
- 메모리 누수 가능성

### 4. 보안
- SQL 인젝션, XSS 등 취약점
- 민감 정보 노출 여부

## 출력 형식

### ✅ 잘된 점
[좋은 부분 나열]

### ⚠️ 개선 제안
[개선이 필요한 부분과 구체적인 수정 방법]

### 🔴 즉시 수정 필요
[버그나 보안 문제]

# 테스트 코드 작성

## 대상

{args}로 지정된 파일 또는 함수에 대한 테스트를 작성합니다.

## 테스트 작성 원칙

1. **AAA 패턴**: Arrange(준비) → Act(실행) → Assert(검증)
2. **엣지 케이스**: 빈 입력, null, 최대/최소값, 예외 상황
3. **명확한 테스트명**: "should_[동작]_when_[조건]" 형식

## 출력

- 프레임워크: 현재 프로젝트에서 사용 중인 테스트 프레임워크 자동 감지
- 파일명: `{원본파일명}.test.{확장자}`
- 커버리지 목표: 주요 케이스 80% 이상

예시 1: 교안 검토 스킬

바이브코딩 가이드 교안의 품질을 자동으로 검토하는 스킬입니다.

---
name: lesson-review

# 바이브코딩 교안 검토

## 대상 파일
{args}로 지정된 파일을 검토합니다. 미지정 시 현재 작업 중인 교안 파일을 검토합니다.

## 검토 체크리스트

### 🔒 개인정보 보호 (필수)
- 개별 발언자 실명 인용 → "— 실사용자 후기"로 변환 필요
- 카카오톡 닉네임 노출 여부 확인

### 📚 내용 품질
- 학습 목표가 명확한가
- 실습 예제가 현재 도구 버전에 맞는가
- 초보자가 따라할 수 있는 수준인가

### 🎨 HTML 구조
- topnav 링크 5개 정상 여부
- sidebar 앵커 링크와 섹션 id 매칭
- common.css, common.js 로드 순서

### 🎨 색상 테마
- 도구별 --primary 색상 적용 여부
- :root 색상 변수 오버라이드 존재 여부

## 출력 형식
검토 결과를 표 형식으로 정리하고, 수정이 필요한 항목은 구체적인 수정 방법을 제시합니다.

예시 2: 스마트 커밋 스킬

이 프로젝트의 커밋 규칙을 자동으로 적용하는 커밋 스킬입니다.

---
name: smart-commit
description: 바이브코딩 가이드 프로젝트에 맞는 한국어 커밋 메시지를 생성하고 커밋할 때 사용
---

# 스마트 커밋

## 변경사항 분석
1. `git status`와 `git diff`를 실행하여 변경사항을 파악합니다
2. 변경된 파일의 성격을 분류합니다 (새 교안, 수정, 스타일, 문서 등)

## 커밋 메시지 규칙
- **한국어**로 작성
- 변경 내용을 명확하게 요약 (50자 이내)
- 접두어 사용: 새 교안 추가, 교안 수정, 스타일 개선, 문서 업데이트, 버그 수정

## 예시 메시지
- "Claude Code 스킬 완전 가이드 페이지 추가"
- "러버블 초보자편 개인정보 익명화 처리"
- "공통 네비게이션 링크 업데이트"

## 실행
{args}가 있으면 커밋 메시지로 사용합니다.
없으면 변경사항을 분석하여 적절한 메시지를 제안하고 확인 후 커밋합니다.

반드시 아래 형식으로 커밋합니다:
```
Co-Authored-By: Claude Sonnet 4.6 
```

## 분석 대상
{args}로 PR 번호나 브랜치명을 지정합니다.

## PR 설명 구조

### Summary

### Screenshots (해당 시)

스킬 vs 메모리 vs 훅

Claude Code의 자동화 시스템은 세 가지 레이어로 구성됩니다. 각각의 역할과 사용 시점을 정확히 이해하면 효과적으로 조합할 수 있습니다.

⚡
스킬 (Skills)
이 페이지 주제
재사용 가능한 워크플로우 템플릿
  • 슬래시 명령어로 명시적 호출
  • 복잡한 다단계 작업 자동화
  • 팀 공유 및 버전 관리 가능
  • 조건부 자동 트리거 지원
🧠
메모리 (Memory)
CLAUDE.md / 메모리 파일
컨텍스트 기억 및 지속적 적용
  • 매 대화에 자동 적용
  • 사용자 선호도 저장
  • 프로젝트 정보 기억
  • 수동 호출 불필요
🪝
훅 (Hooks)
settings.json
이벤트 기반 자동 실행
  • 파일 저장, 커밋 전후 트리거
  • 조건부 실행 (파일 패턴 등)
  • 외부 도구 연동
  • 시스템 레벨 자동화

상세 비교 테이블

항목 스킬 메모리 훅
호출 방법 명시적 / 자동 항상 자동 이벤트 자동
정의 위치 skills/*.md CLAUDE.md settings.json
인수 전달 지원 없음 환경변수
팀 공유 Git으로 공유 Git으로 공유 Git으로 공유
주요 용도 반복 워크플로우 프로젝트 맥락 파일/커밋 이벤트
복잡도 중간 낮음 높음
실전 조합 팁: 메모리(CLAUDE.md)로 프로젝트 컨텍스트를 제공하고, 스킬로 반복 작업을 자동화하고, 훅으로 품질 검사를 강제하면 강력한 자동화 시스템을 구축할 수 있습니다.

고급 활용

기본 스킬 사용에 익숙해졌다면 고급 패턴을 활용해 더 강력한 자동화 시스템을 구축하세요.

🔗 스킬 체이닝

스킬 본문 안에서 다른 스킬을 호출하도록 지시할 수 있습니다. 예: deploy 스킬이 내부적으로 /test → /commit → /push를 순서대로 실행.

🧠 스킬 + 메모리 조합

CLAUDE.md에 프로젝트 정보를 저장하고, 스킬에서 "CLAUDE.md를 읽어서 프로젝트 컨텍스트를 파악한 후 ~하라"고 지시합니다. 스킬이 항상 최신 프로젝트 정보를 활용합니다.

🪝 스킬 + 훅 조합

훅에서 특정 이벤트 발생 시 Claude에게 메시지를 전달하고, 그 메시지가 스킬의 자동 트리거 조건과 매칭되면 스킬이 자동으로 실행됩니다.

📦 팀 스킬 저장소

프로젝트의 .claude/skills/ 폴더를 Git으로 관리합니다. 팀원 모두가 동일한 스킬을 사용하고, PR로 스킬 변경을 리뷰하며, 버전 히스토리를 추적합니다.

🔍 스킬 디버깅

스킬이 작동하지 않을 때 확인할 사항: (1) 파일 위치가 올바른가, (2) frontmatter YAML 문법 오류가 없는가, (3) Claude Code를 재시작했는가, (4) 스킬명과 파일명이 일치하는가.

🌐 스킬 공개 공유

GitHub에 스킬 컬렉션 저장소를 만들어 커뮤니티와 공유할 수 있습니다. 다른 사람의 스킬을 가져와 프로젝트에 맞게 수정하는 것도 좋은 방법입니다.

스킬 디버깅 체크리스트

# 1. 스킬 파일 위치 확인
ls ~/.claude/skills/         # 전역 스킬
ls .claude/skills/           # 프로젝트 스킬

# 2. frontmatter 문법 확인 (YAML 린터 활용)
# name: 값에 따옴표 불필요, 특수문자 있으면 따옴표 사용

# 3. 파일 내용 확인
cat ~/.claude/skills/my-skill.md

# 4. Claude Code 재시작
# 새로운 터미널에서 claude 명령어 재실행

# 5. 슬래시 자동완성으로 스킬 로드 확인
# Claude Code 입력창에서 / 입력 후 목록 확인
주의: 스킬 파일의 YAML frontmatter 문법 오류는 해당 스킬뿐만 아니라 같은 디렉토리의 다른 스킬 로딩에도 영향을 줄 수 있습니다. YAML 문법을 반드시 지켜주세요.
