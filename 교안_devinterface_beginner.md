## 1. 개발 환경이 뭔가요?

### 학습 목표

- GUI와 CLI의 차이를 이해한다
- 바이브코딩 도구마다 사용하는 인터페이스가 다름을 안다
- 개발 환경을 왜 배워야 하는지 동기를 갖는다

### 상세 설명

#### 비유로 이해하기: 자동차와 도로

바이브코딩 도구(Lovable, Cursor 등)가 **자동차**라면, 개발 환경(터미널, IDE 등)은 **도로와 신호등**입니다.

- 자동차(바이브코딩 도구)가 아무리 좋아도, 도로(개발 환경)가 없으면 달릴 수 없습니다
- 신호등(명령어)을 이해해야 안전하게 운전할 수 있습니다
- 도로 위의 표지판(파일 경로)을 읽을 줄 알아야 원하는 곳에 도착합니다

개발 환경을 배운다는 것은 "도로 위에서 운전하는 법"을 배우는 것과 같습니다. 처음엔 복잡해 보이지만, 한번 익히면 어떤 자동차(도구)든 운전할 수 있게 됩니다.

#### GUI vs CLI — 무엇이 다를까?

컴퓨터를 다루는 방법은 크게 두 가지입니다:

| 구분 | GUI (그래픽 사용자 인터페이스) | CLI (명령줄 인터페이스) |
|------|------------------------------|------------------------|
| **조작 방식** | 마우스로 클릭, 드래그 | 키보드로 명령어 입력 |
| **일상 비유** | 식당 메뉴판에서 손가락으로 가리키기 | 전화로 주문 내용을 말하기 |
| **장점** | 직관적, 쉽게 시작 | 빠르고, 자동화 가능, 정밀 제어 |
| **단점** | 반복 작업이 느림 | 명령어를 외워야 함 |
| **예시** | 파일 탐색기에서 폴더 더블클릭 | `cd Documents` 입력 |

> 여러분이 지금 마우스로 폴더를 더블클릭하는 것이 GUI, 터미널에 `cd Documents`라고 타이핑하는 것이 CLI입니다. 결과는 같지만 방법이 다릅니다!

#### 바이브코딩 5대 도구와 인터페이스

각 바이브코딩 도구는 서로 다른 인터페이스 환경을 사용합니다:

| 도구 | 주요 인터페이스 | 터미널 필요? | 설치 필요? | 난이도 |
|------|---------------|-------------|-----------|--------|
| **Lovable** | 웹 브라우저 (GUI) | 불필요 | 불필요 | 가장 쉬움 |
| **AI Studio** | 웹 브라우저 (GUI) | 불필요 | 불필요 | 쉬움 |
| **AntiGravity** | 데스크톱 앱 (GUI+터미널) | 일부 필요 | 앱 설치 | 보통 |
| **Cursor** | 데스크톱 IDE (GUI+터미널) | 자주 사용 | 앱 설치 | 보통~어려움 |
| **Claude Code** | 터미널 (CLI 전용) | 필수 | CLI 설치 | 어려움 |

> 러버블이나 AI Studio만 쓸 거라면 터미널을 꼭 배울 필요는 없습니다. 하지만 Cursor나 Claude Code로 넘어가려면, 그리고 개발의 원리를 이해하려면 이 교안이 반드시 필요합니다.

#### 왜 마우스만 쓰면 안 되나?

"GUI가 더 쉬운데 왜 CLI를 배워야 하나요?"라는 질문은 당연합니다. 이유가 있습니다:

1. **자동화**: 100개의 파일 이름을 바꿔야 한다면? GUI로는 하나씩 클릭해야 하지만, CLI는 명령어 한 줄이면 됩니다
2. **정밀 제어**: GUI 버튼에 없는 세밀한 옵션도 CLI에서는 사용할 수 있습니다
3. **서버 환경**: 웹사이트를 올리는 서버에는 마우스가 없습니다. 터미널만 존재합니다
4. **개발 도구 호환**: npm, git, pip 같은 핵심 도구들은 CLI로 동작합니다
5. **바이브코딩 확장**: Cursor, Claude Code 같은 도구를 쓰려면 터미널이 필수입니다

### 실습 과제

아직 실습은 없습니다! 다음 질문에 머릿속으로 답해보세요:

- 내가 주로 쓰는 바이브코딩 도구는 무엇인가?
- 그 도구는 GUI인가, CLI인가?
- 다음에 써보고 싶은 도구는 터미널이 필요한가?

### 퀴즈

**Q. 다음 중 CLI(명령줄 인터페이스) 도구는 무엇일까요?**

1. 러버블(Lovable)의 웹 에디터
2. 파일 탐색기에서 폴더 더블클릭
3. 터미널에서 `git commit -m "저장"` 입력
4. 크롬 브라우저에서 주소 입력

> **정답**: 3번. 터미널에 명령어를 직접 입력하는 것이 CLI입니다. 나머지는 모두 GUI 방식이에요.

## 2. 용어 사전

### 학습 목표

- 개발 환경에서 자주 등장하는 12개 핵심 용어를 이해한다
- 각 용어를 일상적인 비유와 연결지어 기억한다

### 상세 설명

처음 개발 관련 글을 읽으면 모르는 단어가 쏟아집니다. 걱정하지 마세요. 아래 12개만 알면 이 교안의 나머지를 이해할 수 있습니다.

| 번호 | 용어 | 한 줄 정의 | 비유 |
|------|------|-----------|------|
| 1 | **터미널 (Terminal)** | 명령어를 입력하는 텍스트 기반 창 | 컴퓨터와 대화하는 채팅방 |
| 2 | **셸 (Shell)** | 명령어를 해석하는 프로그램 (bash, zsh, PowerShell 등) | 채팅방에서 내 말을 통역해주는 통역사 |
| 3 | **CLI (Command Line Interface)** | 텍스트 명령어로 컴퓨터를 조작하는 방식 | 전화로 음식 주문하기 (말로 설명) |
| 4 | **CMD (명령 프롬프트)** | Windows 전통 셸 프로그램 | 오래된 전화기 (기본 기능만) |
| 5 | **PowerShell** | Windows 최신 셸 프로그램 | 스마트폰 (기능이 많음) |
| 6 | **IDE (통합 개발 환경)** | 코드 작성+실행+디버깅을 한곳에서 하는 프로그램 | 요리사의 전문 주방 (도구가 다 갖춰진 곳) |
| 7 | **코드 에디터** | 코드를 편집하는 프로그램 (IDE보다 가벼움) | 좋은 펜이 있는 책상 (필기 도구만) |
| 8 | **경로 (Path)** | 파일이나 폴더의 위치를 나타내는 주소 | 집 주소 (시/구/동/번지) |
| 9 | **디렉토리 (Directory)** | 폴더의 다른 이름 | 서류철, 서랍 |
| 10 | **환경변수 (Environment Variable)** | 컴퓨터 전체에서 사용하는 설정값 | 건물 안내 데스크 (어디에 뭐가 있는지 알려줌) |
| 11 | **패키지 매니저** | 프로그램/라이브러리를 자동 설치해주는 도구 | 앱스토어 (필요한 앱을 검색해서 설치) |
| 12 | **npm / pip** | JavaScript / Python의 패키지 매니저 | JavaScript 앱스토어 / Python 앱스토어 |

#### 조금 더 자세히 알아볼 용어들

**터미널과 셸의 차이**

많은 사람들이 터미널과 셸을 같은 것으로 생각하지만, 엄밀히는 다릅니다:

- **터미널**: 글자가 표시되는 **창(화면)** 그 자체
- **셸**: 그 창 안에서 명령어를 **이해하고 실행하는 프로그램**

비유하면, 터미널은 **TV 화면**이고 셸은 그 안에서 돌아가는 **프로그램(채널)**입니다. 하나의 터미널에서 다른 셸(bash, PowerShell 등)을 바꿔 쓸 수 있습니다.

> 처음에는 "터미널 = 검은 화면에 글자 치는 것" 정도로 이해해도 충분합니다. 나중에 자연스럽게 구분이 됩니다!

**IDE vs 코드 에디터**

- **코드 에디터**: 코드를 예쁘게 편집할 수 있는 도구 (문법 강조, 자동완성)
- **IDE**: 코드 편집 + 실행 + 디버깅 + 버전 관리 등 모든 것이 한곳에

VS Code는 원래 코드 에디터이지만, 확장 프로그램(Extensions)을 설치하면 IDE처럼 사용할 수 있어서 실질적으로 IDE로 분류합니다.

## 3. 터미널 열기 (OS별 상세)

### 학습 목표

- 자신의 운영체제(Windows/Mac)에서 터미널을 열 수 있다
- 터미널 화면의 기본 구성 요소(프롬프트)를 이해한다
- 첫 번째 명령어(`whoami`)를 실행할 수 있다

### 상세 설명

#### Windows에서 터미널 열기

Windows에서는 터미널을 여는 방법이 여러 가지입니다. 가장 쉬운 방법부터 알려드릴게요.

**방법 1: 시작 메뉴에서 열기 (가장 쉬움)**

1. 키보드에서 `Windows 키`를 누릅니다 (키보드 왼쪽 아래 창문 모양 키)
2. "터미널" 또는 "terminal"을 입력합니다
3. **Windows 터미널** 앱이 검색됩니다. 클릭하세요

> 화면에 이런 모습이 나타납니다: 검정 또는 파란색 배경에 흰 글씨가 보이고, 맨 아래에 깜빡이는 커서가 있습니다. 상단에 탭이 있어서 여러 터미널을 동시에 열 수 있습니다.

**방법 2: 단축키로 열기**

- `Windows 키 + X`를 누른 후, 나타나는 메뉴에서 "터미널" 선택

**방법 3: 폴더에서 바로 열기**

- 파일 탐색기에서 아무 폴더를 연 상태에서 주소창에 `cmd` 또는 `powershell`을 입력하고 Enter

**CMD vs PowerShell 선택**

Windows 터미널을 열면 기본적으로 **PowerShell**이 실행됩니다. 상단 탭 옆의 `+` 버튼 옆 화살표(`v`)를 클릭하면 다른 셸을 선택할 수 있습니다:

- **PowerShell**: 파란색 배경, `PS C:\Users\이름>` 형태의 프롬프트
- **명령 프롬프트(CMD)**: 검정 배경, `C:\Users\이름>` 형태의 프롬프트

**관리자 권한으로 열기**

일부 작업(프로그램 설치 등)은 관리자 권한이 필요합니다:

1. 시작 메뉴에서 "터미널" 검색
2. 마우스 오른쪽 버튼 클릭
3. "관리자 권한으로 실행" 선택
4. "이 앱이 변경할 수 있도록 허용하시겠습니까?" → "예" 클릭

> 프롬프트 제목에 "관리자:"가 표시되면 관리자 모드입니다. 일반 작업에서는 관리자 권한이 필요 없으니, 보통은 그냥 열면 됩니다.

#### Mac에서 터미널 열기

**방법 1: Spotlight로 열기 (가장 쉬움)**

1. `Command(⌘) + Space`를 동시에 누릅니다
2. 화면 중앙에 검색창이 나타납니다
3. "터미널" 또는 "terminal"을 입력합니다
4. **터미널** 앱이 검색됩니다. Enter를 누르세요

> 화면에 이런 모습이 나타납니다: 흰색(또는 검정) 배경에 글씨가 보이고, 맨 아래에 `사용자이름@컴퓨터이름 ~ %`와 같은 텍스트와 깜빡이는 커서가 있습니다.

**방법 2: Finder에서 열기**

1. Finder 열기
2. 상단 메뉴 "이동" → "유틸리티"
3. "터미널" 더블클릭

**Mac 터미널 기본 셸: zsh**

Mac은 기본적으로 **zsh**(Z shell)이라는 셸을 사용합니다. 프롬프트 끝에 `%` 기호가 보입니다:

```
사용자이름@MacBookPro ~ %
```

- `사용자이름`: 현재 로그인한 계정 이름
- `@MacBookPro`: 컴퓨터 이름
- `~`: 현재 위치 (홈 디렉토리)
- `%`: zsh 셸이라는 표시 (입력 대기 중)

> 처음엔 누구나 헷갈립니다. 지금은 "`%` 뒤에 명령어를 치면 된다"는 것만 기억하세요!

#### 프롬프트란?

터미널에서 깜빡이는 커서 앞에 있는 텍스트를 **프롬프트(prompt)**라고 합니다. "명령어를 입력하세요"라는 뜻입니다.

| OS | 셸 | 프롬프트 예시 |
|----|-----|-------------|
| Windows | CMD | `C:\Users\이름>` |
| Windows | PowerShell | `PS C:\Users\이름>` |
| Mac | zsh | `이름@Mac ~ %` |
| Linux | bash | `이름@PC:~$` |

### 실습 과제

터미널을 열고 첫 번째 명령어를 입력해봅시다!

```bash
whoami
```

이 명령어는 "나는 누구인가?"를 뜻합니다. 현재 로그인한 사용자 이름이 출력됩니다.

**Windows에서의 결과 예시:**
```
PS C:\Users\홍길동> whoami
desktop-abc123\홍길동
```

**Mac에서의 결과 예시:**
```
홍길동@MacBookPro ~ % whoami
홍길동
```

> 여러분의 사용자 이름이 출력되었다면 성공입니다!

## 4. CMD vs PowerShell — 뭘 써야 하나?

### 학습 목표

- CMD와 PowerShell의 차이를 이해한다
- 각각의 프롬프트 모양을 구분할 수 있다
- 어떤 것을 기본으로 사용할지 결정한다

### 상세 설명

> 이 섹션은 **Windows 사용자**를 위한 내용입니다. Mac 사용자는 건너뛰어도 됩니다 (Mac은 zsh/bash를 사용합니다).

Windows에는 두 가지 셸이 있습니다:

#### CMD (명령 프롬프트)

- Windows 초창기부터 있던 전통적인 셸
- 프롬프트: `C:\Users\이름>`
- 명령어가 단순하고 기능이 적음
- 오래된 튜토리얼에서 많이 사용

#### PowerShell

- Microsoft가 새로 만든 현대적 셸
- 프롬프트: `PS C:\Users\이름>`
- 명령어가 "영어 문장"처럼 읽힘 (예: `Get-ChildItem`)
- 훨씬 강력한 기능, 스크립트 작성 가능
- Windows 10/11에서 기본 셸

#### 같은 작업, 다른 명령어

| 하고 싶은 것 | CMD | PowerShell | Mac/Linux (Bash) |
|-------------|-----|------------|-----------------|
| 현재 위치 보기 | `cd` (인수 없이) | `Get-Location` 또는 `pwd` | `pwd` |
| 폴더 내용 보기 | `dir` | `Get-ChildItem` 또는 `ls` | `ls` |
| 폴더 이동 | `cd 폴더이름` | `Set-Location 폴더이름` 또는 `cd 폴더이름` | `cd 폴더이름` |
| 폴더 만들기 | `mkdir 이름` | `New-Item -ItemType Directory -Name 이름` 또는 `mkdir 이름` | `mkdir 이름` |
| 화면 지우기 | `cls` | `Clear-Host` 또는 `cls` | `clear` |
| 파일 내용 보기 | `type 파일` | `Get-Content 파일` 또는 `cat 파일` | `cat 파일` |
| 파일 삭제 | `del 파일` | `Remove-Item 파일` 또는 `rm 파일` | `rm 파일` |

> 눈치채셨나요? PowerShell에서는 `ls`, `cd`, `pwd` 같은 Bash 명령어도 사용할 수 있습니다! PowerShell이 이런 명령어를 **별명(Alias)**으로 등록해두었기 때문입니다. 그래서 PowerShell을 쓰면 Bash 명령어도 함께 익힐 수 있습니다.

#### 결론: PowerShell을 기본으로 쓰세요

이 교안에서는 **PowerShell을 기본으로** 사용합니다. 이유는:

1. Windows 최신 기본 셸입니다
2. Bash 명령어와 호환됩니다 (`ls`, `cd`, `pwd` 등)
3. Cursor, Claude Code 같은 바이브코딩 도구와도 잘 연동됩니다
4. 추후 스크립트 자동화로 확장이 가능합니다

> 단, 인터넷 검색 시 CMD 기반 명령어가 많이 나옵니다. 그래서 CMD도 함께 알아두면 좋습니다. 이 교안에서는 두 가지를 병기합니다.

### 실습 과제

CMD와 PowerShell을 각각 열어서 같은 작업을 해봅시다.

**CMD에서:**
```cmd
C:\Users\이름> dir
```

**PowerShell에서:**
```powershell
PS C:\Users\이름> Get-ChildItem
```

두 명령어의 출력 결과를 비교해보세요. 보여주는 정보가 살짝 다르지만, 둘 다 현재 폴더의 파일 목록을 보여줍니다.

> PowerShell에서 `ls`를 입력해도 같은 결과가 나옵니다. `ls`는 `Get-ChildItem`의 별명이니까요!

- [ ] `dir`과 `Get-ChildItem`(`ls`)이 같은 역할임을 이해했는가?
- [ ] 앞으로 PowerShell을 기본으로 사용하기로 결정했는가?

## 5. 파일과 폴더 — 컴퓨터 속 서랍장

### 학습 목표

- 파일과 폴더(디렉토리)의 관계를 이해한다
- 절대 경로와 상대 경로의 차이를 안다
- 홈 디렉토리와 현재 디렉토리 개념을 이해한다

### 상세 설명

#### 비유: 컴퓨터는 서랍장

컴퓨터의 파일 시스템은 큰 서랍장과 같습니다:

- **컴퓨터** = 큰 서랍장
- **폴더(디렉토리)** = 서랍, 칸막이
- **파일** = 서랍 안에 들어있는 물건(문서, 사진 등)
- **경로(Path)** = 물건이 어디에 있는지 알려주는 주소

#### 디렉토리 트리 (폴더 구조)

텍스트로 표현하면 이렇게 생겼습니다:

**Windows:**
```
C:\
├── Users\
│   └── 홍길동\           ← 홈 디렉토리
│       ├── Desktop\       ← 바탕화면
│       ├── Documents\     ← 문서
│       ├── Downloads\     ← 다운로드
│       └── my-project\    ← 내 프로젝트 폴더
│           ├── index.html
│           ├── style.css
│           └── images\
│               └── logo.png
├── Program Files\
└── Windows\
```

**Mac:**
```
/
├── Users/
│   └── 홍길동/            ← 홈 디렉토리
│       ├── Desktop/        ← 바탕화면
│       ├── Documents/      ← 문서
│       ├── Downloads/      ← 다운로드
│       └── my-project/     ← 내 프로젝트 폴더
│           ├── index.html
│           ├── style.css
│           └── images/
│               └── logo.png
├── Applications/
└── System/
```

#### 절대 경로 vs 상대 경로

**절대 경로**는 처음부터 끝까지 전체 주소를 다 쓰는 것입니다:

```
Windows: C:\Users\홍길동\Documents\my-project\index.html
Mac:     /Users/홍길동/Documents/my-project/index.html
```

**상대 경로**는 "현재 내가 있는 위치"를 기준으로 쓰는 것입니다:

```
현재 위치가 my-project라면:
  index.html          ← 같은 폴더에 있는 파일
  images/logo.png     ← 하위 폴더의 파일
  ../                 ← 한 칸 위(상위) 폴더
```

> **비유**: 절대 경로는 "서울시 강남구 테헤란로 123번지"처럼 전체 주소를 말하는 것이고, 상대 경로는 "여기서 오른쪽 두 번째 건물"처럼 현재 위치 기준으로 말하는 것입니다.

#### Windows 경로 vs Mac 경로

| 구분 | Windows | Mac/Linux |
|------|---------|-----------|
| 구분자 | `\` (역슬래시) | `/` (슬래시) |
| 루트(최상위) | `C:\` | `/` |
| 홈 디렉토리 | `C:\Users\이름` | `/Users/이름` |
| 홈 축약 | `%USERPROFILE%` | `~` (물결표) |
| 예시 | `C:\Users\홍길동\Desktop` | `~/Desktop` |

> Mac에서 `~`(물결표, 틸드)는 홈 디렉토리를 의미합니다. `cd ~`이라고 치면 어디에 있든 홈으로 돌아갑니다. Windows PowerShell에서도 `~`이 작동합니다!

#### 자주 하는 실수: 경로에 한글이나 공백 넣기

경로에 한글이나 공백이 있으면 문제가 생길 수 있습니다:

```
# 문제가 될 수 있는 경로
C:\Users\홍길동\내 프로젝트\새 폴더\index.html

# 더 안전한 경로
C:\Users\홍길동\my-project\index.html
```

**주의사항:**

- 프로젝트 폴더 이름은 **영문, 숫자, 하이픈(-)** 만 사용하세요
- 공백 대신 **하이픈(-)**이나 **밑줄(_)**을 사용하세요
- 사용자 이름에 한글이 들어가 있는 경우 일부 도구에서 오류가 날 수 있습니다

> "처음 프로젝트 폴더 이름을 한글로 지었다가 나중에 오류가 나서 다 바꿨어요. 처음부터 영어로 짓는 게 편합니다." — 실사용자 경험

### 실습 과제

현재 위치를 확인하고 폴더 내용을 봅시다.

**Windows (PowerShell):**
```powershell
# 현재 위치 확인
pwd

# 현재 폴더의 내용 보기
ls
```

**Mac (터미널):**
```bash
# 현재 위치 확인
pwd

# 현재 폴더의 내용 보기
ls
```

> `pwd`는 "Print Working Directory"의 약자로, "현재 내가 어디에 있는지" 알려줍니다. 화면에 `/Users/홍길동` 같은 경로가 출력되면 성공입니다!

- [ ] `ls` 명령어로 폴더 내용을 확인했는가?
- [ ] 절대 경로와 상대 경로의 차이를 설명할 수 있는가?
- [ ] 프로젝트 폴더 이름에 한글/공백을 피해야 하는 이유를 이해했는가?

## 6. 첫 번째 명령어 — 이동하고 만들기

### 학습 목표

- `cd`, `mkdir`, `ls`/`dir`, `cd ..`, `clear`/`cls` 5가지 명령어를 사용할 수 있다
- 폴더를 만들고, 그 안으로 이동하고, 다시 돌아올 수 있다

### 상세 설명

명령어를 하나씩 배워봅시다. 어렵지 않습니다. 하나씩 따라 하면 됩니다!

#### 명령어 1: `cd` — 폴더 이동 (Change Directory)

"cd"는 다른 폴더로 이동하는 명령어입니다. 파일 탐색기에서 폴더를 더블클릭하는 것과 같습니다.

```bash
cd Desktop
```

**입력하면 무슨 일이 일어나나요?**
- 프롬프트의 경로 부분이 바뀝니다
- 예: `PS C:\Users\홍길동>` → `PS C:\Users\홍길동\Desktop>`
- 이제 바탕화면(Desktop) 폴더 안에 있다는 뜻입니다

#### 명령어 2: `mkdir` — 폴더 만들기 (Make Directory)

새로운 폴더를 만드는 명령어입니다.

```bash
mkdir my-first-project
```

**입력하면 무슨 일이 일어나나요?**
- 아무 메시지 없이 다음 줄로 넘어갑니다 (에러가 없으면 성공!)
- 파일 탐색기를 열어보면 "my-first-project" 폴더가 새로 만들어져 있습니다

> CLI의 특징: 명령이 성공하면 대부분 아무 말 없이 다음 줄로 넘어갑니다. "조용하면 성공"이라고 기억하세요!

#### 명령어 3: `ls` / `dir` — 목록 보기 (List)

현재 폴더에 어떤 파일과 폴더가 있는지 보여줍니다.

```bash
# PowerShell / Mac
ls

# CMD (Windows 전용)
dir
```

**출력 결과 예시 (PowerShell):**
```
    디렉터리: C:\Users\홍길동\Desktop

Mode                 LastWriteTime         Length Name
----                 -------------         ------ ----
d----          2026-03-09    10:30                my-first-project
```

- `d----`의 `d`는 디렉토리(폴더)를 의미합니다
- 방금 만든 `my-first-project` 폴더가 보입니다!

#### 명령어 4: `cd ..` — 상위 폴더로 이동

`..`은 "한 칸 위" 폴더를 의미합니다. 뒤로가기 버튼과 같습니다.

```bash
cd ..
```

**입력하면 무슨 일이 일어나나요?**
- 프롬프트의 경로가 한 단계 위로 변경됩니다
- 예: `PS C:\Users\홍길동\Desktop>` → `PS C:\Users\홍길동>`

> `cd` 뒤에 `..(점 두 개)`을 입력합니다. `cd`와 `..` 사이에 공백이 있어야 합니다!

#### 명령어 5: `clear` / `cls` — 화면 지우기

터미널 화면이 지저분해지면 깨끗하게 지울 수 있습니다.

```bash
# PowerShell / Mac
clear

# CMD (Windows 전용)
cls
```

> 화면이 깨끗해지고 프롬프트만 남습니다. 이전에 입력한 내용이 사라진 것처럼 보이지만, 실제로 작업 내용은 그대로입니다. 화면 정리만 된 것이에요.

#### 명령어 비교표

| 하고 싶은 것 | Bash (Mac/Linux) | CMD (Windows) | PowerShell (Windows) |
|-------------|-----------------|--------------|---------------------|
| 폴더 이동 | `cd 이름` | `cd 이름` | `cd 이름` |
| 폴더 만들기 | `mkdir 이름` | `mkdir 이름` | `mkdir 이름` |
| 목록 보기 | `ls` | `dir` | `ls` 또는 `dir` |
| 상위 이동 | `cd ..` | `cd ..` | `cd ..` |
| 화면 지우기 | `clear` | `cls` | `clear` 또는 `cls` |

> 좋은 소식! `cd`, `mkdir`, `cd ..`는 세 가지 셸 모두 동일합니다!

### 실습 과제 (5단계)

아래 단계를 순서대로 따라 해보세요:

```bash
# 1단계: 바탕화면으로 이동
cd Desktop

# 2단계: 새 폴더 만들기
mkdir cli-practice

# 3단계: 만든 폴더 안으로 이동
cd cli-practice

# 4단계: 현재 위치 확인 (비어 있을 것입니다)
ls

# 5단계: 상위 폴더(바탕화면)로 돌아가기
cd ..
```

**Mac 사용자 참고:** Mac의 바탕화면 폴더 이름은 영어 시스템이면 `Desktop`, 한국어 시스템이면 `cd ~/Desktop`으로 이동하세요.

### 체크포인트

- [ ] `cd Desktop`으로 바탕화면에 이동했는가?
- [ ] `mkdir cli-practice`로 폴더를 만들었는가?
- [ ] `ls`로 만든 폴더가 보이는가?
- [ ] `cd ..`로 다시 돌아올 수 있는가?

> 축하합니다! 이 5개 명령어만 알면 폴더를 자유롭게 돌아다닐 수 있습니다. 이것만으로도 절반은 온 것입니다!

## 7. 두 번째 명령어 — 파일 만들고 지우기

### 학습 목표

- 터미널에서 파일을 만들고, 내용을 확인하고, 복사/이동/삭제할 수 있다
- 파일 삭제 시 주의사항을 이해한다

### 상세 설명

이제 폴더 안에서 **파일**을 다뤄봅시다.

#### 파일 만들기

**Windows (PowerShell):**
```powershell
# 빈 파일 만들기
New-Item hello.txt

# 내용이 있는 파일 만들기
"안녕하세요, 첫 번째 파일입니다!" | Out-File hello.txt
```

**Mac (터미널):**
```bash
# 빈 파일 만들기
touch hello.txt

# 내용이 있는 파일 만들기
echo "안녕하세요, 첫 번째 파일입니다!" > hello.txt
```

> 파일 이름 뒤의 `.txt`는 **확장자**라고 합니다. 파일의 종류를 나타냅니다 (`.txt`=텍스트, `.html`=웹페이지, `.css`=스타일시트).

#### 파일 내용 보기

```bash
# PowerShell
Get-Content hello.txt
# 또는 줄여서
cat hello.txt

# CMD
type hello.txt

# Mac
cat hello.txt
```

화면에 "안녕하세요, 첫 번째 파일입니다!"가 출력되면 성공입니다.

#### 파일 복사

```bash
# PowerShell / Mac
cp hello.txt hello-copy.txt
# PowerShell 정식 명령
Copy-Item hello.txt hello-copy.txt

# CMD
copy hello.txt hello-copy.txt
```

`ls`로 확인하면 `hello.txt`와 `hello-copy.txt` 두 파일이 보입니다.

#### 파일 이동 / 이름 바꾸기

"이동"과 "이름 바꾸기"는 같은 명령어를 사용합니다:

```bash
# PowerShell / Mac: 이름 바꾸기
mv hello-copy.txt greeting.txt
# PowerShell 정식 명령
Move-Item hello-copy.txt greeting.txt
# 또는 이름 바꾸기 전용
Rename-Item hello-copy.txt greeting.txt

# CMD: 이름 바꾸기
ren hello-copy.txt greeting.txt
```

#### 파일 삭제

```bash
# PowerShell / Mac
rm greeting.txt
# PowerShell 정식 명령
Remove-Item greeting.txt

# CMD
del greeting.txt
```

> **주의!** 터미널에서 `rm`(또는 `del`)으로 삭제한 파일은 **휴지통에 가지 않습니다**. 즉시 영구 삭제됩니다! 파일 탐색기에서 삭제하면 휴지통에 들어가지만, 터미널은 그렇지 않으니 주의하세요.

#### 와일드카드(*) — 조심해서 사용하세요

`*`(별표)는 "모든 것"을 의미합니다:

```bash
# *.txt = 모든 txt 파일
rm *.txt   # 현재 폴더의 모든 txt 파일 삭제!
```

> **경고**: `rm *`은 현재 폴더의 모든 파일을 삭제합니다. 절대 무심코 사용하지 마세요! 특히 `rm -rf *`는 하위 폴더까지 모조리 삭제하는 매우 위험한 명령입니다.

#### 파일 명령어 비교표

| 하고 싶은 것 | Bash (Mac/Linux) | CMD (Windows) | PowerShell (Windows) |
|-------------|-----------------|--------------|---------------------|
| 빈 파일 만들기 | `touch 파일` | `type nul > 파일` | `New-Item 파일` |
| 파일 내용 보기 | `cat 파일` | `type 파일` | `cat 파일` 또는 `Get-Content 파일` |
| 파일 복사 | `cp 원본 사본` | `copy 원본 사본` | `cp 원본 사본` 또는 `Copy-Item` |
| 파일 이동/이름 변경 | `mv 원래 새이름` | `ren 원래 새이름` | `mv 원래 새이름` 또는 `Move-Item` |
| 파일 삭제 | `rm 파일` | `del 파일` | `rm 파일` 또는 `Remove-Item` |

### 실습 과제

아래 단계를 순서대로 따라 해보세요 (먼저 `cli-practice` 폴더로 이동하세요):


## 8. PowerShell 더 알아보기

### 학습 목표

- PowerShell의 Cmdlet(커맨들릿) 이름 규칙을 이해한다
- 자주 쓰는 10가지 Cmdlet을 안다
- Tab 자동완성과 Get-Help를 활용할 수 있다
- 실행 정책(Execution Policy)이 무엇인지 이해한다

### 상세 설명

> 이 섹션은 **Windows 사용자**를 위한 내용입니다. Mac 사용자는 Bash/zsh 관련 팁을 참고하세요.

#### Cmdlet(커맨들릿)이란?

PowerShell의 명령어를 **Cmdlet**(커맨들릿)이라고 합니다. 독특한 이름 규칙이 있습니다:

```
동사-명사 (Verb-Noun)
```

마치 영어 문장처럼 읽힙니다:

- `Get-ChildItem` → "자식 항목을 가져와라" → 폴더 내용 보기
- `Set-Location` → "위치를 설정해라" → 폴더 이동
- `New-Item` → "새 항목을 만들어라" → 파일/폴더 생성
- `Remove-Item` → "항목을 제거해라" → 삭제

> Cmdlet 이름이 길어서 불편해 보이지만, 규칙이 명확하여 모르는 명령어도 의미를 추측할 수 있습니다. 예를 들어 `Copy-Item`이 무슨 뜻인지 짐작이 가시죠?

#### 자주 쓰는 Cmdlet Top 10

| 번호 | Cmdlet | 별명(Alias) | 기능 | 예시 |
|------|--------|------------|------|------|
| 1 | `Get-ChildItem` | `ls`, `dir` | 폴더 내용 보기 | `ls` |
| 2 | `Set-Location` | `cd` | 폴더 이동 | `cd Documents` |
| 3 | `Get-Location` | `pwd` | 현재 위치 확인 | `pwd` |
| 4 | `New-Item` | `ni` | 파일/폴더 만들기 | `ni test.txt` |
| 5 | `Remove-Item` | `rm`, `del` | 삭제 | `rm test.txt` |
| 6 | `Copy-Item` | `cp`, `copy` | 복사 | `cp a.txt b.txt` |
| 7 | `Move-Item` | `mv`, `move` | 이동/이름 변경 | `mv a.txt b.txt` |
| 8 | `Get-Content` | `cat`, `type` | 파일 내용 보기 | `cat hello.txt` |
| 9 | `Clear-Host` | `clear`, `cls` | 화면 지우기 | `clear` |
| 10 | `Get-Help` | `help` | 도움말 보기 | `Get-Help ls` |

> 별명(Alias) 덕분에 평소에는 `ls`, `cd`, `cat` 같은 짧은 명령어를 쓰면 됩니다. Cmdlet 정식 이름은 스크립트를 작성할 때 사용합니다.

#### Tab 자동완성

긴 명령어나 파일 이름을 다 타이핑할 필요가 없습니다. **Tab 키**를 누르면 자동으로 완성해줍니다!

```powershell
# "Get-Chi"만 입력하고 Tab을 누르면:
Get-Chi[Tab] → Get-ChildItem

# 파일 이름도 자동완성됩니다:
cat hel[Tab] → cat hello.txt

# 여러 후보가 있으면 Tab을 반복해서 누릅니다
```

> Tab 자동완성은 오타를 줄여주는 매우 유용한 기능입니다. 습관적으로 Tab을 누르세요!

#### Get-Help — 명령어 사용법 보기

모르는 명령어의 사용법을 터미널에서 바로 확인할 수 있습니다:

```powershell
# 기본 도움말
Get-Help Get-ChildItem

# 예시 포함
Get-Help Get-ChildItem -Examples

# 상세 도움말
Get-Help Get-ChildItem -Detailed
```

처음 실행 시 "도움말을 업데이트하시겠습니까?"라는 메시지가 나올 수 있습니다. "Y"를 입력하면 됩니다.

#### 실행 정책 (Execution Policy)

PowerShell 스크립트를 실행하려 할 때 이런 오류가 나올 수 있습니다:

```
이 시스템에서 스크립트를 실행할 수 없습니다...
```

이것은 보안을 위한 **실행 정책** 때문입니다. 해결 방법:

```powershell
# 현재 정책 확인
Get-ExecutionPolicy

# 로컬 스크립트 실행 허용 (관리자 권한 필요)
Set-ExecutionPolicy RemoteSigned
```

- `Restricted`: 스크립트 실행 불가 (기본값)
- `RemoteSigned`: 로컬 스크립트는 실행 가능, 다운로드한 것은 서명 필요
- `Unrestricted`: 모든 스크립트 실행 가능 (보안상 비추천)

> 바이브코딩 도구(특히 Cursor, Claude Code)를 사용할 때 이 오류를 만나면, `RemoteSigned`로 변경하면 해결됩니다.

#### 종합 비교표: PowerShell vs CMD vs Bash

| 기능 | PowerShell | CMD | Bash (Mac/Linux) |
|------|-----------|-----|-----------------|
| 명령어 스타일 | 동사-명사 (Cmdlet) | 짧은 약어 | 짧은 약어 |
| 파이프라인 | 객체 전달 | 텍스트 전달 | 텍스트 전달 |
| 자동완성 | Tab (강력) | Tab (기본) | Tab (강력) |
| 스크립트 확장자 | `.ps1` | `.bat`, `.cmd` | `.sh` |
| 도움말 | `Get-Help` | `/?` 플래그 | `man`, `--help` |
| 대소문자 | 구분 안 함 | 구분 안 함 | 구분함 |
| 변수 | `$변수` | `%변수%` | `$변수` |
| 주석 | `# 주석` | `REM 주석` | `# 주석` |

> Mac에서도 PowerShell을 설치할 수 있습니다! 하지만 Mac에서는 기본 zsh/bash를 사용하는 것이 일반적입니다.

### 실습 과제

```powershell
# 1. Get-Help로 Get-ChildItem의 사용법을 확인해보세요
Get-Help Get-ChildItem -Examples

# 2. Tab 자동완성을 연습해보세요

# 3. 현재 실행 정책을 확인해보세요

```

## 9. VS Code 설치 (단계별 안내)

### 학습 목표

- VS Code를 다운로드하여 설치할 수 있다
- 한국어 팩과 기본 설정을 적용할 수 있다
- 필수 확장 프로그램을 설치할 수 있다

### 상세 설명

#### 왜 VS Code인가?

"메모장으로도 코드를 쓸 수 있지 않나요?" 맞습니다. 하지만:

| 메모장 | VS Code |
|--------|---------|
| 모든 글자가 같은 색 | 코드 종류별로 색상이 달라 읽기 쉬움 |
| 오타를 찾기 어려움 | 오타를 빨간 밑줄로 표시 |
| 파일 하나만 열림 | 프로젝트 전체 파일을 한눈에 |
| 터미널을 따로 열어야 함 | 에디터 안에서 터미널 사용 가능 |
| 확장 기능 없음 | 수천 개의 확장 프로그램 |

> VS Code는 **무료**이고, 세계에서 가장 많이 사용하는 코드 에디터입니다. Cursor도 VS Code를 기반으로 만들어졌으므로, VS Code를 배워두면 Cursor로 넘어가는 것이 매우 수월합니다.

#### Windows 설치 방법

1. **다운로드**: 브라우저에서 `https://code.visualstudio.com` 접속
2. 파란색 **"Download for Windows"** 버튼 클릭
3. 다운로드된 설치 파일(`VSCodeUserSetup-x64-*.exe`) 실행
4. **설치 옵션 화면**에서 아래 4개를 모두 체크하세요:

```
[v] "Code(으)로 열기" 작업을 Windows 탐색기 파일 상황에 맞는 메뉴에 추가
[v] "Code(으)로 열기" 작업을 Windows 탐색기 디렉터리 상황에 맞는 메뉴에 추가
[v] 지원되는 파일 형식에 대한 편집기로 Code를 등록합니다
[v] PATH에 추가(다시 시작한 후 사용 가능)    ← 이것이 특히 중요!
```

5. "설치" 버튼 클릭 → 완료까지 기다림
6. "Visual Studio Code 시작" 체크 후 "마침"

> **PATH에 추가** 옵션은 반드시 체크하세요! 이것을 체크해야 터미널에서 `code .` 명령어로 VS Code를 열 수 있습니다. 놓쳤다면 14번(트러블슈팅) 섹션을 참고하세요.

#### Mac 설치 방법

1. **다운로드**: 브라우저에서 `https://code.visualstudio.com` 접속
2. **"Download for Mac"** 버튼 클릭
3. 다운로드된 `.zip` 파일의 압축이 자동으로 풀립니다
4. "Visual Studio Code" 앱을 **Applications** 폴더로 드래그
5. Applications에서 VS Code를 더블클릭하여 실행

> Mac에서는 처음 실행 시 "인터넷에서 다운로드된 앱입니다. 열겠습니까?" 경고가 나옵니다. "열기"를 클릭하세요.

#### 첫 실행 화면

VS Code를 처음 열면 **Welcome 탭**이 나타납니다:

- 왼쪽에 **사이드바** (파일 목록이 표시될 영역)
- 가운데에 **Welcome 페이지** (시작 가이드)
- 아래쪽에 **패널 영역** (터미널, 출력 등이 나올 곳)
- 맨 아래에 **상태 표시줄** (파란색 바)

> 영어로 표시되어 있어도 놀라지 마세요. 지금부터 한국어로 바꿔보겠습니다!

#### 5가지 초기 설정

**설정 1: 한국어 팩 설치**

1. 왼쪽 사이드바에서 네모 4개 모양 아이콘(Extensions) 클릭
   또는 단축키: `Ctrl+Shift+X` (Mac: `Cmd+Shift+X`)
2. 검색창에 "Korean" 입력
3. "Korean Language Pack for VS Code" 항목의 "Install" 버튼 클릭

1. `Ctrl+K, Ctrl+T` (Mac: `Cmd+K, Cmd+T`)

오른쪽에 나오는 작은 코드 미리보기(미니맵)가 불필요하다면:

## 10. VS Code 화면 구성과 핵심 기능

### 학습 목표

- VS Code의 5가지 주요 영역을 구분할 수 있다
- 필수 단축키 10개를 사용할 수 있다
- Emmet으로 HTML 기본 구조를 빠르게 만들 수 있다

### 상세 설명

#### VS Code의 5가지 주요 영역

```
+--+------------------------------------------+
|사 |          에디터 영역                       |
|이 |    (코드를 작성하는 곳)                     |
|드 |                                           |
|바 |                                           |
|  |                                           |
|  +------------------------------------------+
|  |     통합 터미널 / 패널                      |
|  |  (터미널, 출력, 문제 등)                     |
+--+------------------------------------------+
|           상태 표시줄 (맨 아래 파란 바)           |
+------------------------------------------------+
        ↑ 상단: 명령 팔레트 (Ctrl+Shift+P)
```

1. **사이드바 (왼쪽)**: 파일 탐색기, 검색, Git, 확장 등의 패널
2. **에디터 영역 (가운데)**: 코드를 작성하고 편집하는 핵심 공간
3. **통합 터미널 (아래)**: VS Code 안에서 바로 사용하는 터미널
4. **명령 팔레트 (상단)**: 모든 명령을 검색하고 실행 (`Ctrl+Shift+P`)
5. **확장 마켓 (사이드바)**: 확장 프로그램 검색 및 설치

#### 필수 단축키 10선

| 번호 | 기능 | Windows | Mac |
|------|------|---------|-----|
| 1 | 명령 팔레트 열기 | `Ctrl+Shift+P` | `Cmd+Shift+P` |
| 2 | 파일 빠른 열기 | `Ctrl+P` | `Cmd+P` |
| 3 | 새 파일 | `Ctrl+N` | `Cmd+N` |
| 4 | 저장 | `Ctrl+S` | `Cmd+S` |
| 5 | 되돌리기 | `Ctrl+Z` | `Cmd+Z` |
| 6 | 통합 터미널 열기/닫기 | `` Ctrl+` `` | `` Cmd+` `` |
| 7 | 사이드바 토글 | `Ctrl+B` | `Cmd+B` |
| 8 | 파일 내 검색 | `Ctrl+F` | `Cmd+F` |
| 9 | 전체 검색 | `Ctrl+Shift+F` | `Cmd+Shift+F` |
| 10 | 확장 프로그램 | `Ctrl+Shift+X` | `Cmd+Shift+X` |

> 한 번에 다 외우지 않아도 됩니다. 먼저 **저장(`Ctrl+S`)**, **터미널 열기(`` Ctrl+` ``)**, **명령 팔레트(`Ctrl+Shift+P`)** 3개만 기억하세요!

#### 파일 만들기 / 열기 / 저장

**새 파일 만들기:**
1. `Ctrl+N`으로 새 파일 생성
2. 내용 입력
3. `Ctrl+S`로 저장 → 파일 이름과 위치 지정

**폴더 열기 (프로젝트 열기):**
1. 메뉴 "파일" → "폴더 열기"
2. 또는 터미널에서 `code 폴더경로` 입력
3. 사이드바에 폴더 구조가 나타남

> VS Code에서는 "폴더"를 여는 것이 "프로젝트"를 여는 것과 같습니다.

#### Emmet — HTML을 빠르게 만드는 마법

VS Code에는 **Emmet**이라는 기능이 내장되어 있습니다. HTML 파일에서 특수한 약어를 입력하고 `Tab`을 누르면 자동으로 코드가 완성됩니다.

**가장 유용한 Emmet 단축키:**

`.html` 파일에서 `!`를 입력하고 `Tab`을 누르면:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>

</body>
</html>
```

이 전체 코드가 자동으로 생성됩니다! 일일이 타이핑할 필요가 없습니다.

### 실습 과제

1. VS Code에서 새 파일을 만드세요 (`Ctrl+N`)
2. `Ctrl+S`를 눌러 `index.html`로 저장하세요
3. `!`를 입력하고 `Tab`을 누르세요 (Emmet)
4. `<body>` 태그 안에 `<h1>안녕하세요!</h1>`을 입력하세요
5. `Ctrl+S`로 저장하세요
6. Live Server 확장이 설치되어 있다면: 에디터에서 마우스 오른쪽 클릭 → "Open with Live Server"

> 브라우저에 "안녕하세요!"가 크게 표시되면 성공입니다! 코드를 수정하면 브라우저에 즉시 반영됩니다.

## 11. VS Code에서 터미널 사용하기

### 학습 목표

- VS Code 통합 터미널을 열고 사용할 수 있다
- 기본 셸을 변경할 수 있다 (CMD → PowerShell → Git Bash)
- 여러 터미널을 동시에 사용할 수 있다
- `code .` 명령어를 사용할 수 있다

### 상세 설명

#### 통합 터미널 열기

VS Code 안에서 터미널을 바로 사용할 수 있습니다. 별도의 터미널 앱을 열 필요가 없어요!

**여는 방법:**
- 단축키: `` Ctrl+` `` (키보드 왼쪽 위, 숫자 1 옆의 백틱 키)
- 또는 메뉴: "보기" → "터미널"
- 또는 메뉴: "터미널" → "새 터미널"

> VS Code 하단에 터미널 패널이 나타납니다. 여기서 앞에서 배운 모든 명령어(`cd`, `ls`, `mkdir` 등)를 그대로 사용할 수 있습니다.

#### 기본 셸 변경하기

VS Code 터미널의 기본 셸을 변경하는 방법입니다:

**방법 1: 명령 팔레트 사용**

1. `Ctrl+Shift+P` (Mac: `Cmd+Shift+P`)로 명령 팔레트 열기
2. "Terminal: Select Default Profile" 입력
3. 목록에서 원하는 셸 선택:
   - **PowerShell** (추천)
   - **Command Prompt** (CMD)
   - **Git Bash** (Git 설치 후 사용 가능)

**방법 2: 터미널 드롭다운**

1. 터미널 패널 오른쪽 상단의 `+` 버튼 옆 화살표(`v`) 클릭
2. "기본 프로필 선택" 클릭
3. 원하는 셸 선택

#### 여러 터미널 동시 사용

터미널을 여러 개 열어서 각각 다른 작업을 할 수 있습니다:

- **새 터미널 추가**: `+` 버튼 클릭 또는 `Ctrl+Shift+`(백틱)
- **터미널 분할**: 터미널 패널의 분할 아이콘 클릭 (두 터미널을 나란히)
- **터미널 전환**: 터미널 패널의 드롭다운에서 선택

> 예를 들어, 한 터미널에서는 서버를 실행하고, 다른 터미널에서는 Git 명령어를 사용할 수 있습니다.

#### `code .` — 터미널에서 VS Code 열기

터미널에서 현재 폴더를 VS Code로 열 수 있는 매우 편리한 명령어입니다:

```bash
# 현재 폴더를 VS Code로 열기
code .

# 특정 폴더를 VS Code로 열기
code ~/Desktop/my-project

# 특정 파일 열기
code index.html
```

**Windows:** 설치 시 "PATH에 추가" 옵션을 체크했다면 바로 작동합니다.

**Mac:** 처음 한 번 설정이 필요합니다:
1. VS Code에서 `Cmd+Shift+P` → "Shell Command: Install 'code' command in PATH" 선택
2. 터미널을 재시작하면 `code .`이 작동합니다

> `code .`은 개발자들이 가장 자주 사용하는 명령어 중 하나입니다. 터미널에서 프로젝트 폴더로 이동한 후 `code .`만 치면 VS Code가 해당 프로젝트를 열어줍니다!

### 실습 과제

```bash
# 1. VS Code 통합 터미널 열기
# Ctrl+` 누르기

# 2. 기본 셸을 PowerShell로 변경

# 3. 폴더 내용 확인
Get-ChildItem
# 또는
ls

# 4. 새 폴더 만들기
mkdir test-folder

# 5. 사이드바에서 test-folder가 나타나는지 확인
# (사이드바의 파일 탐색기에 자동 반영됨!)
```

> 터미널에서 `mkdir`로 폴더를 만들면, VS Code 사이드바에도 즉시 반영됩니다. 터미널과 GUI가 실시간으로 연동되는 것이 통합 터미널의 장점입니다!

## 12. Git — 내 코드의 타임머신

### 학습 목표

- Git이 무엇이고 왜 필요한지 이해한다
- Git을 설치할 수 있다
- `git init`, `git add`, `git commit`의 기본 흐름을 이해하고 실행할 수 있다

### 상세 설명

#### 비유: 게임 세이브 포인트

게임을 할 때 **세이브 포인트**를 저장하면, 실수해도 이전 상태로 돌아갈 수 있죠? Git은 코드의 세이브 포인트를 만들어주는 도구입니다.

- **세이브(Save)** = **커밋(Commit)**: 현재 상태를 저장
- **로드(Load)** = **체크아웃(Checkout)**: 이전 상태로 돌아가기
- **세이브 파일 목록** = **로그(Log)**: 저장 기록 보기

#### 왜 Git이 필요한가?

| Git 없이 | Git 있으면 |
|---------|----------|
| `보고서_최종.doc` | 히스토리 관리 자동 |
| `보고서_최종_수정.doc` | 어떤 변경이 있었는지 기록 |
| `보고서_최종_진짜최종.doc` | 실수하면 이전 버전으로 복구 |
| `보고서_최종_진짜최종2.doc` | 여러 사람이 동시에 작업 가능 |

> "처음에는 '나 혼자 하는데 Git이 필요할까?' 싶었는데, 코드를 잘못 수정했을 때 예전 버전으로 돌아갈 수 있어서 정말 유용했어요." — 실사용자 경험

#### Git 설치

**Windows:**

1. https://git-scm.com 에서 "Download for Windows" 클릭
2. 설치 파일 실행
3. 설치 옵션 대부분 기본값 유지 (Next 클릭)
4. 중요 옵션들:
   - **Default editor**: VS Code 선택 (드롭다운에서 찾기)
   - **Adjusting your PATH**: "Git from the command line and also from 3rd-party software" 선택 (기본값)
   - **Line ending**: "Checkout Windows-style, commit Unix-style" 선택 (기본값)
5. 설치 완료 후 터미널을 **재시작**

**Mac:**

방법 1 (가장 간단): 터미널에서 아래 명령어 입력
```bash
xcode-select --install
```
팝업에서 "설치" 클릭 → Xcode Command Line Tools와 함께 Git이 설치됩니다.

방법 2 (Homebrew 사용):
```bash
brew install git
```

**설치 확인:**
```bash
git --version
# 결과 예시: git version 2.44.0
```

버전 번호가 나오면 성공입니다!

**최초 설정 (이름과 이메일):**

Git은 "누가 이 변경을 했는지" 기록합니다. 처음 한 번만 설정하면 됩니다:

```bash
git config --global user.name "홍길동"
git config --global user.email "hong@example.com"
```

> 이메일은 나중에 GitHub 가입할 때 사용할 이메일과 같은 것을 넣는 게 좋습니다.

#### Git 3단계 워크플로우

Git의 기본 작업 흐름을 비유로 설명하겠습니다:

```
[작업 폴더]  →  [스테이지]  →  [저장소]
 파일 수정      git add       git commit
   ↓              ↓              ↓
사진 찍을      테이블에       사진 찍기!
물건 고르기    올려놓기       (기록 완료)
```

1. **파일 수정**: 코드를 작성하거나 수정합니다 (사진 찍을 물건을 고르는 단계)
2. **`git add`**: 저장할 파일을 선택합니다 (물건을 테이블에 올려놓는 단계)
3. **`git commit`**: 선택한 파일들의 현재 상태를 저장합니다 (사진을 찍는 단계!)

> 왜 바로 저장하지 않고 `add`와 `commit` 두 단계로 나뉠까요? 10개 파일을 수정했지만 5개만 저장하고 싶을 때가 있기 때문입니다. `add`로 원하는 것만 골라서 `commit`할 수 있습니다.

#### VS Code에서 Git 사용하기 (GUI 방식)

터미널 명령어가 어렵다면, VS Code의 Git 패널을 사용할 수 있습니다:

1. 사이드바에서 **소스 제어** 아이콘 클릭 (가지가 갈라지는 모양)
   - 또는 `Ctrl+Shift+G` (Mac: `Cmd+Shift+G`)
2. 변경된 파일 목록이 표시됩니다
3. 각 파일 옆의 **`+`** 버튼을 클릭하면 `git add`와 같습니다
4. 상단 텍스트 입력창에 커밋 메시지를 입력합니다
5. **체크(v) 버튼**을 클릭하면 `git commit`과 같습니다

> 터미널 명령어든, VS Code GUI든, 하는 일은 동일합니다. 편한 것을 사용하세요!

### 실습 과제

```bash
# 1단계: 프로젝트 폴더로 이동
cd ~/Desktop/cli-practice

# 2단계: Git 초기화 (이 폴더를 Git으로 관리하겠다고 선언)
git init

# 3단계: 현재 상태 확인
git status
# → hello.txt가 빨간색(Untracked)으로 표시됨

# 4단계: 파일을 스테이지에 올리기
git add hello.txt

# 5단계: 다시 상태 확인
git status
# → hello.txt가 초록색(Staged)으로 바뀜

# 6단계: 커밋 (저장!)
git commit -m "첫 번째 커밋: hello.txt 추가"

# 7단계: 로그 확인
git log
# → 방금 만든 커밋이 보임!
```

**커밋 메시지 작성 팁:**
- `-m` 뒤에 큰따옴표로 메시지를 씁니다
- 무엇을 했는지 간결하게 적습니다
- 예: `"로그인 페이지 추가"`, `"버그 수정: 버튼 클릭 오류"`, `"README 파일 작성"`

### 체크포인트

- [ ] `git --version`으로 Git 설치를 확인했는가?
- [ ] `git init`으로 저장소를 초기화했는가?
- [ ] `git add` → `git commit` 흐름을 실행했는가?

## 13. 종합 미니 프로젝트

### 학습 목표

- 지금까지 배운 모든 것(터미널, VS Code, Git)을 연결하여 하나의 프로젝트를 완성한다
- "나만의 자기소개 페이지"를 만들고 Git으로 2번 커밋한다

### 미션: 나만의 자기소개 페이지 만들기 + Git 커밋 2번

지금까지 배운 것을 총동원하여 자기소개 웹페이지를 만들어봅시다!

#### Step 1: PowerShell에서 프로젝트 폴더 만들기

```powershell
# 바탕화면으로 이동
cd ~/Desktop

# 프로젝트 폴더 생성
mkdir my-profile

# 폴더로 이동
cd my-profile

# 현재 위치 확인
pwd
# → C:\Users\이름\Desktop\my-profile (또는 Mac: /Users/이름/Desktop/my-profile)
```

#### Step 2: VS Code로 프로젝트 열기

```powershell
# VS Code 열기
code .
```

VS Code가 열리면서 왼쪽 사이드바에 `my-profile` 폴더가 표시됩니다. 아직 비어있습니다.

#### Step 3: index.html 만들기

1. VS Code에서 `Ctrl+N`으로 새 파일 생성
2. `Ctrl+S`로 저장 → 파일 이름: `index.html`
3. `!` 입력 후 `Tab` → Emmet으로 HTML 기본 구조 자동 생성
4. 아래와 같이 내용을 수정합니다:

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>나의 자기소개</title>
    <style>
        body {
            font-family: sans-serif;
            max-width: 600px;
            margin: 50px auto;
            padding: 20px;
            background-color: #f5f5f5;
        }
        h1 {
            color: #333;
            border-bottom: 2px solid #007bff;
            padding-bottom: 10px;
        }
        .info {
            background: white;
            padding: 20px;
            border-radius: 8px;
            margin-top: 20px;
        }
    </style>
</head>
<body>
    <h1>안녕하세요! 저를 소개합니다</h1>
    <div class="info">
        <p><strong>이름:</strong> (여기에 이름)</p>
        <p><strong>관심사:</strong> (여기에 관심사)</p>
        <p><strong>한마디:</strong> 오늘 CLI와 VS Code를 배웠습니다!</p>
    </div>
</body>
</html>
```

5. `Ctrl+S`로 저장

#### Step 4: Live Server로 브라우저 확인

1. `index.html` 탭에서 마우스 오른쪽 클릭
2. "Open with Live Server" 선택
3. 브라우저가 열리면서 자기소개 페이지가 표시됩니다!

#### Step 5: Git 초기화 + 첫 번째 커밋

VS Code 통합 터미널(`` Ctrl+` ``)에서:

```bash
# Git 초기화
git init

# 파일 추가
git add index.html

# 첫 번째 커밋
git commit -m "자기소개 페이지 초안 생성"
```

#### Step 6: 내용 수정 + 두 번째 커밋

`index.html`에서 이름과 관심사를 실제 내용으로 수정하고:

```html
<p><strong>이름:</strong> 김코딩</p>
<p><strong>관심사:</strong> 바이브코딩, 교육, 여행</p>
<p><strong>한마디:</strong> CLI가 생각보다 재미있네요!</p>
```

저장 후 커밋:

```bash
# 변경 확인
git status

# 수정된 파일 추가
git add index.html

# 두 번째 커밋
git commit -m "자기소개 내용 보강"

# 커밋 기록 확인
git log --oneline
```

## 14. 트러블슈팅

### 학습 목표

### 상세 설명

#### 문제 1: "'xxx'은(는) 내부 또는 외부 명령... 으로 인식되지 않습니다"

**원인:** 해당 프로그램이 설치되지 않았거나, 시스템 PATH에 등록되지 않은 경우입니다.

**해결:**
- 프로그램이 실제로 설치되었는지 확인합니다
- 설치 시 "PATH에 추가" 옵션을 체크했는지 확인합니다
- 설치 후 터미널을 **닫았다가 다시 열어야** PATH가 적용됩니다
- 그래도 안 되면 시스템 환경변수에 수동으로 추가해야 합니다:
  - Windows: "시스템 환경 변수 편집" 검색 → "환경 변수" → "Path" 편집 → 프로그램 경로 추가

#### 문제 2: "이 시스템에서 스크립트를 실행할 수 없습니다" (PowerShell)

**원인:** PowerShell의 실행 정책(Execution Policy)이 Restricted로 설정되어 있습니다.

**해결:**
```powershell
# 관리자 권한 PowerShell에서 실행
Set-ExecutionPolicy RemoteSigned
# "Y"를 입력하고 Enter
```

#### 문제 3: 경로에 한글이나 공백이 있어서 오류 발생

**원인:** 일부 도구(특히 npm, pip, Git)는 경로에 한글이나 공백이 있으면 제대로 작동하지 않습니다.

**해결:**
- 프로젝트 폴더는 영문, 숫자, 하이픈만 사용하여 만듭니다
- 공백이 있는 경로는 큰따옴표로 감싸서 사용합니다: `cd "My Folder"`
- Windows 사용자 이름이 한글이라면, `C:\dev` 같은 짧은 경로에 프로젝트를 만드세요

#### 문제 4: `code .` 명령어가 작동하지 않음

**원인:** VS Code의 `code` 명령이 시스템 PATH에 등록되지 않았습니다.

**해결 (Windows):**
- VS Code를 재설치하면서 "PATH에 추가" 옵션 체크
- 또는 VS Code 설치 경로(예: `C:\Users\이름\AppData\Local\Programs\Microsoft VS Code\bin`)를 직접 PATH에 추가

**해결 (Mac):**
1. VS Code 열기
2. `Cmd+Shift+P` → "Shell Command: Install 'code' command in PATH" 선택
3. 터미널 재시작

#### 문제 5: Git 설치 후 `git` 명령어가 인식되지 않음

**원인:** Git 설치 후 터미널을 재시작하지 않았거나, PATH에 추가되지 않았습니다.

**해결:**
- 터미널을 완전히 닫고 새로 열어보세요
- VS Code도 재시작해보세요
- Windows의 경우 Git 설치 경로(보통 `C:\Program Files\Git\cmd`)가 PATH에 있는지 확인하세요
- 그래도 안 되면 Git을 재설치하세요 (기본 옵션 유지)

#### 문제 6: VS Code 한국어 팩이 적용되지 않음

**원인:** 한국어 팩 설치 후 VS Code를 재시작하지 않았거나, 표시 언어 설정이 변경되지 않았습니다.

**해결:**
1. `Ctrl+Shift+P` → "Configure Display Language" 입력
2. "ko" (한국어) 선택
3. VS Code 재시작

#### 문제 7: "권한이 거부되었습니다" / "Access is denied" 오류

**원인:** 관리자 권한이 필요한 작업을 일반 모드에서 실행했습니다.

**해결:**
- **Windows**: 터미널을 "관리자 권한으로 실행"합니다
  - 시작 메뉴 → "터미널" 검색 → 오른쪽 클릭 → "관리자 권한으로 실행"
- **Mac**: 명령어 앞에 `sudo`를 붙입니다
  ```bash
  sudo npm install -g something
  ```
  비밀번호를 입력하라는 메시지가 나옵니다 (입력해도 화면에 표시되지 않는 것이 정상입니다)

#### 문제 8: 터미널에 이상한 글자가 나옴 (깨진 글씨)

**원인:** 인코딩(문자 표시 방식) 설정이 맞지 않습니다.

**해결 (Windows PowerShell):**
```powershell
# 현재 인코딩 확인
[Console]::OutputEncoding

# UTF-8로 변경
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
chcp 65001
```

**해결 (CMD):**
```cmd
chcp 65001
```

> 매번 입력하기 귀찮다면, PowerShell 프로필에 자동으로 설정되도록 추가할 수 있습니다 (중급편에서 다룹니다).

## 15. 자주 묻는 질문 (FAQ)

### Q1. CMD와 PowerShell 중 뭘 써야 하나요?

**PowerShell을 추천합니다.** Windows의 최신 기본 셸이고, `ls`, `cd` 같은 Bash 명령어도 호환됩니다. CMD는 오래된 튜토리얼 참고용으로만 알아두세요.

### Q2. VS Code와 Cursor의 차이는 뭔가요?

Cursor는 VS Code를 기반(포크)으로 만든 AI 특화 코드 에디터입니다. 인터페이스가 거의 동일하므로, VS Code를 익히면 Cursor로 바로 넘어갈 수 있습니다. 차이점은 Cursor에 AI 대화 기능이 기본 내장되어 있다는 점입니다.

### Q3. Mac에도 PowerShell이 있나요?

네, Mac에서도 PowerShell을 설치할 수 있습니다. 하지만 Mac에서는 기본 zsh/bash를 사용하는 것이 일반적이고 자연스럽습니다. 굳이 PowerShell을 설치할 필요는 없습니다.

### Q4. 터미널에서 실수로 뭔가를 삭제했어요. 복구할 수 있나요?

안타깝게도, `rm` 명령어로 삭제한 파일은 휴지통을 거치지 않으므로 **일반적인 방법으로는 복구가 불가능**합니다. 이것이 바로 Git이 필요한 이유입니다! Git으로 커밋해둔 파일은 언제든 복구할 수 있습니다.

> 실수를 방지하려면: (1) 삭제 전에 `ls`로 먼저 확인, (2) 중요한 파일은 Git으로 커밋, (3) `rm *` 같은 와일드카드 삭제는 매우 신중하게 사용하세요.

### Q5. Git이 꼭 필요한가요?

바이브코딩만 할 거라면 당장은 아닙니다. 하지만 아래 상황에서는 필수입니다:
- 코드를 실수로 망가뜨렸을 때 복구하고 싶을 때
- GitHub에 코드를 올려서 공유하고 싶을 때
- Cursor, Claude Code 같은 도구를 사용할 때
- 다른 사람과 협업할 때

> 지금 바로 쓰지 않더라도, 기본 사용법(`init`, `add`, `commit`)은 익혀두는 것을 추천합니다.

### Q6. 영어를 몰라도 되나요?

명령어 자체는 영어이지만, 대부분 2~5글자의 짧은 약어입니다: `cd`(change directory), `ls`(list), `rm`(remove). 뜻을 한 번 알아두면 충분합니다. VS Code도 한국어 팩을 설치하면 메뉴가 모두 한국어로 표시됩니다.

### Q7. 코딩을 몰라도 이 교안을 볼 수 있나요?

네! 이 교안은 코딩을 전혀 모르는 분을 위해 만들어졌습니다. HTML을 조금 다루지만, 복사해서 붙여넣기만 하면 됩니다. 코드의 의미를 몰라도 따라 할 수 있도록 단계별로 설명합니다.

### Q8. CLI를 왜 배워야 하나요? GUI가 더 쉬운데요.

GUI는 눈에 보이는 것만 할 수 있지만, CLI는 보이지 않는 것도 할 수 있습니다. 예를 들어:
- 100개 파일의 이름을 한 번에 바꾸기
- 서버에 원격 접속하기
- 자동화 스크립트 실행하기
- 개발 도구(npm, git, pip 등) 사용하기

바이브코딩을 넘어서 실제 개발로 나아가려면, CLI는 반드시 필요한 기초 체력입니다.

### Q9. 환경변수가 뭔가요?

환경변수는 컴퓨터 전체에서 공유하는 **설정값**입니다. 가장 중요한 환경변수는 **PATH**입니다.

PATH는 "이 프로그램을 어디서든 실행할 수 있게 해줘"라는 목록입니다. 예를 들어 `git`이라고만 쳐도 실행되는 이유는, Git의 설치 경로가 PATH에 등록되어 있기 때문입니다.

```powershell
# 현재 PATH 확인 (PowerShell)
$env:PATH -split ';'
```

### Q10. Node.js를 왜 설치해야 하나요?

Node.js는 JavaScript를 웹 브라우저 밖에서도 실행할 수 있게 해주는 프로그램입니다. 직접 사용하지 않더라도, 많은 개발 도구(npm, Vite, React 등)가 Node.js 위에서 동작하기 때문에 설치가 필요합니다.

> 비유하면, Node.js는 "전기"와 같습니다. 전기 자체를 직접 쓰지는 않지만, 전기가 있어야 세탁기(npm), 냉장고(React) 같은 가전제품이 작동하는 것이죠.

이 교안에서는 Node.js 설치를 다루지 않지만, Cursor나 Claude Code를 사용하게 되면 필요합니다. 해당 도구의 교안에서 자세히 안내합니다.
