# Phase 7: 개발 인터페이스 + Codex(ChatGPT) 교안 + AI 비교 페이지 추가

## Context

Phase 6 완료로 5개 바이브코딩 도구가 3수준 교안을 갖추었다. Phase 7에서는 3가지를 추가한다:

1. **개발 인터페이스(CLI/IDE/CMD)** 교안 3수준 — 기초 지식 카테고리
2. **Codex(ChatGPT)** 교안 3수준 — 7번째 바이브코딩 도구
3. **Claude vs ChatGPT 비교** 페이지 — AI 플랫폼 장단점 비교

### 핵심 설계 원칙 (사용자 지정)

1. **도구별 분리 ❌ → 역량 기준 통합 ✅**: CLI/IDE/CMD를 단계마다 같이 다루되 깊이가 달라짐
2. **각 섹션 내부 구조**: 학습 목표 → 상세 설명 → 실습 과제 → 퀴즈
3. **학습 진행 흐름**: 파일 조작 → 개발 실행 → Git → 자동화 → DevOps
4. **단계 설계**: 초반(분리 학습) → 중반(통합 활용) → 후반(자동화)

### 2-1. 초보편 — "CLI와 IDE로 시작하는 개발 환경 기초"

```
과정명: CLI와 IDE로 시작하는 개발 환경 기초
대상: 터미널/CLI를 한 번도 써본 적 없는 완전 입문자
목표: 명령줄 환경과 IDE를 활용하여 기본적인 개발 작업을 수행할 수 있다
주요 도구: Visual Studio Code, Git, Windows CMD·PowerShell / Mac 터미널
소요: 약 4~5시간 (설치 시간 포함)
```

> **초보편 작성 원칙** (중급·고급과 차별화)
> - 모든 단계에 **"왜 이것을 배우는가?"** 설명을 먼저 제시
> - 전문 용어는 **처음 등장 시 반드시 쉬운 비유**와 함께 설명 (예: "셸은 운영체제와 대화하는 통역사")
> - 명령어마다 **입력 전·후 화면 변화**를 텍스트로 상세 묘사 (스크린샷 대체)
> - 각 섹션 끝에 **✅ 체크포인트** (학습자가 스스로 확인할 수 있는 자가 점검 항목)
> - **흔한 실수와 해결법**을 각 섹션에 인라인으로 포함 (트러블슈팅 섹션까지 기다리지 않음)
> - OS별(Windows/Mac) 차이가 있는 곳은 **항상 양쪽 다 설명** (탭 또는 표로 분리)
> - 격려 어투 사용: "어렵지 않습니다", "처음엔 누구나 헷갈립니다"
