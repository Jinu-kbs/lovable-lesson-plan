AI Studio의 위치

AI Studio 빌드는 "설치 없이 브라우저에서 가장 빠르게 프로토타입을 만들 수 있는 도구"입니다. 초보자편에서는 한 페이지 프로토타입을 만들었다면, 중급자편에서는 기획 문서를 기반으로 더 복잡한 구조의 앱을 만들어봅니다.

2. AI Studio 심화 워크플로우

초보자 vs 중급자 워크플로우

초보자편에서는 "프롬프트 → 프리뷰 → 수정"의 단순한 흐름을 사용했습니다. 중급자편에서는 기획 단계를 추가하여 더 체계적으로 접근합니다.

단계초보자중급자
1단계바로 프롬프트 입력기획 문서(PRD/IA) 먼저 작성
2단계결과 보며 즉흥 수정기획 기반 구조화된 프롬프트
3단계완성되면 공유반복 수정 (3~5회 체계적 개선)
4단계—코드 검토 및 Export
5단계—외부 배포 (GitHub Pages / Vercel)

5단계 반복 개선 워크플로우

1단계: 기획
  ↓ PRD/IA 작성 (Gemini Gem 활용)
2단계: 초기 빌드
  ↓ AI Studio Build 탭 → 기획 기반 프롬프트 입력
3단계: 구조 검토
  ↓ 레이아웃, 섹션 구성, 네비게이션 확인
4단계: 반복 수정 (3~5회)
  ↓ 세부 디자인, 기능, 인터랙션 개선
5단계: 완성 및 배포
  ↓ 코드 Export → 로컬 테스트 → 배포

프롬프트 체이닝 전략

복잡한 앱을 만들 때는 하나의 프롬프트로 모든 것을 요청하지 마세요. 프롬프트를 단계별로 나누어 보내는 "프롬프트 체이닝" 전략이 효과적입니다.

[1번째 프롬프트] 전체 레이아웃과 네비게이션 구조를 먼저 만들어줘.
[2번째 프롬프트] 첫 번째 섹션(대시보드)의 콘텐츠를 채워줘.
[3번째 프롬프트] 두 번째 섹션(목록 페이지)을 추가해줘.
[4번째 프롬프트] 디자인을 통일하고 반응형으로 수정해줘.
[5번째 프롬프트] 인터랙션(버튼 클릭, 필터링 등)을 추가해줘.
활용 가이드: "큰 집을 짓듯이, 먼저 뼈대(구조)를 잡고 → 벽(콘텐츠)을 채우고 → 인테리어(디자인)를 하는 순서가 좋습니다." — 활용 가이드

Gemini API 키 설정 (선택)

AI Studio 빌드 탭에서는 API 키 없이도 사용할 수 있습니다. 하지만 코드를 Export한 후 로컬에서 Gemini API를 직접 호출하고 싶다면 API 키가 필요합니다.

  1. AI Studio 좌측 메뉴에서 Get API key 클릭
  2. Create API key 버튼 클릭
  3. 발급된 키를 안전한 곳에 저장
주의: API 키는 절대 공개 저장소에 올리지 마세요. 코드에 직접 쓰지 말고 환경변수로 관리하세요.

4. 기획 문서 작성 실습

PRD / IA 개념 (AI Studio 버전)

기획 문서를 먼저 작성하면 AI Studio에서 훨씬 정확한 결과물을 얻을 수 있습니다. AI Studio에서는 ERD(데이터 구조)까지는 불필요하고, PRD와 IA 수준이면 충분합니다.

문서역할AI Studio에서의 활용
PRD무엇을 만들 것인가?프로젝트 목적, 기능 목록, 디자인 요구사항 정리
IA어떻게 구조를 잡을 것인가?섹션 구성, 탭 네비게이션, 콘텐츠 배치 계획
참고: 러버블이나 안티그래비티에서는 ERD(데이터 구조)도 작성하지만, AI Studio는 데이터베이스를 직접 연동하지 않으므로 PRD와 IA만으로 충분합니다.

Gemini Gem으로 기획 문서 생성

기획 도우미 Gem (peace 제작/공유)을 활용하면 기획 문서를 빠르게 작성할 수 있습니다.

사용법:

  1. Gem 링크에 접속
  2. 만들고 싶은 프로젝트를 설명
  3. Gem이 PRD와 IA를 자동 생성
  4. 생성된 기획 문서를 AI Studio 프롬프트에 활용

PRD 작성 프롬프트 예시

AI(ChatGPT, 제미나이 등)에게 PRD 작성을 요청하세요:

교사용 학급 관리 대시보드의 PRD를 작성해줘.

포함할 내용:
1. 프로젝트 개요 (목적, 대상 사용자)
2. 핵심 기능 목록 (우선순위별)
3. 섹션별 기능 상세 (대시보드, 학생 목록, 일정, 공지)
4. 디자인 요구사항 (색상, 레이아웃, 반응형)

대상: 초등학교 교사
주요 기능: 학생 관리, 출석 현황, 오늘의 일정, 공지사항

IA 작성 프롬프트 예시

위 PRD를 기반으로 IA(정보 구조)를 작성해줘.

포함할 내용:
1. 전체 섹션 구성 (탭 또는 스크롤 기반)
2. 각 섹션의 주요 컴포넌트
3. 네비게이션 흐름
4. 모바일에서의 레이아웃 변화

AI Studio 빌드에서 만들 예정이므로,
단일 HTML 파일에 탭 전환 방식으로 구성해줘.
팁: AI Studio 빌드는 단일 HTML 파일 기반이므로, IA를 작성할 때 "탭 전환" 또는 "스크롤 섹션" 방식으로 멀티 페이지를 구현하는 것이 좋습니다.

5. AI Studio 심화 활용

심화 프롬프트 패턴

중급자는 단순한 요청을 넘어 체계적인 프롬프트 패턴을 활용합니다.

패턴 1: 역할 부여

너는 10년 경력의 프론트엔드 개발자야.
교사용 학급 관리 대시보드를 만들어줘.
모던한 UI/UX를 적용하고, 접근성도 고려해줘.

패턴 2: 단계별 지시

아래 단계별로 진행해줘:

1단계: 전체 레이아웃을 먼저 잡아줘
  - 상단 헤더 (로고, 네비게이션)
  - 좌측 사이드바 (메뉴)
  - 중앙 콘텐츠 영역

2단계: 콘텐츠 영역에 4개 카드 위젯 배치
  - 총 학생 수
  - 오늘 출석률
  - 미제출 과제
  - 다가오는 일정

3단계: 카드 아래에 상세 테이블 추가

패턴 3: 부정 프롬프트 (하지 말아야 할 것)

대시보드를 만들어줘.

[하지 말 것]
- 외부 라이브러리(CDN) 사용하지 마
- 인라인 스타일 사용하지 마
- 스크롤이 끊기는 레이아웃 만들지 마

[해야 할 것]
- 순수 HTML/CSS/JS만 사용
- CSS 변수로 테마 관리
- 부드러운 스크롤과 애니메이션

멀티 컴포넌트 빌드 (탭 구조)

AI Studio에서 멀티 페이지 효과를 내려면 탭 기반 구조를 활용합니다:

탭 기반 학급 관리 앱을 만들어줘.

[탭 구성]
- 탭 1: 대시보드 (요약 카드 4개 + 차트)
- 탭 2: 학생 목록 (테이블 + 검색 + 필터)
- 탭 3: 일정 관리 (캘린더 뷰)
- 탭 4: 공지사항 (목록 + 작성 폼)

[공통 요소]
- 상단 네비게이션 바 (탭 전환)
- 하단 푸터

탭 전환 시 페이지 리로드 없이 JavaScript로 전환되도록 해줘.

CSS 커스터마이징 프롬프트

디자인을 세밀하게 조절하고 싶을 때:

현재 디자인에서 아래만 수정해줘:

1. 전체 색상 테마를 아래 팔레트로 변경:
   - 메인 색상: #4CAF50
   - 보조 색상: #A5D6A7
   - 배경: #F5F5F5
   - 텍스트: #333333

2. 카드에 그림자 효과 추가 (box-shadow)
3. 호버 시 카드가 살짝 올라오는 애니메이션
4. 폰트를 Pretendard로 변경 (CDN 사용 가능)

코드 Export 및 수정 기초

AI Studio에서 생성된 코드는 Export하여 로컬에서 수정할 수 있습니다:

  1. 생성된 코드를 전체 선택하여 복사
  2. 텍스트 에디터에 붙여넣고 index.html로 저장
  3. 브라우저에서 열어 동작 확인
  4. 필요한 부분을 직접 수정 (코드 이해가 필요)
팁: 코드를 직접 수정하는 것이 어렵다면, AI Studio에서 프롬프트로 수정하는 것이 더 효율적입니다. 코드 수정은 "배우면 좋지만 필수는 아닌" 영역입니다.

6. 실전 프로젝트: 멀티 섹션 대시보드

교사용 학급 관리 대시보드를 단계별로 만들어봅시다.

Step 1: 기획 및 프롬프트 작성 (PRD 기반)

먼저 기획 문서를 정리하고, 이를 기반으로 첫 번째 프롬프트를 작성합니다:

교사용 학급 관리 대시보드를 만들어줘.

## 프로젝트 개요
- 대상: 초등학교 담임교사
- 목적: 학급 현황을 한눈에 파악

## 구조 (탭 기반)
- 탭 1 — 대시보드: 요약 카드 4개 (학생 수, 출석률, 미제출 과제, 다음 일정)
- 탭 2 — 학생 목록: 학생 이름, 출석 상태, 메모 (테이블)
- 탭 3 — 일정: 이번 주 일정 목록
- 탭 4 — 공지: 공지사항 목록

## 디자인 요구사항
- 색상: 초록 계열 (#4CAF50 메인)
- 카드 UI, 둥근 모서리, 그림자
- 반응형 (모바일 대응)
- 탭 전환은 JavaScript로 처리 (새로고침 없이)

## 데이터
- localStorage로 데이터 저장 (서버 없이)
- 샘플 데이터 5명분 미리 포함

Step 2: 빌드 및 반복 수정 (3~5회 반복)

첫 번째 결과를 확인한 후, 아래와 같이 반복 수정합니다:

수정 1회차 — 구조 보완:

대시보드 탭의 요약 카드에 아이콘을 추가해줘.
카드를 클릭하면 해당 탭으로 이동하게 해줘.

수정 2회차 — 기능 추가:

학생 목록 탭에 검색 기능을 추가해줘.
이름으로 검색하면 테이블이 필터링되도록.
출석 상태를 클릭하면 토글(출석/결석/지각)되도록 해줘.

수정 3회차 — 디자인 개선:

전체적으로 디자인을 다듬어줘:
- 탭 전환 시 부드러운 페이드 애니메이션
- 카드 호버 시 살짝 올라오는 효과
- 모바일에서 탭이 가로 스크롤되도록
- 현재 선택된 탭에 하이라이트 표시

수정 4회차 — 데이터 연동:

학생 데이터와 일정 데이터를 localStorage에 저장하고 불러오게 해줘.
새 학생 추가, 학생 삭제 기능도 넣어줘.
일정 추가 폼도 만들어줘.

Step 3: 완성 및 코드 검토

최종 결과물이 만족스러우면 코드를 검토합니다:

현재 코드를 검토하고, 아래 사항을 확인해줘:
1. HTML 구조가 시맨틱하게 작성되었는지
2. CSS에 중복이 없는지
3. JavaScript에 오류가 없는지
4. 모바일에서 깨지는 부분이 없는지
5. localStorage 저장/불러오기가 정상 동작하는지
실사용자 후기: "처음에는 한 번에 완벽하게 만들려고 했는데, 3~4번 나눠서 요청하니까 훨씬 결과가 좋았어요. AI도 단계별로 하는 게 더 정확합니다." — 실사용자 후기

7. 배포

방법 1: AI Studio 내 공유 링크 (간단)

가장 간단한 방법입니다. AI Studio에서 바로 공유할 수 있습니다.

  1. 빌드 완료 후 Deploy 또는 공유 버튼 클릭
  2. 생성된 URL을 복사
  3. 카카오톡, 이메일 등으로 공유

장점: 즉시 공유 가능, 별도 설정 불필요
단점: AI Studio 서비스에 의존, 커스텀 도메인 불가

방법 2: 코드 Export → GitHub Pages (본격)

코드를 내 저장소에 올리고 GitHub Pages로 배포합니다.

Step 1: 코드 Export

  1. AI Studio에서 생성된 전체 코드를 복사
  2. 텍스트 에디터에 index.html로 저장
  3. 로컬 브라우저에서 열어 정상 동작 확인

Step 2: GitHub 저장소 생성 및 업로드

  1. github.com에서 New Repository 생성
  2. Upload files → index.html 파일 드래그 앤 드롭 → Commit changes

Step 3: GitHub Pages 활성화

  1. 저장소 Settings → Pages
  2. Branch를 main 선택 → Save
  3. 1~2분 후 https://사용자이름.github.io/저장소이름 으로 접속 가능

방법 3: Vercel 배포 (선택)

더 빠른 배포와 커스텀 도메인을 원한다면:

  1. vercel.com에서 GitHub 계정으로 로그인
  2. Import Project → GitHub 저장소 선택
  3. 자동 빌드 및 배포 (1분 이내)
  4. https://프로젝트명.vercel.app으로 접속 가능

10. 문제가 생겼을 때 (트러블슈팅)

상황 1: 빌드가 실패할 때

AI Studio에서 프롬프트를 보냈는데 결과가 생성되지 않거나 빈 화면이 나올 때:

  1. 프롬프트가 너무 길거나 복잡하지 않은지 확인
  2. 프롬프트를 짧고 구체적으로 다시 작성
  3. 브라우저를 새로고침(F5)한 후 다시 시도
  4. 새 대화를 시작하여 처음부터 다시 빌드
상황 2: 프리뷰가 깨질 때

레이아웃이 엉키거나, 요소가 겹치거나, 스타일이 적용되지 않을 때:

  1. "레이아웃이 깨져있어. 전체 CSS를 점검하고 수정해줘"라고 요청
  2. 구체적으로 깨진 부분을 설명: "학생 목록 테이블이 화면 밖으로 벗어나"
  3. "모바일 뷰에서 확인하고 반응형으로 수정해줘"라고 요청
  4. 심한 경우 새 대화에서 다시 빌드하는 것이 빠를 수 있음
상황 3: 코드 Export가 안 될 때

코드를 복사했는데 로컬에서 동작하지 않을 때:

  1. 외부 CDN 리소스(폰트, 아이콘 등)가 로드되는지 확인
  2. 인터넷 연결 상태 확인 (CDN은 온라인 필요)
  3. 코드에 <html>, <head>, <body> 태그가 완전한지 확인
  4. AI Studio에 "이 코드를 완전한 HTML 문서로 만들어줘"라고 요청
상황 4: API 할당량 초과

"Rate limit exceeded" 또는 사용량 초과 메시지가 나올 때:

  1. 잠시 기다리기 (무료 할당량은 시간/일 단위로 리셋)
  2. 프롬프트를 더 효율적으로 작성 (한 번에 여러 수정사항 전달)
  3. 기다리는 동안 기획 문서를 보완하거나 아이디어를 정리
  4. 다음 날 다시 시도
상황 5: 응답이 너무 느릴 때

AI의 응답이 30초 이상 걸리거나 멈춰 있을 때:

  1. 요청이 너무 복잡하지 않은지 확인 — 단순화 시도
  2. 브라우저 탭을 닫지 말고 기다려보기 (최대 2~3분)
  3. 그래도 안 되면 페이지 새로고침 후 다시 시도
  4. 사용자가 많은 시간대(오전 9~11시)를 피해서 시도

11. 자주 묻는 질문 (FAQ)

Q1: 초보자편과 뭐가 다른가요?

초보자편은 AI Studio의 기본 사용법과 간단한 한 페이지 프로토타입 제작에 초점을 맞춥니다. 중급자편에서는 기획 문서를 기반으로 체계적인 프로젝트를 만들고, 멀티 섹션 앱 구축, 코드 Export 및 배포, 외부 서비스 연동까지 다룹니다.

Q2: AI Studio에서 멀티 페이지를 만들 수 있나요?

URL 기반의 실제 멀티 페이지는 어렵지만, 탭 전환 또는 섹션 스크롤 방식으로 멀티 페이지 효과를 낼 수 있습니다. JavaScript로 뷰를 전환하는 SPA(Single Page Application) 방식입니다.

Q3: AI Studio에서 만든 것을 러버블로 옮길 수 있나요?

직접적인 Import 기능은 없지만, AI Studio의 코드를 러버블 프롬프트에 붙여넣으며 "이 코드를 기반으로 만들어줘"라고 요청하면 됩니다. 또는 디자인 스크린샷을 참고하며 러버블에서 새로 만드는 방법도 있습니다.

Q4: 기획 문서(PRD/IA)를 꼭 써야 하나요?

필수는 아니지만, 강력히 추천합니다. 기획 문서 없이 바로 빌드하면 수정 횟수가 많아지고, 결과물의 일관성이 떨어집니다. 5~10분 투자하여 기획 문서를 정리하면 전체 작업 시간이 오히려 줄어듭니다.

Q5: Gemini API 키가 꼭 필요한가요?

AI Studio Build 탭에서 프로토타입을 만드는 데는 API 키가 필요 없습니다. 코드를 Export한 후 로컬에서 Gemini API를 직접 호출하고 싶을 때만 필요합니다.

Q6: 코드를 전혀 몰라도 중급자편을 따라할 수 있나요?

네, 코드를 직접 수정하지 않아도 프롬프트만으로 진행할 수 있습니다. 다만 코드 Export 후 수정, Gemini API 연동 등 일부 심화 내용은 기본적인 HTML/CSS 이해가 있으면 더 수월합니다.

Q7: 여러 사람이 동시에 편집할 수 있나요?

AI Studio 빌드의 동시 편집(협업) 기능은 제한적입니다. 한 명이 빌드한 후 공유 링크를 통해 결과물을 공유하는 방식이 현실적입니다. 협업이 중요하다면 GitHub 기반 도구(커서, 안티그래비티)를 추천합니다.

Q8: localStorage에 저장한 데이터가 사라질 수 있나요?

네. localStorage는 브라우저 캐시를 삭제하거나, 시크릿 모드에서 접속하면 데이터가 사라집니다. 중요한 데이터는 Firebase 등 외부 DB에 저장하거나, Export 기능을 만들어 백업하세요.

세션 관리 (대화 흐름 유지)

AI Studio에서는 대화가 길어질수록 Context Window에 가까워져 이전 지시를 잊을 수 있습니다.

세션 관리 팁:

  • 하나의 대화에서 10~15회 이내로 수정 요청
  • 큰 변경이 필요하면 새 대화를 시작하고 기존 코드를 붙여넣기
  • 중간 결과물을 반드시 코드로 백업 (복사해두기)
  • 대화 시작 시 기획 문서(PRD)를 먼저 전달하면 일관성 유지

반복 패턴 (점진적 개선)

효율적인 빌드를 위한 반복 패턴:

Round 1: 전체 구조와 레이아웃 (뼈대 잡기)
  → 확인: 섹션이 모두 있는지, 네비게이션이 동작하는지

Round 2: 콘텐츠 채우기 (내용물 넣기)
  → 확인: 텍스트, 데이터, 이미지가 적절한지

Round 3: 디자인 다듬기 (인테리어)
  → 확인: 색상, 폰트, 간격, 그림자가 균일한지

Round 4: 인터랙션 추가 (기능 동작)
