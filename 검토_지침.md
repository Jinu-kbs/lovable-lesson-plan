# 바이브코딩 가이드 교안 검토 지침

## 개요
본 문서는 김선생의 바이브코딩 가이드 교안의 품질을 유지하기 위한 검토 체크리스트입니다.
모든 교안(HTML/MD)은 아래 24개 항목을 기준으로 검토합니다.

## 검토 기준

### A. 자기주도학습 지원 (6항목)
- [ ] **A-1**: 자가 진단 테이블에 각 추천 교안으로의 하이퍼링크가 존재하는가?
- [ ] **A-2**: "다음 단계" 섹션의 모든 교안 참조에 클릭 가능한 하이퍼링크(`<a href>`)가 있는가?
- [ ] **A-3**: 각 주요 섹션의 학습 목표가 첫머리에 명시되어 있는가?
- [ ] **A-4**: 전문 용어가 처음 등장할 때 쉬운 설명 또는 용어 해설이 함께 제공되는가?
- [ ] **A-5**: 실습 단계가 번호 매긴 step-by-step으로 구성되어 있는가?
- [ ] **A-6**: "선수 학습" 안내가 존재하는가? (lessons-data.js의 prereqs 기반 자동 생성 또는 수동)

### B. 하이퍼링크 완전성 (6항목)
- [ ] **B-1**: "다음 단계" 테이블의 모든 교안 참조가 `<a href="파일명.html">` 형태의 링크인가?
- [ ] **B-2**: 본문 내 다른 도구/교안을 언급할 때 최소 1회는 하이퍼링크가 걸려있는가?
- [ ] **B-3**: "추천 리소스" 테이블의 외부 URL이 모두 유효한가? (HTTP 200 응답)
- [ ] **B-4**: 외부 링크에 `target="_blank"` 속성이 있는가?
- [ ] **B-5**: 내부 링크(다른 교안) 파일명이 실제 존재하는 HTML 파일과 정확히 일치하는가?
- [ ] **B-6**: 앵커 링크(`#id`)가 실제 존재하는 섹션 ID(`<h2 id="...">`)와 일치하는가?

### C. 네비게이션 구조 (4항목)
- [ ] **C-1**: 브레드크럼이 자동 표시되는가? (common.js + lessons-data.js 기반)
- [ ] **C-2**: sidebar TOC의 링크 수가 실제 `<h2>` 태그 수와 일치하는가?
- [ ] **C-3**: 이전/다음 교안 footer 네비게이션이 자동 표시되는가? (common.js 기반)
- [ ] **C-4**: 관련 교안 카드 섹션이 자동 표시되는가? (common.js + lessons-data.js 기반)

### D. 콘텐츠 품질 (4항목)
- [ ] **D-1**: 개인 발언 인용문이 익명 처리되어 있는가? (예: "— 실사용자 후기")
- [ ] **D-2**: 페이지 색상 테마(CSS 변수)가 CLAUDE.md 색상 규칙과 일치하는가?
- [ ] **D-3**: 코드 블록에 복사 버튼이 정상 작동하는가?
- [ ] **D-4**: 이미지 및 외부 미디어가 정상 로드되는가?

### E. MD↔HTML 동기화 (4항목)
- [ ] **E-1**: MD 파일의 섹션 구조(##)가 HTML의 `<h2>` 구조와 일치하는가?
- [ ] **E-2**: MD의 "다음 단계" 내용이 HTML의 "다음 단계" 내용과 동일한가?
- [ ] **E-3**: MD 파일에도 하이퍼링크가 마크다운 문법(`[텍스트](url)`)으로 포함되어 있는가?
- [ ] **E-4**: HTML에 신규 추가된 섹션이 MD에도 반영되어 있는가?

## 검토 절차
1. 위 체크리스트를 각 교안에 대해 순회하며 점검
2. 불합격 항목은 즉시 수정 후 재검토
3. 모든 항목 통과 시 ✅ 표시
4. MD↔HTML 동기화는 변경 시마다 확인
