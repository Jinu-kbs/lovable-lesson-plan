# 러버블(Lovable) 입문 교안 - 중급자편

> **대상**: AI 도구(ChatGPT, 제미나이 등)는 써봤지만 웹앱 빌더는 처음인 분
> **목표**: 체계적인 기획 문서로 기능성 웹앱을 제작하고 배포하기
> **소요 시간**: 약 4~5시간

---

## 이 교안이 맞는 분

- ChatGPT, 제미나이, 클로드 등 AI 챗봇을 사용해본 경험이 있다
- "프롬프트"가 뭔지 알고, AI에게 원하는 결과를 요청하는 데 익숙하다
- 하지만 웹사이트를 직접 만들어본 적은 없거나, 러버블 같은 빌더는 처음이다
- 코딩 경험이 없거나 매우 기초적이다

> 코딩 경험이 있다면 → **개발자편** 교안을 추천합니다.
> AI 도구 자체가 처음이라면 → **초보자편** 교안을 먼저 보세요.

## 1. 바이브코딩 도구 비교

AI 도구 경험이 있는 분이라면, 바이브코딩 도구의 차이를 이해하면 상황에 맞는 도구를 선택할 수 있습니다.

### 주요 도구 비교표

| 항목 | **Lovable** | **안티그래비티** | **커서(Cursor)** | **AI Studio 빌더** |
|------|-------------|-----------------|-----------------|-------------------|
| **설치** | 불필요 (웹) | 불필요 (웹) | 설치 필요 (IDE) | 불필요 (웹) |
| **난이도** | 매우 쉬움 | 쉬움 | 중간 | 매우 쉬움 |
| **코드 접근** | 가능 | 가능 | 완전 제어 | 제한적 |
| **가격** | 유료 (이벤트 시 무료) | 무료+유료 | 유료 | 무료 |
| **DB 지원** | 외부 연결 필요 | 지원 | 자유 | 제한적 |
| **배포** | 자체 배포 + GitHub | 내장 | 수동 | 내장 |
| **주요 기술** | React (Anthropic Claude) | Flutter | 다양 | Google AI |
| **추천 대상** | 빠른 프로토타입 | 범용 제작 | 개발 경험자 | AI 입문자 |

### 어떤 도구를 선택할까?

- **러버블**: 빠르게 프로토타입을 만들고 싶을 때, 설치 없이 시작하고 싶을 때
- **안티그래비티**: 무료로 시작하고 싶을 때, Flutter 기반 앱을 만들고 싶을 때
- **커서**: 코드를 직접 제어하고 싶을 때, 복잡한 프로젝트를 진행할 때
- **AI Studio 빌더**: 바이브코딩 첫 경험으로, IDE 개념을 배우기 전 단계로

> **실사용자 경험**: "처음하시는 선생님들이 하기엔 러버블이 좋아요. 따로 설치할 게 없으니까요"
>
> "AI 스튜디오 빌더만 해봐도 '아~ 이런 거구나' 느낌을 받으시더라고요"
>
> "휘둘리지 않으셔도 돼요. 하나에 익숙해지는 게 중요합니다"

### 학습 경로 추천

```
[입문] AI Studio 빌더 (무료, 개념 이해)
  ↓
[성장] 러버블 (프로토타입 빠른 제작) ← 오늘의 교안
  ↓
[확장] 안티그래비티 or 커서 (본격 제작)
```

## 2. 효율적인 워크플로우

러버블에 바로 프롬프트를 던지면 크레딧을 낭비하기 쉽습니다. 아래 워크플로우를 따라주세요.

### 5단계 워크플로우

```
① 아이디어 구상
      ↓
② AI로 기획 문서 작성 (PRD / IA / ERD)
      ↓
③ 러버블에서 제작 (Chat 모드 활성화)
      ↓
④ 배포 (러버블 Publish 또는 GitHub + Vercel)
      ↓
⑤ SNS 공유 (이벤트 미션)
```

### 왜 이 순서인가?

| 단계 | 하는 일 | 도구 | 비용 |
|------|---------|------|------|
| ① 아이디어 | 뭘 만들지 구상 | 메모장 | 무료 |
| ② 기획 문서 | PRD, IA, ERD 작성 | ChatGPT, 제미나이, 클로드 | 무료~저렴 |
| ③ 제작 | 기획 기반 빌드 | Lovable | 크레딧 소모 |
| ④ 배포 | 웹에 공개 | Lovable Publish 또는 Vercel | 무료 |
| ⑤ 공유 | SNS 공유 | 인스타, X 등 | 무료 |

> **핵심 포인트**: ②번에서 충분히 정리해야 ③번에서 크레딧을 아낄 수 있습니다. 다른 AI에서 미리 정리해두면 러버블에서 불필요하게 왔다 갔다 안 해도 됩니다.

## 4. 기획 문서 작성 실습

러버블에 들어가기 전, AI를 활용해 3가지 핵심 문서를 작성합니다.

### 4-1. PRD (Product Requirements Document, 요구정의서)

가장 중요한 문서입니다. 무엇을 만들 것인지 명확하게 정의합니다.

**AI에게 PRD 작성 요청하는 프롬프트:**

```
나는 [직업/역할]이고, [목적]을 위한 웹앱을 만들고 싶습니다.

다음 내용을 포함한 PRD(요구정의서)를 작성해주세요:

1. 프로젝트 개요 (이름, 목적, 핵심 가치)
2. 대상 사용자 및 사용자 시나리오
3. 핵심 기능 목록 (우선순위별)
4. 페이지 구성 및 화면 흐름
5. 비기능 요구사항 (반응형, 성능, 접근성)
6. 디자인 가이드라인 (색상, 톤앤매너)
7. 제약 조건 및 가정

기능 설명은 구체적으로, 각 기능별로 사용자 스토리 형식으로 작성해주세요.
```

### 4-2. IA (Information Architecture, 정보 구조)

페이지 간의 구조와 관계를 정리합니다. 사이트 전체의 "지도"라고 생각하면 됩니다.

**AI에게 IA 작성 요청하는 프롬프트:**

```
위 PRD를 기반으로 IA(정보 구조도)를 작성해주세요.

포함할 내용:
1. 전체 사이트맵 (트리 구조)
2. 각 페이지의 주요 컴포넌트
3. 페이지 간 이동 흐름
4. 네비게이션 구조

마크다운 형식으로 작성해주세요.
```

**IA 예시 (프리랜서 포트폴리오):**

```
홈 (/)
├── 소개 (/about)
├── 포트폴리오 (/portfolio)
│   └── 프로젝트 상세 (/portfolio/:id)
├── 서비스 (/services)
└── 연락처 (/contact)
```

**IA 예시 (학원/강좌 안내):**

```
홈 (/)
├── 강좌 안내 (/courses)
│   └── 강좌 상세 (/courses/:id)
├── 강사 소개 (/instructors)
├── 수강 후기 (/reviews)
├── 공지사항 (/notices)
└── 문의하기 (/contact)
```

### 4-3. ERD (Entity Relationship Diagram, 데이터 구조)

데이터베이스가 필요한 프로젝트라면 ERD도 준비합니다. 소개 사이트처럼 DB가 필요 없는 경우에는 건너뛰어도 됩니다.

**AI에게 ERD 작성 요청하는 프롬프트:**

```
위 PRD를 기반으로 ERD(데이터 구조)를 작성해주세요.

포함할 내용:
1. 필요한 데이터 테이블 목록
2. 각 테이블의 항목 (이름, 종류, 설명)
3. 테이블 간 관계

마크다운 테이블 형식으로 쉽게 작성해주세요.
```

### PRD 작성 시 직군별 포인트

| 직군 | PRD에 꼭 포함할 내용 |
|------|----------------------|
| **교사** | 학부모 접근성, 모바일 최적화, 개인정보 주의 |
| **강사/튜터** | 수강 신청 흐름, 일정 관리, 결제 연동 여부 |
| **프리랜서** | 작업물 쇼케이스, 문의 양식, SEO 키워드 |
| **소상공인** | 메뉴/상품 정보, 위치·영업시간, SNS 연결 |
| **동호회** | 회원 관리, 일정 공유, 사진 갤러리 |

## 5. 러버블 심화 활용

### 5-1. Chat 모드 활용 전략

러버블의 Chat 버튼을 활성화하면 **플랜 모드**처럼 동작합니다.

| 모드 | 동작 방식 | 크레딧 | 결과 품질 |
|------|-----------|--------|-----------|
| **일반 모드** | 바로 코드 생성 | 적음 | 보통 |
| **Chat 모드** | 먼저 분석 → 보완 → 코드 생성 | 약 1크레딧 추가 | 높음 |

**Chat 모드 활용 팁:**

1. **첫 프롬프트는 길고 구체적으로**: PRD 전체를 붙여넣기
2. **수정은 짧고 명확하게**: "헤더 색상을 파란색 계열로 변경해주세요"
3. **한 번에 하나씩**: 여러 수정을 한꺼번에 요청하지 말고 나눠서
4. **참고 이미지 활용**: 원하는 디자인 참고 사이트가 있으면 설명에 포함

### 5-2. 프롬프트 작성 노하우 (AI 경험자용)

AI 도구 경험이 있는 분이라면 아래 테크닉을 활용해보세요:

**역할 부여:**
```
너는 한국 시장을 이해하는 시니어 웹 개발자야.
아래 PRD를 기반으로 한국 사용자에게 최적화된 웹사이트를 만들어줘.
```

**단계별 지시:**
```
Step 1: 먼저 전체 페이지 레이아웃을 잡아줘.
Step 2: 각 페이지에 더미 데이터를 넣어서 실제 느낌이 나게 해줘.
Step 3: 모바일에서도 잘 보이게 반응형으로 조정해줘.
```

**부정 프롬프트 (하지 말아야 할 것):**
```
- 영어로 된 텍스트는 사용하지 마세요 (Lorem ipsum 등)
- 너무 화려한 애니메이션은 넣지 마세요
- 팝업이나 모달은 사용하지 마세요
```

### 5-3. 프로젝트 파일 관리

- **다운로드**: 마음에 드는 버전은 바로 소스 코드 다운로드
- **이어서 작업**: 다운로드한 파일은 다른 도구(커서, VS Code 등)에서 이어서 작업 가능
- **버전 관리**: 주요 변경 전후로 다운로드해두면 롤백이 가능

## 6. 실전 프로젝트: 기능성 웹앱 만들기

### 직군별 추천 프로젝트

| 직군 | 프로젝트 | 난이도 | 핵심 기능 |
|------|----------|--------|-----------|
| **교사** | 수업 자료 공유 플랫폼 | ★★☆ | 카드 목록, 필터링, 검색 |
| **강사** | 온라인 강좌 안내 사이트 | ★★☆ | 강좌 목록, 일정, 수강 신청 양식 |
| **프리랜서** | 서비스 소개 + 견적 요청 사이트 | ★★☆ | 포트폴리오, 문의 양식, 가격표 |
| **소상공인** | 주문/예약 안내 사이트 | ★★★ | 메뉴, 예약 양식, 위치 안내 |
| **동호회** | 커뮤니티 사이트 | ★★★ | 게시판, 일정, 갤러리 |

### 실습 예시: 온라인 강좌 안내 사이트

#### Step 1: PRD 준비

```markdown
# PRD: 코딩 클래스 안내 사이트

## 프로젝트 개요
바이브코딩 강좌를 안내하고 수강 신청을 받는 사이트

## 핵심 기능
1. 강좌 목록 보기 (카드 그리드 레이아웃)
2. 강좌별 상세 정보 (커리큘럼, 일정, 강사 소개)
3. 수강 후기 섹션
4. 수강 신청 문의 양식 (이름, 연락처, 관심 강좌)
5. 공지사항

## 페이지 구성
- 홈: 대표 배너 + 인기 강좌 하이라이트
- 강좌 목록: 전체 강좌 + 카테고리 필터
- 강좌 상세: 개별 강좌 상세 정보
- 수강 후기: 수강생 리뷰
- 문의하기: 수강 신청 양식
- 소개: 강사/기관 소개

## 디자인
- 전문적이면서 친근한 느낌 (파란/보라 톤)
- 카드 기반 레이아웃
- 반응형 (모바일 최적화)
- 한국어 인터페이스
```

#### Step 2: 러버블에서 빌드

1. 새 프로젝트 생성
2. **Chat 모드 활성화**
3. PRD 전체를 프롬프트에 입력
4. 결과 확인 후 단계별 수정

#### Step 3: 반복 수정 예시

```
1차: "강좌 카드에 '신규' 뱃지와 '인기' 뱃지를 추가해주세요.
      신규는 초록색, 인기는 빨간색으로요."

2차: "문의 양식에 '관심 강좌' 드롭다운을 추가하고,
      강좌 목록에서 선택할 수 있게 해주세요."

3차: "모바일에서 강좌 카드가 한 줄에 하나씩 보이게 조정해주세요.
      카드 사이 간격도 좀 넓혀주세요."
```

## 7. 웹사이트 공개하기 (배포)

### 방법 1: 러버블 자체 배포 (Publish)

가장 빠른 방법입니다:

1. **Publish** 버튼 클릭
2. 자동 생성된 URL로 바로 접속 가능
3. 수정하면 다시 Publish하여 업데이트

### 방법 2: GitHub + Vercel 배포 (추천)

더 안정적이고 나만의 URL을 가질 수 있습니다.

**GitHub 연동:**

1. 프로젝트 설정에서 **GitHub 연동** 선택
2. 저장소(Repository) 생성 또는 기존 저장소 선택
3. **Push** 클릭하면 코드가 자동으로 GitHub에 업로드

**Vercel 배포:**

1. Vercel 대시보드 → **Add New Project**
2. GitHub 저장소 Import
3. Framework 자동 감지 → **Deploy** 클릭
4. 약 1~2분 후 배포 완료

**자동 배포의 장점:**

Vercel은 GitHub과 연동되면 **코드 변경 시 자동 재배포**됩니다:

```
러버블에서 수정 → GitHub Push → Vercel 자동 재배포 → 사이트 업데이트
```

## 8. 데이터베이스 연결

러버블 자체에는 데이터베이스가 내장되어 있지 않습니다. 동적 데이터가 필요한 경우 외부 DB를 연결해야 합니다.

### DB가 필요한 경우 vs 불필요한 경우

| DB 불필요 (정적 사이트) | DB 필요 (동적 사이트) |
|--------------------------|------------------------|
| 소개 사이트 | 게시판/블로그 |
| 포트폴리오 | 회원 관리 |
| 메뉴판/가격표 | 예약/주문 시스템 |
| 이벤트 안내 | 데이터 검색/필터 |

> **추천**: 처음에는 DB 없이 정적 사이트로 시작하고, 익숙해진 후 DB를 연결하세요.

### 추천 외부 DB 서비스

| 서비스 | 특징 | 무료 티어 | 난이도 |
|--------|------|-----------|--------|
| **Supabase** | PostgreSQL 기반, 실시간 기능 | 충분한 무료 | 중 |
| **Firebase** | Google 제공, 실시간 DB | 넉넉한 무료 | 중 |
| **Airtable** | 스프레드시트 느낌의 DB | 기본 무료 | 하 |

### DB 연결 시 러버블 프롬프트 예시

```
이 프로젝트에 Supabase를 데이터베이스로 연결해주세요.

테이블 구성:
- courses (강좌): id, title, category, description, schedule, price, created_at
- inquiries (문의): id, name, phone, email, course_id, message, created_at

Supabase URL과 API 키는 환경변수로 설정해주세요.
```

## 10. 문제가 생겼을 때 (트러블슈팅)

### "Chat 모드에서 크레딧이 너무 빨리 줄어요"

- 첫 프롬프트에 PRD를 충분히 상세하게 작성하세요
- 수정 요청은 한 번에 하나씩만
- 이벤트 기간에는 무제한이니 마음껏 실험하세요

### "러버블이 의도와 다른 결과를 만들었어요"

1. **되돌리기(Undo)** 기능으로 이전 상태로 복원
2. 더 구체적으로 재요청 (색상, 위치, 크기 등 명시)
3. "현재 상태를 유지하면서 [X] 부분만 변경해줘"로 범위 제한

### "코드가 깨져서 아무것도 안 보여요"

1. 당황하지 말고 **되돌리기(Undo)** 클릭
2. 여러 번 되돌려도 안 되면 마지막으로 다운로드한 파일에서 재시작
3. 새 프로젝트를 만들어서 PRD를 다시 투입하는 것도 방법

### "DB 연결 후 에러가 발생해요"

- 처음부터 DB를 연결하지 말고, UI(화면)를 먼저 완성하세요
- UI가 마음에 든 후에 DB 연결을 시도하세요
- Supabase 환경변수(URL, API키)가 정확한지 확인하세요

## 11. 자주 묻는 질문 (FAQ)

### Q: 러버블에 자체 DB가 있나요?

아니요. 러버블 자체에는 데이터베이스가 없습니다. 정적 소개 사이트는 DB 없이 가능하지만, 데이터를 저장·관리하려면 Supabase 같은 외부 서비스를 연결해야 합니다.

### Q: 러버블에서 만든 사이트를 다른 도구에서 이어서 작업할 수 있나요?

네! 프로젝트 파일을 다운로드하면 VS Code, Cursor, 클로드 코드 등에서 이어서 수정할 수 있습니다. GitHub에 올려놓으면 어디서든 접근 가능합니다.

### Q: 러버블과 안티그래비티, 뭐가 더 좋나요?

정답은 없습니다. 러버블은 웹(React) 기반으로 빠른 프로토타입에 강하고, 안티그래비티는 Flutter 기반으로 앱 제작에 강합니다. 오늘 이벤트로 러버블을 체험해보고, 취향에 맞는 도구를 선택하세요.

### Q: 한 번 만든 프로젝트를 팀원과 공유할 수 있나요?

GitHub에 올려두면 팀원들과 함께 수정할 수 있습니다. Vercel에 배포하면 URL로 결과물도 바로 공유 가능합니다.

### Q: 이벤트가 끝나면 만든 사이트는 어떻게 되나요?

- 러버블 자체 배포(Publish): 러버블 계정이 유지되는 한 접속 가능
- GitHub + Vercel 배포: 이벤트와 무관하게 계속 유지 (무료)
- **파일 다운로드**해두면 어디서든 다시 사용 가능
