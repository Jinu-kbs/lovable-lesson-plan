MCP가 뭔가요?

MCP는 Model Context Protocol(모델 컨텍스트 프로토콜)의 줄임말입니다. Anthropic이 2024년 11월에 발표한 오픈 표준으로, AI 모델이 외부 도구와 서비스에 연결될 수 있게 해주는 규격입니다.

쉬운 비유로 이해하기

MCP는 "Claude에게 새로운 손을 붙여주는 표준 규격"이라고 생각하면 됩니다.

기본 Claude

  • 텍스트만 읽고 쓰기 가능
  • 현재 프로젝트 파일만 접근
  • 인터넷 검색 불가
  • 외부 서비스 연동 불가

MCP 연결 후

  • 로컬 파일 자유롭게 읽기/쓰기
  • GitHub PR·이슈 직접 조회
  • 실시간 웹 검색 가능
  • Notion·Slack 등 연동

왜 MCP가 필요한가?

Claude Code는 기본적으로 현재 열린 프로젝트 파일만 볼 수 있습니다. 하지만 실제 개발 작업에서는 훨씬 다양한 도구가 필요합니다.

MCP가 없을 때 vs. 있을 때

MCP 없이

  • GitHub 이슈 보려면 직접 브라우저 열기
  • 파일 내용 복사해서 붙여넣기
  • Slack 메시지 직접 작성 후 전송
  • DB 쿼리 결과 수동으로 붙여넣기

MCP 연결 후

  • "우리 레포 오픈 이슈 보여줘" 한 마디
  • Claude가 직접 파일 시스템 탐색
  • Claude가 Slack 채널에 메시지 전송
  • Claude가 직접 DB 쿼리 실행·분석
"이제 Claude에게 '우리 GitHub 레포의 오픈 이슈 목록 보여줘'라고 말할 수 있습니다. 불필요한 복사-붙여넣기 없이 Claude가 직접 연결해서 가져옵니다."

MCP는 단순한 편의 기능이 아닙니다. AI가 실제 작업 흐름에 깊숙이 통합되어 진짜 에이전트처럼 동작할 수 있게 해주는 핵심 기술입니다.

Claude Code에서 설정하기

MCP 서버는 JSON 설정 파일에 등록합니다. 설정 파일 위치는 두 가지입니다:

설정 범위파일 위치용도
전역 설정~/.claude/settings.json모든 프로젝트에서 사용
프로젝트 설정.claude/settings.json특정 프로젝트에서만 사용

처음에는 전역 설정에 추가하는 것이 편리합니다. 이후 프로젝트별로 다른 설정이 필요할 때 프로젝트 설정을 활용하세요.

기본 설정 구조

// ~/.claude/settings.json { "mcpServers": { "filesystem": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/사용자/Desktop"] }, "github": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"], "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_여기에토큰입력" } } } }

설정 후 연결 확인

Claude Code를 재시작한 뒤 채팅창에 아래 명령어를 입력하면 연결된 MCP 서버 목록과 상태를 확인할 수 있습니다:

/mcp

연결이 성공하면 각 서버 옆에 ✓ connected 표시가 나타납니다.

실습 1: 파일시스템 MCP

가장 간단하게 시작할 수 있는 MCP입니다. 별도 API 키 없이 로컬 파일 시스템에 접근할 수 있게 해줍니다.

1

Node.js 버전 확인

MCP 서버를 실행하려면 Node.js v18 이상이 필요합니다. 터미널에서 확인하세요.

node -v
# v18.0.0 이상이면 OK
2

settings.json에 파일시스템 서버 추가

~/.claude/settings.json 파일을 열고 아래 내용을 추가합니다. /Users/사용자이름/Desktop 부분은 실제 경로로 바꿔주세요.

3

Claude Code 재시작

설정 파일 변경 후에는 Claude Code를 완전히 종료하고 다시 실행해야 설정이 반영됩니다.

4

/mcp 명령어로 연결 확인

채팅창에 /mcp를 입력하면 filesystem 서버가 목록에 표시되어야 합니다.

5

동작 테스트

Claude에게 "바탕화면의 파일 목록을 보여줘"라고 요청해 보세요.

파일시스템 설정 예시 (전체 JSON)

{ "mcpServers": { "filesystem": { "command": "npx", "args": [ "-y", "@modelcontextprotocol/server-filesystem", "/Users/사용자이름/Desktop", // 허용할 디렉토리 "/Users/사용자이름/Documents" // 여러 경로 추가 가능 ] } } }

실제 대화 예시

바탕화면에 있는 파일들 보여줘
바탕화면에서 다음 파일들을 찾았습니다:
📄 프로젝트계획서.docx
📁 사진들/
📄 TODO.txt
📄 회의록_2026.pdf
... 총 12개 파일/폴더
TODO.txt 내용 읽어줘
TODO.txt 내용입니다:
- [ ] 프로젝트 기획서 완성
- [x] 팀 미팅 준비
- [ ] 데모 영상 촬영

실습 2: GitHub MCP

개발자라면 가장 자주 쓰게 될 MCP입니다. GitHub 레포지토리의 이슈·PR·코드를 Claude가 직접 조회·수정할 수 있게 됩니다.

1

GitHub Personal Access Token 발급

GitHub 웹사이트에서 토큰을 발급합니다:
github.com → 프로필 → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token

2

토큰 권한(Scope) 설정

아래 권한을 체크합니다:

  • repo — 레포지토리 읽기·쓰기
  • read:org — 조직 정보 읽기
  • read:user — 사용자 정보 읽기
3

settings.json에 GitHub 서버 추가

발급받은 토큰을 env 항목에 입력합니다. 토큰 값은 절대 코드에 직접 넣지 마세요.

4

연결 테스트

Claude Code를 재시작하고 "내 GitHub 레포 목록 보여줘"라고 요청해 보세요.

GitHub 설정 예시

{ "mcpServers": { "github": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"], "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_여기에토큰입력" } } } }

실제 대화 예시

우리 레포에서 오픈된 이슈 5개 보여줘
'my-project' 레포의 오픈 이슈입니다:
#42 로그인 버튼이 모바일에서 안 눌림 (2일 전) — bug
#38 다크모드 색상 깨짐 (5일 전) — ui
#35 검색 결과 로딩 속도 개선 필요 (1주 전) — enhancement
#31 이미지 업로드 실패 오류 (2주 전) — bug
#28 알림 설정 페이지 미완성 (3주 전) — feature
#42 이슈를 분석해서 수정 방법을 제안해줘
#42 이슈를 확인했습니다. 모바일 터치 이벤트 처리 문제로 보입니다. 관련 코드를 분석하겠습니다...
토큰은 .env 파일이나 환경변수로 관리하세요. settings.json을 Git 레포에 올릴 경우 반드시 .gitignore에 추가해 토큰이 외부에 노출되지 않도록 주의하세요.

실습 3: Notion MCP

기획 문서나 회의록이 Notion에 있다면, Claude가 직접 내용을 읽고 정리하거나 새로운 페이지를 작성할 수 있습니다.

1

Notion Integration 생성

notion.so/my-integrations에 접속 → New integration 클릭 → 이름 입력(예: "Claude MCP") → Submit
생성 후 Internal Integration Secret 키를 복사해 둡니다.

2

연결할 페이지에 Integration 초대

Notion에서 연결할 페이지를 열고, 우측 상단 ··· → Connections → 방금 만든 Integration 추가
이 단계를 거친 페이지와 하위 페이지만 Claude가 접근할 수 있습니다.

3

settings.json에 Notion 서버 추가

복사한 Integration Secret을 환경변수로 입력합니다.

{ "mcpServers": { "notion": { "command": "npx", "args": ["-y", "mcp-notion-server"], "env": { "NOTION_API_KEY": "secret_여기에키입력" } } } }
4

연결 테스트

Claude Code를 재시작한 뒤 "내 노션 페이지 목록 보여줘"라고 요청해 보세요.

활용 예시

노션에서 '프로젝트 회의록' 페이지 찾아서 지난 주 결정 사항 요약해줘
프로젝트 회의록 페이지를 찾았습니다. 지난 주 주요 결정 사항을 요약합니다:
1. 출시일을 4월 15일로 확정
2. 모바일 앱 개발은 2차 계획으로 이동
3. 베타 테스터 20명 모집 시작

자주 겪는 문제

MCP 설정 과정에서 자주 발생하는 문제와 해결 방법입니다.

MCP 서버가 연결되지 않아요 (not connected)

원인: Node.js 버전 문제 또는 npx 캐시 문제가 가장 흔합니다.

해결 방법:

  1. 터미널에서 node -v로 버전 확인 → v18 미만이면 업그레이드
  2. npx 캐시 클리어: npx clear-npx-cache
  3. 수동으로 패키지 설치 테스트: npx -y @modelcontextprotocol/server-filesystem --help
  4. Claude Code 완전 종료 후 재시작
permission denied 에러가 발생해요

원인: settings.json에 상대 경로나 잘못된 경로를 입력했을 때 발생합니다.

해결 방법:

  1. 경로를 반드시 절대 경로로 지정합니다 (예: /Users/사용자이름/Desktop)
  2. macOS/Linux: ~/Desktop 대신 /Users/사용자이름/Desktop 사용
  3. Windows: C:\\Users\\사용자이름\\Desktop (역슬래시 두 개)
  4. 해당 디렉토리가 실제로 존재하는지 확인
GitHub 토큰 인증 실패 에러

원인: 토큰 권한이 부족하거나 만료된 경우입니다.

해결 방법:

  1. GitHub → Settings → Developer settings → Personal access tokens에서 토큰 재확인
  2. repo, read:org, read:user 권한이 모두 체크되어 있는지 확인
  3. 토큰 만료 여부 확인 (만료됐으면 새 토큰 발급)
  4. settings.json에 토큰 값이 올바르게 복사됐는지 공백·줄바꿈 없이 확인
Claude가 MCP 도구를 사용하지 않아요

원인: MCP 서버가 실제로 연결되지 않은 상태일 수 있습니다.

해결 방법:

  1. /mcp 명령어로 서버 연결 상태를 먼저 확인합니다
  2. 서버 상태가 connected인지 확인 (not connected면 위의 해결방법 참고)
  3. Claude Code를 완전 종료 후 재시작
  4. 요청을 더 명확하게 표현: "파일시스템 MCP를 사용해서 바탕화면 파일 목록 보여줘"
Windows에서 npx 명령어를 못 찾아요

원인: Node.js가 PATH에 등록되지 않았거나, command를 "npx"가 아닌 다른 방식으로 지정해야 할 수 있습니다.

해결 방법:

  1. Node.js를 공식 사이트에서 재설치 (PATH 자동 등록 선택)
  2. PowerShell에서 npx --version으로 인식 여부 확인
  3. 필요 시 절대 경로로 지정: "command": "C:\\Program Files\\nodejs\\npx.cmd"

학습 완료 체크리스트

  • MCP가 무엇인지 설명할 수 있다
  • Node.js v18 이상 설치 완료
