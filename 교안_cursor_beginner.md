## 1. 바이브코딩이란?

**바이브코딩(Vibe Coding)**은 AI에게 자연어로 지시하여 소프트웨어를 만드는 새로운 개발 방식입니다. 코드를 직접 작성하지 않고, "이런 것을 만들어 줘"라고 말하면 AI가 코드를 생성합니다.

### 왜 주목받고 있을까?

| 기존 개발 방식 | 바이브코딩 |
|---------------|-----------|
| 프로그래밍 언어 학습 필요 | 자연어(한국어)로 지시 |
| 수백 줄의 코드를 직접 작성 | AI가 코드를 자동 생성 |
| 디버깅에 많은 시간 소요 | AI가 오류를 자동 수정 |
| 개발자만 가능 | 누구나 시작 가능 |

### 바이브코딩 학습 경로

```
AI Studio (입문) → Lovable (웹앱) → AntiGravity (에이전트) → Cursor (AI IDE) → Claude Code (CLI)
                                                                ↑ 지금 여기!
```

> **활용 가이드**: "바이브코딩은 코딩 실력이 없어도 아이디어만 있으면 시작할 수 있습니다. 커서는 그중에서도 AI와 함께 코드를 직접 다루며 배울 수 있는 도구입니다." — 활용 가이드

## 2. 커서(Cursor)란?

**커서(Cursor)**는 Anysphere가 개발한 **AI 네이티브 코드 에디터(IDE)**입니다. VS Code를 기반(포크)으로 만들어져 친숙한 인터페이스를 제공하면서도, AI가 코드 작성 전 과정을 보조합니다. 2026년 3월 기준 최신 버전은 **v2.6**입니다.

### 러버블·안티그래비티와 뭐가 다른가요?

| 항목 | 러버블 (Lovable) | 안티그래비티 (AntiGravity) | 커서 (Cursor) |
|------|-----------------|--------------------------|---------------|
| **유형** | 웹 기반 노코드 빌더 | 데스크톱 에이전트 IDE | AI 네이티브 코드 에디터 |
| **설치** | 불필요 (브라우저) | 데스크톱 앱 설치 | 데스크톱 앱 설치 |
| **작동 방식** | 채팅 → 완성 결과 | 에이전트 위임 → 자율 실행 | AI 보조 → 개발자가 코드 제어 |
| **코딩 필요** | 불필요 | 선택적 | 선택적 (이해하면 유리) |
| **핵심 강점** | 빠른 프로토타입 | 복잡한 프로젝트, 병렬 에이전트 | 정밀한 코드 제어, 자유로운 스택 |
| **비용** | 유료 (이벤트 시 무료) | 퍼블릭 프리뷰 무료 | Hobby 무료 / Pro $20/월 |

### 커서의 3가지 핵심 AI 기능

| 기능 | 설명 |
|------|------|
| **Tab (자동완성)** | 코드를 타이핑할 때 AI가 다음에 올 코드를 자동으로 제안하고, Tab 키로 수락 |
| **Composer (다중 파일 편집)** | 여러 파일에 걸친 변경 사항을 한 번의 지시로 일괄 생성·수정 |
| **Agent (자율 실행)** | 프로젝트 전체 맥락을 이해하고, 터미널 명령어 실행까지 자율적으로 수행 |

> **활용 가이드**: "커서는 '내가 직접 코드를 작성하되, AI가 옆에서 도와주는 것'입니다. 러버블이 완전 자동이라면, 커서는 반자동 — 내가 운전하고 AI가 네비게이션을 해주는 느낌입니다." — 활용 가이드

### 참고: VS Code와의 호환성

커서는 VS Code 포크이므로 **기존 VS Code 확장(Extensions)을 그대로 사용**할 수 있습니다. VS Code에서 사용하던 설정과 확장을 그대로 가져올 수 있어 전환이 쉽습니다.

## 3. 용어 해설

커서를 사용하면서 자주 만나는 용어들입니다.

| 용어 | 뜻 | 쉬운 비유 |
|------|-----|-----------|
| **Tab (자동완성)** | 코드 작성 중 AI가 다음 코드를 제안 | 문장을 치면 자동완성 후보가 뜨는 것 |
| **Composer** | 여러 파일을 한 번에 편집하는 AI 모드 | 여러 서류를 동시에 수정하는 비서 |
| **Agent** | 맥락을 이해하고 자율적으로 작업하는 AI 모드 | 일을 시키면 알아서 해결하는 직원 |
| **Background Agent** | 백그라운드에서 비동기로 작업하는 에이전트 | 뒤에서 조용히 일하는 인턴 |
| **.cursor/rules** | 프로젝트별 AI 동작 규칙 파일 (MDC 형식) | 프로젝트 전용 업무 매뉴얼 |
| **MCP** | 외부 서비스와 연결하는 프로토콜 | USB 같은 연결 규격 |
| **Context (맥락)** | AI가 참조하는 코드·파일 정보 | 책상 위에 올려놓은 참고 자료 |
| **Codebase Indexing** | 프로젝트 전체 코드를 AI가 미리 분석 | 도서관 책에 색인을 다는 작업 |
| **Checkpoint** | 코드 변경 이력을 저장하여 되돌릴 수 있는 지점 | 게임의 세이브 포인트 |
| **Chat** | AI와 대화하며 질문·요청하는 창 | 카카오톡으로 AI에게 물어보기 |
| **Inline Edit** | 코드 내 특정 부분만 선택하여 AI에게 수정 요청 | 형광펜으로 밑줄 치고 "여기 고쳐줘" |
| **Terminal Cmd+K** | 터미널에서 자연어로 명령어를 생성 | "이 폴더 목록 보여줘" → `ls -la` 자동 생성 |

## 4. 사전 준비

### 필요한 것

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| **이메일 계정** | 가입용 (Google, GitHub 등) | 필수 |
| **데스크톱 앱** | cursor.com에서 다운로드 | 필수 |
| **Node.js** | 웹 프로젝트 실행에 필요 (권장) | 권장 |
| **Git** | 버전 관리 및 배포에 필요 | 권장 |

### 설치 가이드

#### Step 1: 커서 다운로드 및 설치

1. [cursor.com](https://www.cursor.com) 접속
2. "Download" 버튼 클릭
3. 운영체제에 맞는 설치 파일 다운로드:
   - **Windows**: `.exe` 설치 파일 실행
   - **Mac**: `.dmg` 파일 열기 → Applications로 드래그
   - **Linux**: `.AppImage` 또는 `.deb` 패키지 설치
4. 설치 완료 후 앱 실행

#### Step 2: 계정 생성 및 로그인

1. 커서 앱 실행 시 로그인 화면 표시
2. Google, GitHub, 또는 이메일로 가입
3. 무료 Hobby 플랜 자동 적용

#### Step 3: VS Code 설정 가져오기 (선택)

1. 기존 VS Code 사용자라면 설정 임포트 가능
2. 커서 첫 실행 시 "Import VS Code Settings" 옵션 선택
3. 확장, 키 바인딩, 테마가 자동으로 가져와짐

### ✅ 사전 준비 체크리스트

- [ ] 이메일 또는 GitHub 계정 준비 완료
- [ ] 커서 데스크톱 앱 설치 완료
- [ ] 앱 실행 및 로그인 완료
- [ ] Node.js 설치 확인 (터미널에서 `node -v` 입력)

## 6. 프롬프트 작성하기

### Chat vs Composer vs Agent — 언제 뭘 쓸까?

| 모드 | 언제 사용? | 예시 |
|------|-----------|------|
| **Chat** | 코드에 대해 질문하거나 설명이 필요할 때 | "이 함수가 뭐 하는 건지 설명해줘" |
| **Composer** | 여러 파일을 한 번에 만들거나 수정할 때 | "로그인 페이지와 API 라우트를 만들어줘" |
| **Agent** | 프로젝트 전체를 자율적으로 빌드할 때 | "이 PRD를 기반으로 전체 앱을 만들어줘" |

### 효과적인 프롬프트 패턴

**나쁜 예시:** `앱 만들어줘` (너무 모호!)

**좋은 예시:**
```
할 일 관리 웹앱을 만들어주세요.
## 기술 스택
- React + TypeScript + TailwindCSS + localStorage
## 핵심 기능
1. 할 일 추가/삭제/완료 체크
2. 카테고리별 분류 (업무, 개인, 학습)
3. 우선순위 설정 (높음/중간/낮음)
4. 완료율 통계 표시
## 디자인
- 깔끔한 미니멀 디자인, 모바일 반응형
```

### PRD(요구정의서) 간단 소개

PRD(Product Requirements Document)는 "무엇을 만들지" 정리한 기획 문서입니다. Agent 모드에서 PRD를 전달하면 AI가 체계적으로 프로젝트를 빌드합니다.

```
# 스터디 타이머 PRD
## 목적
학생들의 집중력 향상을 위한 포모도로 타이머 웹앱
## 주요 기능
1. 25분/5분 타이머 (포모도로 기법)
2. 시작, 일시정지, 리셋 버튼
3. 완료 세션 카운터 + 알림음
## 기술 스택: React + TypeScript + TailwindCSS
## 디자인: 미니멀, 다크 모드 기본, 모바일 반응형
```

> **활용 가이드**: "커서 Agent 모드에 PRD를 전달하면, AI가 프로젝트 구조를 알아서 잡고 파일을 하나씩 만들어갑니다. 중간에 체크포인트가 생기니 마음에 안 들면 되돌릴 수 있습니다." — 활용 가이드

## 7. 커서(Cursor) 실습: 나만의 프로젝트 만들기

### Step 1: 프로젝트 폴더 열기

1. 커서 앱 실행
2. `File → Open Folder` 클릭
3. 새 폴더를 생성하거나 기존 폴더 선택 (예: `my-first-app`)
4. 폴더가 열리면 좌측에 파일 탐색기가 표시됨

### Step 2: Chat으로 계획 세우기

1. `Ctrl+L` (Mac: `Cmd+L`)을 눌러 Chat 패널 열기
2. 만들고 싶은 프로젝트에 대해 AI와 대화:

```
포모도로 타이머 웹앱을 만들고 싶어.
어떤 기술 스택을 사용하는 게 좋을까?
프로젝트 구조는 어떻게 잡으면 좋을지 알려줘.
```

3. AI가 추천한 구조와 기술 스택을 확인

### Step 3: Composer로 여러 파일 생성

1. `Ctrl+I` (Mac: `Cmd+I`)를 눌러 Composer 열기
2. 프롬프트 입력:
```
포모도로 타이머 웹앱을 만들어줘.
- React + TypeScript + TailwindCSS
- 25분 집중 / 5분 휴식 타이머
- 시작/일시정지/리셋 버튼, 완료 세션 카운터
- 미니멀한 다크 모드 디자인
```
3. AI가 여러 파일의 변경 사항을 제안
4. 각 파일의 Diff(변경 내용)를 확인 후 **Accept** 또는 **Reject**

### Step 4: Agent 모드로 자율 빌드

1. Composer에서 모드를 **Agent**로 전환
2. Agent에게 전체 프로젝트 빌드 요청:
```
이 프로젝트를 처음부터 세팅해줘.
1. 프로젝트 초기화 (npm create vite 등)
2. 필요한 패키지 설치
3. 모든 컴포넌트 파일 생성
4. 스타일링 적용 후 개발 서버 실행
```
3. Agent가 터미널 명령어 실행, 파일 생성, 패키지 설치를 자동으로 수행
4. 중간에 **Checkpoint**가 자동 생성되어 되돌리기 가능

### Step 5: 결과 확인 및 수정

1. 개발 서버가 실행되면 브라우저에서 결과 확인 (보통 `http://localhost:5173`)
2. 수정이 필요하면 Chat 또는 Composer로 추가 요청:
```
타이머 숫자 폰트를 더 크게 해줘.
버튼을 둥글게 만들고, 호버 시 색상이 부드럽게 전환되게 해줘.
```
3. 특정 코드 부분만 수정하고 싶으면 **Inline Edit** 사용:
   - 코드 블록 선택 → `Ctrl+K` (Mac: `Cmd+K`) → 수정 요청 입력

> **활용 가이드**: "커서의 Checkpoint 기능을 적극 활용하세요. Agent가 코드를 크게 변경한 후 마음에 안 들면, Checkpoint로 이전 상태로 쉽게 되돌릴 수 있습니다." — 활용 가이드

## 10. 문제가 생겼을 때 (트러블슈팅)

### 상황 1: 설치가 안 돼요

| 문제 | 해결 방법 |
|------|----------|
| 다운로드 오류 | 공식 사이트(cursor.com)에서 다시 다운로드 |
| Windows 설치 오류 | 설치 파일 우클릭 → "관리자 권한으로 실행" |
| Mac 보안 차단 | 시스템 설정 → 보안 및 개인정보 → "확인 없이 열기" |
| Linux 권한 오류 | `chmod +x cursor.AppImage` 후 실행 |

### 상황 2: Tab 자동완성이 작동하지 않아요

1. 커서 설정(`Settings`)에서 Tab 기능이 활성화되어 있는지 확인
2. 파일이 지원되는 언어인지 확인 (대부분 지원됨)
3. 무료 플랜의 일일 사용량이 초과되었을 수 있음 → 잠시 기다리기
4. 커서 앱 재시작

### 상황 3: Agent가 작업 도중 멈춰요

1. 인터넷 연결 확인
2. Agent 모드를 중단 후 재시작
3. 요청을 더 작은 단위로 나눠서 재시도
4. 다른 AI 모델로 전환 시도 또는 무료 플랜 제한 대기

### 상황 4: 코드베이스 인덱싱이 느려요

1. 프로젝트 크기가 매우 클 경우 시간이 걸릴 수 있음
2. `.cursorignore` 파일에 불필요한 폴더 제외:
   ```
   node_modules/
   .git/
   dist/
   build/
   ```
3. 커서 하단 상태바에서 인덱싱 진행률 확인

### 상황 5: 모델 사용 한도에 도달했어요

1. Hobby (무료) 플랜은 일일/월간 요청 제한이 있음
2. 제한 리셋까지 기다리거나 Pro 플랜 업그레이드 고려
3. 간단한 작업은 Tab 자동완성으로 해결 (Agent보다 적은 할당량 소모)

## 11. 자주 묻는 질문 (FAQ)

### Q1. 커서는 무료인가요?

**Hobby 플랜은 무료**입니다. 신용카드 없이 가입할 수 있으며 Tab 자동완성과 Agent를 제한적으로 사용할 수 있습니다. 더 많은 사용량이 필요하면 Pro($20/월)로 업그레이드할 수 있습니다.

### Q2. VS Code와 뭐가 다른가요?

커서는 VS Code를 포크(fork)하여 만들어졌으므로 인터페이스가 거의 동일합니다. 핵심 차이는 **AI 기능(Tab, Composer, Agent)이 기본 내장**되어 있다는 점입니다. VS Code의 확장(Extensions)도 그대로 사용할 수 있습니다.

### Q3. 한국어로 사용할 수 있나요?

앱 인터페이스(메뉴, 버튼)는 영어이지만, **AI에게 한국어로 지시**할 수 있습니다. 한국어 프롬프트도 잘 이해하고 한국어로 응답합니다. 코드 주석도 한국어로 작성 가능합니다.

### Q4. 안티그래비티와 뭐가 다른가요?

커서는 **개발자가 AI 보조를 받으며 코드를 직접 제어**하는 방식이고, 안티그래비티는 **에이전트에게 작업을 위임하여 자율 실행**하는 방식입니다. 커서는 코드를 직접 보고 수정하며, 안티그래비티는 Manager View에서 에이전트를 관리합니다.

### Q5. 코딩을 전혀 몰라도 사용할 수 있나요?

Agent 모드를 사용하면 코딩 경험 없이도 프로젝트를 만들 수 있습니다. 하지만 파일 구조, 터미널 명령어 등 **기본 개념을 이해하면 훨씬 효과적**입니다. 완전 입문자라면 **러버블**부터 시작하는 것을 추천합니다.

### Q6. VS Code 확장(Extension)을 그대로 쓸 수 있나요?

네! 커서는 VS Code 포크이므로 **거의 모든 VS Code 확장이 호환**됩니다. VS Code에서 사용하던 확장을 그대로 설치하고 사용할 수 있습니다.

### Q7. 코드가 클라우드로 전송되나요?

AI 기능을 사용할 때 코드가 AI 모델 서버로 전송됩니다. 커서는 **Privacy Mode** 설정을 제공하여 코드가 학습 데이터로 사용되지 않도록 할 수 있습니다. 민감한 프로젝트라면 설정에서 Privacy Mode를 활성화하세요.

### Q8. 여러 기기에서 사용할 수 있나요?

하나의 계정으로 **Mac, Windows, Linux** 어디서든 로그인하여 사용할 수 있습니다. 단, 설정과 확장은 기기별로 따로 설치해야 합니다.

## 12. 주의사항 및 팁

### 보안 관련 이해하기

커서에서 AI 기능을 사용하면 코드가 클라우드의 AI 모델로 전송됩니다:

| 항목 | 설명 |
|------|------|
| **Privacy Mode** | 설정에서 활성화하면 코드가 학습 데이터로 사용되지 않음 |
| **민감 정보** | API 키, 비밀번호 등은 `.env` 파일에 저장하고 `.gitignore`에 추가 |
| **Codebase Indexing** | 민감한 파일은 `.cursorignore`에 추가하여 제외 |

### 무료 플랜 한도 관리

Hobby 플랜은 Tab/Agent/Composer 모두 일일 또는 월간 제한이 있습니다. 간단한 코드는 직접 작성하고, 큰 작업에만 Agent를 사용하며, 변경 사항을 모아서 한 번에 요청하면 효율적입니다.

### 자주 하는 실수

| 실수 | 올바른 방법 |
|------|-----------|
| 프롬프트가 너무 모호함 | 기술 스택, 기능, 디자인을 구체적으로 명시 |
| `@file` 참조를 안 씀 | Chat/Composer에서 `@파일명`으로 참조 파일 지정 |
| Checkpoint를 무시함 | 큰 변경 전에 Checkpoint 확인, 필요 시 되돌리기 |
| Agent에게 너무 큰 작업 요청 | 작은 단위로 나눠서 요청하면 정확도 향상 |
| 에러 메시지를 직접 검색 | AI에게 에러 내용을 붙여넣고 수정 요청 |

### 실전 노하우

1. **`@` 기호를 적극 활용하세요**: `@파일명`, `@폴더명`, `@docs`로 AI에게 정확한 맥락 전달
2. **Checkpoint로 안전하게 실험하세요**: 큰 변경 시도 전후로 Checkpoint가 자동 생성됨
3. **단계별로 진행하세요**: 기본 구조 → 핵심 기능 → 디자인 → 세부 조정 순서로
4. **.cursor/rules를 만드세요**: 프로젝트 규칙을 파일로 저장하면 AI가 일관된 코드를 작성
5. **Terminal Cmd+K를 활용하세요**: 터미널 명령어가 기억나지 않을 때 자연어로 입력
