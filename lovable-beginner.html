1. 바이브코딩이란?

바이브코딩(Vibe Coding) 은 코딩을 직접 하지 않고, AI에게 일상 언어로 원하는 것을 설명하면 AI가 코드를 작성해주는 방식입니다.

  • 프로그래밍 언어를 몰라도 됩니다
  • "이런 웹사이트를 만들어줘"라고 말하면 AI가 만들어줍니다
  • 마치 디자이너에게 의뢰하듯이 대화로 진행합니다

왜 주목받고 있을까?

  • 아이디어만 있으면 누구나 웹사이트/앱을 만들 수 있습니다
  • 전문 개발자를 고용하지 않아도 프로토타입을 빠르게 만들 수 있습니다
  • 교육, 소상공인, 프리랜서, 동호회 등 다양한 분야에서 활용 가능합니다

바이브코딩 학습 경로

처음이라면 아래 순서로 단계별로 익혀가는 것을 추천합니다:

1단계: Google AI Studio 빌더 (무료, 가장 쉬움)
   ↓  "아~ 이런 거구나" 느낌 잡기
2단계: 러버블 Lovable (이벤트 시 무료)
   ↓  본격적인 웹사이트 제작
3단계: 커서 Cursor 등 (코드 직접 제어)
   ↓  코딩에 관심이 생겼다면
실사용자 조언: "AI 스튜디오 빌더만 해봐도 '아~ 이런 거구나' 느낌을 받으시더라고요. 그 다음에 러버블로 넘어가면 훨씬 수월합니다" — 실사용자 조언

2. 러버블(Lovable)이란?

Lovable(lovable.dev)은 AI 기반 웹 앱 빌더입니다. 채팅하듯이 원하는 것을 말하면 웹사이트를 만들어줍니다.

러버블의 특징

장점설명
설치 불필요웹 브라우저만 있으면 바로 시작 가능
대화형 제작채팅하듯이 원하는 기능을 요청
즉시 미리보기만든 결과물을 실시간으로 확인 가능
원클릭 배포클릭 몇 번으로 인터넷에 공개 가능
한국어 프롬프트UI는 영어지만, 요청은 한국어로 해도 잘 동작
실전 팁: "대부분 설치하다 끝나는데, 러버블은 따로 설치할 게 없으니 신세계입니다" — 실사용자 후기

추천 영상: 러버블로 5분 만에 사이트를 만드는 과정을 직접 확인해보세요 → YouTube 보기

알아두세요: 러버블 화면은 영어입니다

러버블의 버튼과 메뉴는 영어로 되어 있지만 걱정하지 마세요:

  • 요청(프롬프트)은 한국어로 작성해도 됩니다
  • 주요 버튼만 기억하면 충분합니다:
    • New Project = 새 프로젝트 만들기
    • Chat = 대화 모드 (꼭 활성화!)
    • Preview = 미리보기
    • Publish = 바로 공개하기
    • Download = 파일 내려받기

4. 사전 준비

러버블을 시작하기 전에 아래 3가지 계정을 미리 만들어두세요.

4-1. GitHub 가입

GitHub은 코드를 저장하고 관리하는 서비스입니다. 만든 프로젝트를 안전하게 보관하는 데 사용합니다.

  1. github.com 접속
  2. Sign up 클릭
  3. 이메일, 비밀번호, 사용자 이름 입력
  4. 이메일 인증 완료

4-2. Vercel 가입

Vercel은 웹사이트를 인터넷에 공개(배포)해주는 서비스입니다. 무료로 사용 가능합니다.

  1. vercel.com 접속
  2. Sign Up 클릭
  3. Continue with GitHub 선택 (GitHub 계정으로 간편 가입)

4-3. Lovable 가입

  1. lovable.dev 접속
  2. Sign up 또는 Get Started 클릭
  3. GitHub 계정으로 연동 가입 추천

사전 준비 체크리스트

  • GitHub 계정 생성 완료
  • Vercel 계정 생성 완료 (GitHub 연동)
  • Lovable 계정 생성 완료
  • 만들고 싶은 웹사이트 아이디어 1개 이상 구상

5. 아이디어 구상하기

러버블에 바로 뛰어들기 전에, 무엇을 만들 것인지 먼저 정리하는 것이 중요합니다.

시작 미션 3가지

무엇을 만들지 아직 정하지 못했다면, 아래 미션을 따라해보세요:

  1. 기획 대화하기: 제미나이/ChatGPT에게 하고 싶은 것을 이야기하며 기획하기 (기획 도우미 Gem 활용)
  2. 아이디어 10개 뽑기: 하고 싶은 게 없다면 AI에게 "24시간 동안 할 수 있는 프로젝트 10개를 추천해줘"라고 요청하기
  3. 디자인 참고 수집: 다른 앱이나 사이트를 둘러보며 마음에 드는 디자인을 캡처해두기
추천 첫 프로젝트: 뭘 만들지 모르겠다면 개인 포트폴리오 웹사이트부터 시작해보세요! 가장 간단하면서도 완성감이 있습니다.

6. PRD(요구정의서) 만들기

PRD란?

PRD 는 "내가 만들 것을 정리한 기획서"입니다. 러버블에 입력하기 전에 PRD를 만들어두면 훨씬 좋은 결과물을 얻을 수 있습니다.

실전 팁: "그냥 막 던지면 코드 수정이 어렵거나 돌아올 수 없는 강을 건너기도 해요" — 실사용자 조언

"만들고 있던 앱의 명세서를 넣고 한 번 하니까 크레딧이 끝났어요" — 실사용자 경험

왜 PRD가 필요할까?

  • 없이 시작하면: 크레딧 낭비 + 결과물 뒤죽박죽 + 수정에 수정 반복
  • 있으면: 한 번에 원하는 결과에 가까운 결과물 + 크레딧 절약

간단한 PRD 작성법

AI에게 아래와 같이 요청하세요:

아까 정리한 [프로젝트 이름]에 대해
PRD(요구정의서)를 작성해줘.

아래 항목을 포함해줘:
1. 프로젝트 이름
2. 프로젝트 목적
3. 누가 사용하나?
4. 어떤 기능이 필요한가?
5. 어떤 페이지가 필요한가?
6. 디자인 느낌 (색상, 분위기)
PRD 예시 보기 (카페 소개 사이트) 
# PRD: 해피빈 카페 소개 사이트

## 프로젝트 목적
카페를 방문하려는 손님에게 필요한 정보를 한눈에 제공

## 대상 사용자
- 카페를 찾는 손님
- 카페 사장님 (관리자)

## 핵심 기능
1. 카페 소개 (스토리, 분위기 사진)
2. 메뉴판 (음료/디저트, 가격)
3. 영업시간 및 위치
4. 인스타그램/네이버 지도 링크

## 페이지 구성
- 홈: 카페 대표 이미지 + 한 줄 소개
- 메뉴: 음료/디저트 목록
- 오시는 길: 주소, 영업시간, 지도
- 소개: 카페 이야기

## 디자인 요구사항
- 따뜻하고 아늑한 느낌 (베이지, 브라운 톤)
- 사진이 돋보이는 레이아웃
- 스마트폰에서도 잘 보이게

7. 러버블 실습: 나만의 사이트 만들기

이벤트 시간 활용 전략 (25시간 무료)

주의: 이벤트 시작 전에 러버블을 사용하지 마세요!

무료 25시간은 이벤트 시간에만 적용됩니다. 이벤트 시작 전에 러버블에서 빌드하면 유료 크레딧이 소비됩니다. 실제로 시작 시간을 모르고 미리 사용해서 크레딧을 소진한 사례가 있습니다. 이벤트 시작 전에는 기획(PRD)만 준비하고, 러버블 실습은 이벤트가 시작된 후에 하세요.
시간대할 일설명
시작 전아이디어 + PRD 준비AI 도구로 미리 기획 완료 (러버블 사용 금지!)
처음 1시간첫 빌드 + 전체 구조 확인PRD를 넣고 기본 사이트 생성
2~3시간세부 수정디자인, 내용, 레이아웃 조정
4시간~배포 + 추가 실험GitHub 업로드 → Vercel 배포

Step 1: 프로젝트 시작

  1. lovable.dev 로그인
  2. New Project (새 프로젝트) 클릭
  3. 프로젝트 이름 입력 (예: "my-cafe-site")

Step 2: Chat 버튼 활성화 (매우 중요!)

반드시 기억하세요: 러버블에서 Chat 버튼을 꼭 활성화하세요!

Chat 기능을 켜고 요청하면 러버블이 먼저 생각하고 보완한 후에 코드를 생성합니다. 약 1크레딧이 추가로 소모되지만 훨씬 나은 결과를 받을 수 있습니다. 이것은 일종의 "플랜 모드" 로, AI가 바로 만들지 않고 계획을 세운 후 만듭니다.

Step 3: 첫 프롬프트 입력

Chat 창에 준비한 PRD를 통째로 붙여넣습니다. PRD가 없다면 아래처럼 입력해도 됩니다:

예시 (소상공인 - 카페) 
한국어로 된 카페 소개 웹사이트를 만들어주세요.

포함할 내용:
- 상단에 카페 이름과 대표 이미지
- 메뉴판 (음료, 디저트 목록과 가격)
- 오시는 길 (주소, 영업시간)
- 카페 소개 스토리
- 하단에 인스타그램 링크

디자인:
- 따뜻한 베이지/브라운 톤
- 사진이 돋보이는 레이아웃
- 스마트폰에서도 잘 보이게
예시 (교사 - 학급 사이트) 
한국어로 된 학급 소식 웹사이트를 만들어주세요.

포함할 내용:
- 학급 공지사항 게시판
- 주간 학습 계획표
- 활동 사진 갤러리
- 담임 교사 연락처

디자인:
- 밝고 귀여운 파스텔 톤
- 아이들 느낌의 따뜻한 디자인
- 스마트폰에서도 잘 보이게
예시 (프리랜서 - 포트폴리오) 
한국어로 된 개인 포트폴리오 웹사이트를 만들어주세요.

포함할 내용:
- 상단에 이름과 한 줄 소개
- 자기소개 섹션
- 작업물 목록 (카드 형태, 이미지 + 설명)
- 보유 기술 또는 관심 분야
- 연락처 (이메일, SNS 링크)

디자인:
- 깔끔하고 모던한 느낌
- 스마트폰에서도 잘 보이게
- 부드러운 스크롤 효과

Step 4: 결과 확인 및 수정

  1. 러버블이 코드를 생성하면 미리보기(Preview) 에서 결과를 확인합니다
  2. Visual Edits(디자인 모드) 에서 색상, 레이아웃을 직접 변경할 수도 있습니다
  3. 마음에 들지 않는 부분이 있으면 대화로 수정 요청합니다

수정 요청 예시 (초보자 친화적):

- 전체적인 색상을 좀 더 따뜻한 느낌으로 바꿔주세요
- 메뉴 카드를 좀 더 크게 만들어주세요
- 상단 메뉴바를 스크롤해도 항상 보이게 고정해주세요
- 글자 크기를 좀 더 키워주세요
팁: "좀 더 이쁘게"처럼 모호한 요청보다는 구체적으로 요청하세요. "배경색을 연한 노란색으로", "글자를 더 크게", "사진을 왼쪽에 배치" 등

Step 5: 프로젝트 저장

  • 러버블은 자동 저장됩니다
  • 파일 다운로드: 마음에 드는 결과물은 꼭 소스 코드를 다운로드해두세요
    • 나중에 다른 도구에서 이어서 작업할 수 있습니다

8. 웹사이트 공개하기 (배포)

만든 웹사이트를 인터넷에 공개하는 방법은 3가지가 있습니다.

방법 1: 러버블 자체 배포 (가장 쉬움)

러버블에는 Publish 버튼이 있어서 클릭 한 번으로 바로 공개할 수 있습니다.

  1. 프로젝트 화면에서 Publish 버튼 클릭
  2. 러버블이 자동으로 URL을 생성해줍니다
  3. 해당 URL을 공유하면 끝!
가장 간단하지만, 러버블 서비스에 의존하게 됩니다.

방법 2: GitHub + Vercel 배포 (추천)

좀 더 안정적이고 나만의 주소를 가질 수 있는 방법입니다.

GitHub에 업로드:

  1. 러버블 프로젝트 화면에서 GitHub 연동 버튼 찾기
  2. 연동할 저장소(Repository) 선택 또는 새로 생성
  3. Push 또는 Sync 클릭

Vercel에 배포:

  1. vercel.com 로그인
  2. Add New Project 클릭
  3. Import Git Repository 에서 방금 올린 저장소 선택
  4. Deploy 클릭
  5. 잠시 기다리면 배포 완료!

배포 결과:

  • https://프로젝트이름.vercel.app 형태의 URL이 생성됩니다
  • 이 URL을 공유하면 누구나 웹사이트를 볼 수 있습니다

방법 3: 수동 업로드 (GitHub 파일 업로드)

Git 명령어를 모르는 경우:

  1. 러버블에서 프로젝트 파일 Download (ZIP 다운로드)
  2. GitHub에서 New Repository 생성
  3. Upload files → 다운로드한 파일을 드래그 앤 드롭
  4. Commit changes 클릭
  5. 이후 Vercel에서 Import → Deploy

10. 문제가 생겼을 때 (트러블슈팅)

"결과물이 마음에 안 들어요"
  • 구체적으로 수정 요청하세요: "전체적으로 바꿔줘" 대신 "헤더 부분의 색상을 연한 파란색으로 변경해줘"
  • 한 번에 하나씩 수정하세요: 여러 개를 한꺼번에 요청하면 엉킬 수 있습니다
  • PRD를 다시 확인하세요: 처음 의도와 다르면 PRD 기반으로 재요청
"크레딧이 빨리 줄어들어요"
  • 무료 이벤트 시간을 활용하세요 (이벤트 중에는 무제한)
  • PRD를 미리 작성해서 한 번에 정확하게 요청하세요
  • 수정도 구체적으로 — 모호한 요청은 원하지 않는 결과 → 재수정 → 크레딧 낭비
"러버블이 이상한 결과를 만들었어요"
  1. 당황하지 마세요 — 러버블에는 되돌리기(Undo) 기능이 있습니다
  2. 이전 버전으로 돌아가서 다시 시도하세요
  3. 마음에 드는 상태에서 미리 파일을 다운로드해두면 안전합니다
"GitHub/Vercel 연동이 안 돼요"
  • GitHub 계정으로 Lovable에 가입했는지 확인하세요
  • Lovable 설정에서 GitHub 연결 권한을 허용했는지 확인하세요
  • 그래도 안 되면 방법 3(수동 업로드) 을 시도하세요
"배포 후 사이트가 안 보여요"
  • Vercel에서 배포 상태가 Ready(녹색)인지 확인하세요
  • 빌드 에러가 있다면 러버블에서 코드를 수정하고 다시 Push하세요
  • 5~10분 기다려보세요 — 때로 시간이 걸릴 수 있습니다
"카카오톡에 공유했는데 썸네일이 안 나와요"
  • 카카오톡은 한번 읽어간 썸네일(OG 이미지)을 캐시합니다
  • 사이트를 수정해도 카톡 미리보기가 바로 바뀌지 않습니다
  • 해결 방법: 카카오 OG 캐시 초기화 페이지에서 URL을 입력하고 "캐시 초기화" 버튼을 누르세요
  • OG 이미지가 아예 없다면 러버블 프로젝트에 "OG 메타 태그와 썸네일 이미지를 추가해줘"라고 요청하세요

11. 자주 묻는 질문 (FAQ)

Q: 무료 계정이면 몇 번 대화할 수 있나요?

러버블 무료 계정은 제한된 크레딧이 주어집니다. Chat 모드 1회 요청에 약 1크레딧이 소모되어 금방 소진될 수 있습니다. 이벤트 기간(25시간 무료)을 적극 활용하세요.

Q: 러버블에 자체 데이터베이스가 있나요?

아니요. 러버블 자체에는 데이터베이스가 없습니다. 정적인 소개 사이트는 DB 없이도 가능하지만, 게시판처럼 데이터를 저장하려면 외부 서비스(Supabase, Firebase 등)를 연결해야 합니다.

Q: 프롬프트를 한국어로 써도 되나요?

네! 러버블의 화면(UI)은 영어이지만, 프롬프트(요청)는 한국어로 작성해도 잘 동작합니다. 만들어지는 웹사이트 내용도 한국어로 생성됩니다.

Q: 이벤트 시작 전에 만들어도 되나요?

이벤트 시작 전에는 무료 크레딧이 제한되어 있어서, 금방 제한에 걸릴 수 있습니다. 아이디어와 PRD는 미리 준비하고, 실제 빌드는 이벤트 시작 후에 하는 것을 추천합니다.

Q: 만든 사이트를 나중에 수정할 수 있나요?

네! 러버블에서 언제든 수정 가능하고, GitHub에 올린 코드는 다른 도구(커서, VS Code 등)에서도 수정할 수 있습니다.

Q: 꼭 GitHub과 Vercel을 써야 하나요?

아니요. 러버블 자체 Publish 기능으로도 바로 공개할 수 있습니다. 다만 GitHub + Vercel을 쓰면 나만의 URL을 가질 수 있고, 코드를 안전하게 보관할 수 있습니다.

Q: 처음이라 아무것도 못 만들 것 같은데, 어떤 도구부터 시작하면 좋을까요?

바이브코딩 자체가 처음이라면 AI Studio 빌더부터 시작하는 것도 좋습니다. 설치 없이 '아~ 이런 거구나' 느낌을 받을 수 있어요. 그 다음에 러버블로 넘어오면 훨씬 수월합니다. "휘둘리지 않으셔도 돼요. 하나에 익숙해지는 게 중요합니다" — 실사용자 조언

Q: 러버블 크레딧과 $100 API 크레딧은 다른 건가요?

네, 완전히 다릅니다. 러버블에서 채팅하며 사이트를 만드는 데 쓰이는 것은 러버블 크레딧(이벤트 중 무제한)이고, $100 API 크레딧은 외부에서 Claude API를 연동할 때 쓰는 별도 크레딧입니다. 초보자는 러버블 크레딧만으로 충분하며, API 크레딧은 나중에 고급 기능을 붙일 때 사용합니다.

Q: 디자인 색상을 직접 바꿀 수 있나요?

네! 러버블에는 Visual Edits(디자인 모드) 기능이 있어서 색상, 텍스트, 레이아웃 등을 대화 없이 직접 수정할 수 있습니다. 채팅 대신 화면에서 직접 클릭하여 변경하는 방식이라 초보자도 쉽게 사용할 수 있습니다.

Q: 러버블로 만든 후 구조를 몰라도 괜찮을까요?

처음에는 괜찮지만, 프로젝트가 커지면 구조를 이해하지 못하면 수정이 어렵고 크레딧도 많이 소모됩니다. 그래서 이 교안에서 PRD(기획서)를 먼저 작성하는 것을 강조하는 것입니다. 기본적인 구조를 이해하면서 진행하면 나중에 훨씬 수월합니다.
