MCP 초급 복습

이미 MCP 초급을 완료했다면 이 섹션은 빠르게 훑고 넘어가세요. 처음 보신다면 MCP 초급 교안을 먼저 학습하길 권장합니다.

핵심 개념 요약

MCP = Claude에게 새로운 능력을 연결하는 표준 프로토콜
기본 Claude는 대화만 할 수 있지만, MCP 서버를 연결하면 파일 읽기, GitHub 접근, 웹 검색 등 실제 작업을 수행할 수 있게 됩니다.

settings.json 기본 구조 빠른 복습

{ "mcpServers": { "filesystem": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem", "/workspace"], "env": {} }, "github": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"], "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_여기에토큰입력" } } } }

위 두 서버가 이미 설정된 분은 바로 다음 섹션으로 넘어가세요.

다중 서버 동시 운영

MCP의 진가는 여러 서버를 동시에 연결할 때 나타납니다. Claude는 필요에 따라 자동으로 적절한 서버를 선택해 작업합니다.

4개 서버 동시 운영 예시

{ "mcpServers": { "filesystem": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem", "/workspace"], "env": {} }, "github": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"], "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_여기에토큰입력" } }, "notion": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-notion"], "env": { "NOTION_API_KEY": "secret_여기에노션토큰" } }, "brave-search": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-brave-search"], "env": { "BRAVE_API_KEY": "여기에브레이브API키" } } } }

다중 서버 운영 주의사항

  • 서버 이름 중복 금지: mcpServers 안의 키 이름은 고유해야 합니다
  • 환경변수 분리: 각 서버의 토큰·API 키는 독립적으로 관리합니다
  • 상태 확인: /mcp 명령어를 입력하면 현재 연결된 모든 서버 상태를 한번에 확인할 수 있습니다
  • 서버 충돌 방지: 같은 리소스에 접근하는 서버 2개를 동시에 쓸 때는 권한 충돌에 유의하세요
팁: /mcp를 입력하면 연결된 서버 목록과 각 서버가 제공하는 도구(tools) 목록이 표시됩니다. 새 서버를 추가한 후 반드시 확인하세요.

추천 서버 조합 5가지

어떤 서버를 조합할지 고민된다면 아래 검증된 조합을 참고하세요. 나의 업무 유형에 맞는 조합을 골라 바로 적용할 수 있습니다.

💻 개발자 풀스택
filesystem github brave-search sqlite
코드 읽고 → GitHub 이슈 확인 → 웹 검색 → DB 쿼리까지 전체 개발 사이클을 Claude와 함께. 풀스택 개발자에게 가장 강력한 조합입니다.
📚 교육자 패키지
filesystem notion google-drive
수업자료 파일 관리 → 노션 교안 작성 → 드라이브 파일 첨부. 교사·강사·콘텐츠 크리에이터에게 최적화된 조합입니다.
📋 프로젝트 매니저
github notion slack memory
이슈 확인 → 노션 업데이트 → 슬랙 보고 → 맥락 기억. 팀을 이끄는 PM이 반복 작업을 자동화하는 데 최적입니다.
📊 데이터 분석가
filesystem sqlite brave-search
로컬 파일 → DB 분석 → 최신 정보 검색. CSV·엑셀 데이터를 DB에 넣고 Claude와 함께 분석하는 데이터 분석 워크플로우입니다.
🚀 1인 창업자
github notion slack brave-search
코드 관리 → 문서화 → 팀 소통 → 시장 조사. 혼자서 많은 것을 해야 하는 창업자가 Claude를 팀원처럼 활용하는 조합입니다.
🏢 기업 개발팀 올인원
slack jira (atlassian) confluence github
Jira 이슈 조회 → GitHub PR 연결 → Confluence 문서 업데이트 → Slack 보고. 기업 환경의 전형적인 개발 사이클을 Claude 한 명이 처리합니다.
📣 스프린트 자동화
jira (atlassian) slack memory
스프린트 시작 시 Jira 백로그 정리 → 담당자별 할당 → Slack에 데일리 스탠드업 요약 전송 → Memory로 진행 상황 기억. 스크럼 마스터 업무를 자동화합니다.

고급 설정 옵션

기본 설정을 넘어 환경에 맞게 세밀하게 조정하는 방법을 알아봅니다.

// stdio 방식 — 로컬에 설치된 서버 실행 (가장 일반적) { "filesystem": { "command": "npx", "args": [ "-y", "@modelcontextprotocol/server-filesystem", "/workspace" // 접근 허용할 폴더 경로 ], "timeout": 30000 // 응답 대기시간 (밀리초, 기본값 없음) } }
// SSE 방식 — HTTP 기반 원격 MCP 서버 연결 // 팀 공용 서버나 클라우드 서비스에 활용 { "remote-server": { "type": "sse", "url": "https://my-mcp-server.com/sse", "headers": { "Authorization": "Bearer my-token", "X-Team-ID": "team-123" } } }
// 환경변수 관리 — 토큰을 직접 쓰지 않는 방법 // settings.json에 직접 토큰 작성 대신 시스템 환경변수 참조 // Windows: 시스템 설정 > 환경변수 또는 PowerShell에서 // [System.Environment]::SetEnvironmentVariable("GITHUB_TOKEN","ghp_...","User") // Mac/Linux: ~/.zshrc 또는 ~/.bashrc에 // export GITHUB_TOKEN="ghp_..." // settings.json에서는 이렇게 참조 { "github": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"], "env": { "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_TOKEN}" } } } // .env 파일을 사용하는 경우 (프로젝트 로컬) // .env 파일을 반드시 .gitignore에 추가하세요!

자주 쓰는 추가 설정 옵션

옵션설명기본값
timeout서버 응답 대기 시간 (밀리초)없음 (무한 대기)
disabledtrue로 설정하면 서버를 비활성화false
env서버 실행 시 주입할 환경변수 맵{}
args서버 실행 명령어 인자 배열서버마다 다름

MCP와 함께하는 프롬프트 기법

MCP 서버가 여러 개 연결되어 있을 때, Claude에게 어떻게 말해야 원하는 도구를 정확히 활용하게 할 수 있을까요? 5가지 검증된 패턴을 소개합니다.

1. 명시적 도구 호출

사용할 서버를 명확히 지정하면 Claude가 어떤 도구를 써야 할지 즉시 파악합니다.

// 프롬프트
"GitHub MCP를 써서 my-project 레포의 최근 5개 PR 목록 보여줘"
→ GitHub 서버가 즉시 활성화되어 PR 목록과 각 PR의 제목, 상태, 작성자를 정리해 보여줍니다.

2. 여러 도구 조합 요청

여러 작업을 한 번에 지시하면 Claude가 필요한 서버를 순서대로 호출합니다.

// 프롬프트
"파일시스템에서 README.md 읽고, GitHub에서 오픈 이슈 확인한 다음, 노션에 현재 프로젝트 상태 요약 페이지 만들어줘"
→ filesystem → github → notion 순으로 세 서버를 연속 호출해 하나의 작업 흐름으로 처리합니다.

3. 데이터 분석 요청

// 프롬프트
"SQLite DB에서 지난달 매출 데이터 뽑아서 트렌드 분석해줘"
→ SQLite MCP로 쿼리를 실행하고, 결과 데이터를 기반으로 분석 리포트를 생성합니다.

4. 자동화 워크플로우

// 프롬프트
"Brave Search로 '리액트 최신 보안 취약점' 검색하고, 우리 코드베이스에서 취약한 패턴 찾아서 GitHub 이슈 생성해줘"
→ 웹 검색 → 로컬 코드 분석 → GitHub 이슈 생성까지 3단계를 자동으로 처리합니다.

5. 메모리 서버 활용

// 프롬프트
"이번 미팅 내용을 Memory MCP에 저장해줘. 다음에 '지난 미팅' 물어보면 꺼내쓸 거야"
→ 세션이 끊겨도 맥락이 유지됩니다. 장기 프로젝트에서 진행 상황을 기억시키는 데 매우 유용합니다.

6. Slack + Jira 연계 자동화

// 프롬프트 — 스탠드업 자동화
"Jira에서 오늘 완료된 이슈와 진행 중인 이슈 조회하고, #데일리-스탠드업 슬랙 채널에 아래 형식으로 보내줘: ✅ Done: [이슈 목록] 🔄 In Progress: [이슈 목록] 🚧 Blocker: [있으면 명시]"
→ Jira 조회 후 포맷에 맞춰 슬랙 채널에 자동 전송. 매일 아침 반복 실행하면 스탠드업 준비 시간이 0이 됩니다.

7. Confluence 문서 자동 업데이트

// 프롬프트 — 스프린트 회고 문서화
"이번 스프린트(Sprint 24) Jira 완료 이슈 목록을 가져와서 Confluence '스프린트 회고' 페이지에 자동으로 정리해줘. 잘된 점 / 개선점 / 다음 스프린트 목표 형식으로"
→ Jira 이슈 데이터 → Confluence 페이지 자동 업데이트. 스프린트 회고 문서 작성 시간 90% 단축.

보안 & 권한 관리

MCP 서버는 강력한 만큼 보안 관리가 중요합니다. 아래 체크리스트를 반드시 확인하세요.

🔒 MCP 보안 체크리스트

  • Personal Access Token은 절대 코드에 하드코딩 금지 — 환경변수 또는 별도 설정 파일 사용
  • 토큰이 포함된 settings.json은 .gitignore에 추가 — GitHub에 올라가면 즉시 토큰 무효화
  • 파일시스템 MCP 경로는 최소 권한 원칙 — 전체 루트(/) 말고 특정 작업 폴더만 허용
  • GitHub 토큰 권한은 최소한으로 설정 — 읽기만 필요하면 read 권한만 부여
  • 사용하지 않는 서버는 "disabled": true로 비활성화 또는 설정에서 제거
  • 토큰은 3~6개월마다 갱신 — 특히 오래된 프로젝트 토큰 확인
  • 회사·팀 환경에서는 환경변수 또는 Vault 시스템 사용 (AWS Secrets Manager, HashiCorp Vault 등)
  • 원격(SSE) MCP 서버 연결 시 HTTPS 필수 — HTTP 연결은 토큰이 노출될 수 있음

실수하기 쉬운 보안 사고 시나리오

상황위험대처
settings.json을 GitHub에 커밋 토큰 유출, 무단 접근 .gitignore 추가, 토큰 즉시 재발급
파일시스템 경로를 /로 설정 전체 시스템 파일 노출 /workspace, /projects 등 특정 폴더만 지정
만료된 토큰 방치 서비스 중단, 보안 취약점 캘린더 알람으로 정기 갱신 일정 관리

MCP의 한계와 주의사항

MCP는 강력하지만 모든 문제를 해결해주지는 않습니다. 현실적인 한계를 이해하고 적절히 활용하세요.

📏 컨텍스트 한계
너무 큰 파일이나 대용량 DB를 읽으면 컨텍스트 창이 초과됩니다. 큰 파일은 필요한 섹션만 지정해서 읽도록 프롬프트를 구성하세요.
⏱️ 응답 속도
MCP 호출마다 처리 지연이 발생합니다. 특히 원격(SSE) 서버는 네트워크 레이턴시까지 추가됩니다. 복잡한 다중 호출 작업은 시간이 걸릴 수 있습니다.
💰 토큰 비용
도구 호출 결과도 토큰을 소비합니다. 대량의 파일이나 데이터를 읽으면 비용이 빠르게 늘어납니다. 필요한 데이터만 정확히 요청하는 습관이 중요합니다.
🔌 서비스 의존성
외부 서비스(GitHub, Notion 등)에 연결된 MCP는 해당 서비스의 다운타임에 영향을 받습니다. 서비스 장애 시 MCP 도구를 사용할 수 없습니다.
🔏 데이터 프라이버시
MCP로 읽어온 데이터는 Claude의 대화 처리에 사용됩니다. 기밀 정보나 개인정보가 담긴 파일에 접근할 때는 회사 보안 정책을 확인하세요.
🛠️ 서버 버전 호환성
MCP 서버 패키지는 지속적으로 업데이트됩니다. npx -y는 항상 최신 버전을 사용하므로 주요 버전 업그레이드 시 API가 변경될 수 있습니다.
