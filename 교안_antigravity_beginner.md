## 1. 바이브코딩이란?

**바이브코딩(Vibe Coding)**은 AI에게 자연어로 지시하여 소프트웨어를 만드는 새로운 개발 방식입니다. 코드를 직접 작성하지 않고, "이런 것을 만들어 줘"라고 말하면 AI가 코드를 생성합니다.

### 왜 주목받고 있을까?

| 기존 개발 방식 | 바이브코딩 |
|---------------|-----------|
| 프로그래밍 언어 학습 필요 | 자연어(한국어)로 지시 |
| 수백 줄의 코드를 직접 작성 | AI가 코드를 자동 생성 |
| 디버깅에 많은 시간 소요 | AI가 오류를 자동 수정 |
| 개발자만 가능 | 누구나 시작 가능 |

### 바이브코딩 학습 경로

```
AI Studio (입문) → Lovable (웹앱) → AntiGravity (에이전트 IDE) → Cursor (AI IDE) → Claude Code (CLI)
                                          ↑ 지금 여기!
```

> **활용 가이드**: "바이브코딩은 코딩 실력이 없어도 아이디어만 있으면 시작할 수 있습니다. 하지만 AI에게 명확하게 지시하는 능력이 중요합니다." — 활용 가이드

## 2. 안티그래비티란?

**안티그래비티(Google Antigravity)**는 Google이 2025년 11월에 출시한 **에이전트 기반 개발 플랫폼**입니다. VS Code를 기반으로 만들어진 데스크톱 앱으로, AI 에이전트가 자율적으로 코딩·테스트·브라우저 조작까지 수행합니다.

### 러버블과 뭐가 다른가요?

| 항목 | 러버블 (Lovable) | 안티그래비티 (AntiGravity) |
|------|-----------------|--------------------------|
| **유형** | 웹 기반 노코드 빌더 | 데스크톱 에이전트 IDE |
| **설치** | 불필요 (브라우저) | 데스크톱 앱 설치 필요 |
| **작동 방식** | 채팅으로 지시 → 결과 확인 | 에이전트에게 작업 위임 → 자율 실행 |
| **코딩 필요** | 불필요 | 선택적 (이해하면 도움됨) |
| **핵심 강점** | 빠른 프로토타입 | 복잡한 프로젝트, 병렬 작업 |
| **비용** | 유료 (이벤트 시 무료) | 퍼블릭 프리뷰 무료 |

### 안티그래비티 주요 기능

| 기능 | 설명 |
|------|------|
| **Manager View** | 에이전트에게 작업을 지시하고 진행 상황을 모니터링하는 화면 |
| **Editor View** | VS Code처럼 코드를 직접 보고 수정할 수 있는 화면 |
| **브라우저 에이전트** | Chrome을 자동으로 조작하여 웹 테스트를 수행 |
| **Plan Mode** | 작업 전 상세 계획을 먼저 세우는 모드 |
| **Fast Mode** | 계획 없이 바로 실행하는 빠른 모드 |
| **Artifacts** | 에이전트가 생성하는 산출물 (계획서, 스크린샷 등) |

> **활용 가이드**: "안티그래비티는 '내가 코드를 짜는 것'이 아니라 '에이전트에게 일을 시키는 것'입니다. 관리자의 마인드셋이 필요합니다." — 활용 가이드

### 참고: 영어 UI

안티그래비티의 인터페이스는 **영어**입니다. 하지만 AI 에이전트에게 **한국어로 지시**할 수 있습니다. 메뉴나 버튼은 영어이지만, 작업 내용은 한국어로 소통 가능합니다.

## 3. 용어 해설

안티그래비티를 사용하면서 자주 만나는 용어들입니다.

| 용어 | 뜻 | 쉬운 비유 |
|------|-----|-----------|
| **Agent (에이전트)** | AI가 자율적으로 작업을 수행하는 단위 | 일을 시키면 알아서 하는 직원 |
| **Manager View** | 에이전트에게 작업을 지시하는 화면 | 팀장이 업무를 배분하는 회의실 |
| **Editor View** | 코드를 직접 보고 수정하는 화면 | 서류를 직접 확인하는 책상 |
| **Workspace** | 프로젝트를 담는 작업 공간 | 프로젝트별 폴더 |
| **Artifact** | 에이전트가 만든 산출물 (계획서, 스크린샷 등) | 보고서나 결과물 |
| **Plan Mode** | 작업 전 계획을 세우는 모드 | 설계도를 먼저 그리는 방식 |
| **Fast Mode** | 계획 없이 바로 실행하는 모드 | 바로 만들기 시작하는 방식 |
| **Rules** | 에이전트의 행동 규칙을 정하는 설정 | 업무 매뉴얼 |
| **Workflows** | 자주 하는 작업을 자동화하는 레시피 | 반복 업무 템플릿 |
| **MCP** | 외부 서비스와 연결하는 프로토콜 | USB 같은 연결 규격 |
| **Context Window** | AI가 한 번에 읽을 수 있는 정보량 | 책상 위에 올려놓을 수 있는 서류 양 |
| **Terminal** | 명령어를 입력하는 텍스트 창 | 컴퓨터에게 글로 지시하는 창 |

## 5. 아이디어 구상하기

안티그래비티는 웹사이트, 모바일 앱, CLI 도구 등 **다양한 유형의 프로젝트**를 만들 수 있습니다.

### 시작 미션: 뭘 만들까?

아래 3가지 중 하나를 골라보세요:

1. **나만의 웹사이트** — 포트폴리오, 블로그, 랜딩 페이지
2. **간단한 웹앱** — 할 일 관리, 메모장, 대시보드
3. **도구/유틸리티** — 데이터 변환기, 계산기, 자동화 스크립트

### 직군별 추천 프로젝트

| 직군 | 추천 프로젝트 | 난이도 |
|------|-------------|--------|
| 교사 | 학급 관리 대시보드, 성적 분석 도구 | ★★☆ |
| 소상공인 | 예약 관리 시스템, 재고 관리 앱 | ★★☆ |
| 프리랜서 | 프로젝트 관리 도구, 인보이스 생성기 | ★★★ |
| 학생 | 스터디 타이머, 노트 정리 앱 | ★☆☆ |

### 프롬프트 예시

**교사용:**
```
교사용 학급 관리 대시보드를 만들어주세요.
- 학생 목록 관리 (이름, 반, 번호)
- 출석 체크 기능
- 간단한 성적 입력 및 평균 계산
- 반응형 웹으로 모바일에서도 사용 가능하게
```

**소상공인용:**
```
카페 예약 관리 시스템을 만들어주세요.
- 날짜/시간별 예약 캘린더
- 고객 정보 (이름, 연락처) 입력
- 예약 상태 (대기/확정/취소) 관리

**학생용:**
```
스터디 타이머 웹앱을 만들어주세요.

## 6. 프롬프트 작성하기

### PRD(요구정의서)란?

PRD(Product Requirements Document)는 "무엇을 만들지" 정리한 기획 문서입니다. AI 에이전트에게 명확한 지시를 하기 위해 PRD를 먼저 작성하면 결과물의 품질이 올라갑니다.

### 간단한 PRD 예시

```
# 스터디 타이머 PRD

## 목적
학생들이 집중력을 높이기 위한 포모도로 타이머 웹앱

## 주요 기능
1. 25분/5분 타이머 (포모도로)
2. 시작, 일시정지, 리셋 버튼
3. 완료 세션 카운터
4. 알림음 (타이머 종료 시)

## 디자인
- 미니멀, 다크 모드 기본
- 모바일 반응형

## 기술 제약
- 순수 HTML/CSS/JavaScript (프레임워크 없이)
- 서버 없이 클라이언트만으로 동작
```

### Plan Mode 활용하기

안티그래비티에는 **Plan Mode**라는 기능이 있습니다. 에이전트에게 작업을 요청하면, 바로 실행하지 않고 먼저 상세한 **실행 계획(Plan)**을 세웁니다.

1. PRD를 에이전트에게 전달
2. 에이전트가 Plan(계획)을 Artifact로 생성
3. 계획을 검토하고 수정 요청 가능
4. 승인하면 실행 시작

> **활용 가이드**: "Plan Mode를 사용하면 에이전트가 엉뚱한 방향으로 가는 것을 사전에 막을 수 있습니다. 특히 처음에는 Plan Mode를 권장합니다." — 활용 가이드

## 7. 안티그래비티 실습: 나만의 프로젝트 만들기

### Step 1: 워크스페이스 만들기

1. 안티그래비티 앱 실행
2. "New Workspace" 클릭
3. 프로젝트 이름 입력 (예: `study-timer`)
4. 빈 워크스페이스 또는 기존 폴더 선택

### Step 2: Manager View에서 작업 지시하기

1. Manager View 탭으로 이동
2. 채팅 입력창에 PRD 또는 프롬프트 붙여넣기
3. **Plan Mode** 선택 (처음에는 Plan Mode 추천)
4. "Send" 버튼 클릭

### Step 3: Plan(계획) 검토하기

1. 에이전트가 **Plan Artifact**를 생성합니다
2. 계획서에는 다음이 포함됩니다:
   - 만들 파일 목록
   - 사용할 기술 스택
   - 구현 순서
3. 계획이 마음에 들면 **승인**, 수정이 필요하면 **피드백** 입력

### Step 4: 에이전트 실행 관찰하기

1. 승인 후 에이전트가 자동으로 작업 시작
2. **Artifacts** 탭에서 진행 상황 확인:
   - 생성된 파일 목록
   - 터미널 명령어 실행 로그
   - 스크린샷 (브라우저 에이전트 사용 시)
3. 에이전트가 작업 완료되면 결과 보고

### Step 5: 결과 확인 및 수정

1. **Editor View** 탭에서 생성된 코드 확인
2. 브라우저에서 결과물 미리보기 (터미널에서 `npm run dev` 등)
3. 수정이 필요하면 Manager View로 돌아가 추가 지시:

```
타이머 디자인을 좀 더 둥글게 만들어줘.
버튼 색상을 파란색 계열로 변경하고,
타이머 숫자 폰트를 더 크게 해줘.
```

> **활용 가이드**: "에이전트가 만든 결과를 바로 수정하려 하지 말고, 먼저 Manager View에서 수정 지시를 해보세요. 직접 코드를 수정하면 에이전트와 충돌할 수 있습니다." — 활용 가이드

## 8. 프로젝트 공개하기 (배포)

### 방법 1: GitHub + Vercel (추천)

1. 안티그래비티 터미널에서 Git 초기화:
   ```
   git init
   git add .
   git commit -m "첫 번째 커밋"
   ```
2. GitHub에 저장소 생성 후 push
3. [Vercel](https://vercel.com)에서 GitHub 저장소 연결 → 자동 배포

### 방법 2: Firebase Hosting

1. 에이전트에게 배포 작업 지시:
   ```
   이 프로젝트를 Firebase Hosting으로 배포해줘.
   firebase init과 firebase deploy를 실행해줘.
   ```
2. 에이전트가 Firebase 설정 및 배포를 자동 수행

### 방법 3: 에이전트에게 배포 맡기기

```
이 프로젝트를 인터넷에 공개하고 싶어.
GitHub에 올리고 Vercel로 배포해줘.
저장소 이름은 'study-timer'로 해줘.
```

에이전트가 Git, GitHub, Vercel 설정을 모두 자동으로 처리합니다.

## 10. 문제가 생겼을 때 (트러블슈팅)

### 상황 1: 설치가 안 돼요

| 문제 | 해결 방법 |
|------|----------|
| 다운로드 오류 | 공식 사이트(antigravity.google)에서 다시 다운로드 |
| 설치 중 권한 오류 (Windows) | 설치 파일 우클릭 → "관리자 권한으로 실행" |
| 설치 중 권한 오류 (Mac) | 시스템 설정 → 보안 및 개인정보 → "확인 없이 열기" |
| 실행 시 빈 화면 | 앱 재시작 또는 재설치 |

### 상황 2: 에이전트가 응답하지 않아요

1. 인터넷 연결 확인
2. 속도 제한(Rate Limit)에 걸렸는지 확인 → 5시간마다 리셋
3. 다른 AI 모델로 전환 시도 (Gemini → Claude 등)
4. 앱 재시작

### 상황 3: 브라우저 에이전트가 작동하지 않아요

1. Chrome 확장 프로그램이 설치되어 있는지 확인
2. Chrome이 실행 중인지 확인
3. 안티그래비티 앱에서 브라우저 에이전트 권한이 허용되어 있는지 확인

### 상황 4: 빌드 에러가 발생해요

1. 에이전트에게 에러 내용을 그대로 전달:
   ```
   이런 에러가 발생했어: [에러 메시지 붙여넣기]
   수정해줘.
   ```
2. 대부분의 빌드 에러는 에이전트가 자동으로 수정 가능

### 상황 5: 에이전트가 의도와 다른 결과를 만들어요

1. Plan Mode를 사용하여 계획부터 검토
2. 프롬프트를 더 구체적으로 작성
3. 한 번에 많은 것을 요청하지 말고 단계별로 지시

## 11. 자주 묻는 질문 (FAQ)

### Q1. 안티그래비티는 정말 무료인가요?

현재 퍼블릭 프리뷰 기간으로 **완전 무료**입니다. 신용카드도 필요 없습니다. 다만 5시간마다 속도 제한이 리셋되는 방식으로 사용량이 조절됩니다.

### Q2. 커서(Cursor)와 뭐가 다른가요?

| 항목 | 안티그래비티 | 커서 |
|------|-------------|------|
| 핵심 | 에이전트 위임 | 코파일럿 보조 |
| 작업 방식 | 에이전트가 자율 실행 | 개발자가 코드 제어 |
| 멀티 에이전트 | 지원 (병렬 작업) | 미지원 |
| 브라우저 제어 | 내장 | 미지원 |
| 비용 | 무료 (프리뷰) | 유료 ($20/월) |

### Q3. 한국어로 사용할 수 있나요?

앱 인터페이스(메뉴, 버튼)는 영어이지만, **에이전트에게 한국어로 지시**할 수 있습니다. 한국어 프롬프트도 잘 이해합니다.

### Q4. 어떤 프로그래밍 언어를 지원하나요?

JavaScript, Python, TypeScript, Flutter/Dart, React, Vue, Next.js 등 거의 모든 언어와 프레임워크를 지원합니다.

### Q5. 코딩을 전혀 몰라도 사용할 수 있나요?

가능하지만, 러버블보다는 약간의 기술 이해가 필요합니다. 터미널, 파일 구조 등 기본 개념을 알면 더 효과적으로 사용할 수 있습니다. 완전 입문자라면 **러버블**부터 시작하는 것을 추천합니다.

### Q6. 모바일 앱도 만들 수 있나요?

네! Flutter/Dart를 사용하여 iOS/Android 앱을 만들 수 있습니다. React Native도 지원합니다.

### Q7. 인터넷 없이도 사용할 수 있나요?

AI 에이전트 기능은 인터넷 연결이 필요합니다. 하지만 Editor View에서 코드를 직접 수정하는 것은 오프라인에서도 가능합니다.

### Q8. 만든 프로젝트의 코드 소유권은 누구에게 있나요?

안티그래비티로 생성한 코드의 소유권은 **사용자**에게 있습니다.

## 12. 주의사항 및 팁

### 보안 정책 이해하기

안티그래비티 에이전트는 터미널 명령어를 실행하고 브라우저를 조작할 수 있습니다. 보안 정책 설정을 확인하세요:

| 모드 | 설명 | 추천 대상 |
|------|------|----------|
| **Safe (기본)** | 매 작업마다 사용자 승인 요청 | 초보자 (이 교안 대상) |
| **Auto** | 안전한 작업은 자동 승인 | 중급자 |
| **Turbo** | 대부분 자동 승인 | 개발자 |

### 자주 하는 실수

| 실수 | 올바른 방법 |
|------|-----------|
| 한 번에 너무 많이 요청 | 작은 단위로 나눠서 요청 |
| 에이전트 결과를 직접 수정 | Manager View에서 수정 지시 |
| Fast Mode만 사용 | 중요한 작업에는 Plan Mode 사용 |
| 에러 메시지 무시 | 에이전트에게 에러 내용 전달 |

### 실전 노하우

1. **Plan Mode로 시작하세요**: 처음에는 항상 Plan Mode를 사용하여 에이전트의 계획을 먼저 확인
2. **Artifacts를 확인하세요**: 에이전트가 생성하는 산출물(계획서, 스크린샷)을 통해 진행 상황 파악
3. **단계별로 진행하세요**: 한 번에 완성하려 하지 말고, 기본 → 기능 추가 → 디자인 순으로
4. **저장을 자주 하세요**: Git으로 중간중간 커밋하여 되돌릴 수 있게
