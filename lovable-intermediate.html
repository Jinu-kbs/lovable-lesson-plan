어떤 도구를 선택할까?

  • 러버블: 빠르게 프로토타입을 만들고 싶을 때, 설치 없이 시작하고 싶을 때
  • 안티그래비티: AI 에이전트에게 작업을 맡기고 싶을 때, 복잡한 프로젝트를 자동화하고 싶을 때
  • 커서: 코드를 직접 제어하고 싶을 때, 복잡한 프로젝트를 진행할 때
  • AI Studio 빌더: 바이브코딩 첫 경험으로, IDE 개념을 배우기 전 단계로

2. 효율적인 워크플로우

러버블에 바로 프롬프트를 던지면 크레딧을 낭비하기 쉽습니다. 아래 워크플로우를 따라주세요.

5단계 워크플로우

① 아이디어 구상
      ↓
② AI로 기획 문서 작성 (PRD / IA / ERD)
      ↓
③ 러버블에서 제작 (Chat 모드 활성화)
      ↓
④ 배포 (러버블 Publish 또는 GitHub + Vercel)
      ↓
⑤ SNS 공유 (이벤트 미션)

왜 이 순서인가?

단계하는 일도구비용
① 아이디어뭘 만들지 구상메모장무료
② 기획 문서PRD, IA, ERD 작성ChatGPT, 제미나이, 클로드무료~저렴
③ 제작기획 기반 빌드Lovable크레딧 소모
④ 배포웹에 공개Lovable Publish 또는 Vercel무료
⑤ 공유SNS 공유인스타, X 등무료
핵심 포인트: ②번에서 충분히 정리해야 ③번에서 크레딧을 아낄 수 있습니다. 다른 AI에서 미리 정리해두면 러버블에서 불필요하게 왔다 갔다 안 해도 됩니다.

4. 기획 문서 작성 실습

러버블에 들어가기 전, AI를 활용해 3가지 핵심 문서를 작성합니다.

핵심 인사이트: "구조화 잘된 PRD = 최고의 프롬프트" — 실사용자 인사이트. PRD를 잘 작성하면 그 자체가 러버블에 넣을 최고의 프롬프트가 됩니다.

4-1. PRD (Product Requirements Document, 요구정의서)

가장 중요한 문서입니다. 무엇을 만들 것인지 명확하게 정의합니다.

AI에게 PRD 작성 요청하는 프롬프트:

나는 [직업/역할]이고, [목적]을 위한 웹앱을 만들고 싶습니다.

다음 내용을 포함한 PRD(요구정의서)를 작성해주세요:

1. 프로젝트 개요 (이름, 목적, 핵심 가치)
2. 대상 사용자 및 사용자 시나리오
3. 핵심 기능 목록 (우선순위별)
4. 페이지 구성 및 화면 흐름
5. 비기능 요구사항 (반응형, 성능, 접근성)
6. 디자인 가이드라인 (색상, 톤앤매너)
7. 제약 조건 및 가정

기능 설명은 구체적으로, 각 기능별로 사용자 스토리 형식으로 작성해주세요.

4-2. IA (Information Architecture, 정보 구조)

페이지 간의 구조와 관계를 정리합니다. 사이트 전체의 "지도"라고 생각하면 됩니다.

AI에게 IA 작성 요청하는 프롬프트:

위 PRD를 기반으로 IA(정보 구조도)를 작성해주세요.

포함할 내용:
1. 전체 사이트맵 (트리 구조)
2. 각 페이지의 주요 컴포넌트
3. 페이지 간 이동 흐름
4. 네비게이션 구조

마크다운 형식으로 작성해주세요.

IA 예시 (프리랜서 포트폴리오):

홈 (/)
├── 소개 (/about)
├── 포트폴리오 (/portfolio)
│   └── 프로젝트 상세 (/portfolio/:id)
├── 서비스 (/services)
└── 연락처 (/contact)

IA 예시 (학원/강좌 안내):

홈 (/)
├── 강좌 안내 (/courses)
│   └── 강좌 상세 (/courses/:id)
├── 강사 소개 (/instructors)
├── 수강 후기 (/reviews)
├── 공지사항 (/notices)
└── 문의하기 (/contact)

4-3. ERD (Entity Relationship Diagram, 데이터 구조)

데이터베이스가 필요한 프로젝트라면 ERD도 준비합니다. 소개 사이트처럼 DB가 필요 없는 경우에는 건너뛰어도 됩니다.

AI에게 ERD 작성 요청하는 프롬프트:

위 PRD를 기반으로 ERD(데이터 구조)를 작성해주세요.

포함할 내용:
1. 필요한 데이터 테이블 목록
2. 각 테이블의 항목 (이름, 종류, 설명)
3. 테이블 간 관계

마크다운 테이블 형식으로 쉽게 작성해주세요.

PRD 작성 시 직군별 포인트

직군PRD에 꼭 포함할 내용
교사학부모 접근성, 모바일 최적화, 개인정보 주의
강사/튜터수강 신청 흐름, 일정 관리, 결제 연동 여부
프리랜서작업물 쇼케이스, 문의 양식, SEO 키워드

동호회회원 관리, 일정 공유, 사진 갤러리

5. 러버블 심화 활용

5-1. Chat 모드 활용 전략

러버블의 Chat 버튼을 활성화하면 플랜 모드처럼 동작합니다.

모드동작 방식크레딧결과 품질
일반 모드바로 코드 생성적음보통
Chat 모드먼저 분석 → 보완 → 코드 생성약 1크레딧 추가높음

Chat 모드 활용 팁:

  1. 첫 프롬프트는 길고 구체적으로: PRD 전체를 붙여넣기
  2. 수정은 짧고 명확하게: "헤더 색상을 파란색 계열로 변경해주세요"
  3. 한 번에 하나씩: 여러 수정을 한꺼번에 요청하지 말고 나눠서
  4. 참고 이미지 활용: 원하는 디자인 참고 사이트가 있으면 설명에 포함

5-2. 프롬프트 작성 노하우 (AI 경험자용)

AI 도구 경험이 있는 분이라면 아래 테크닉을 활용해보세요:

역할 부여:

너는 한국 시장을 이해하는 시니어 웹 개발자야.
아래 PRD를 기반으로 한국 사용자에게 최적화된 웹사이트를 만들어줘.

단계별 지시:

Step 1: 먼저 전체 페이지 레이아웃을 잡아줘.
Step 2: 각 페이지에 더미 데이터를 넣어서 실제 느낌이 나게 해줘.
Step 3: 모바일에서도 잘 보이게 반응형으로 조정해줘.

부정 프롬프트 (하지 말아야 할 것):

- 영어로 된 텍스트는 사용하지 마세요 (Lorem ipsum 등)
- 너무 화려한 애니메이션은 넣지 마세요
- 팝업이나 모달은 사용하지 마세요

5-3. Visual Edits (디자인 모드)

러버블에는 채팅 외에도 Visual Edits 기능이 있어서 디자인을 직접 수정할 수 있습니다:

  • 색상, 텍스트, 레이아웃 등을 화면에서 직접 클릭하여 변경
  • 채팅 없이 빠르게 디자인 조정 가능
  • 전체적인 색상 톤이 마음에 안 들 때 유용
실전 팁: "디자인에서 바로 색상 변경되는 거 아시죠? 너무 특정 색이 많으면 바꿔보세요" — 실사용자 팁

5-4. 안정성 체크

프로젝트가 어느 정도 완성되면 안정성 체크 기능을 활용하세요:

  • 마지막에 "안정성 체크"를 실행하면 잠재적 문제를 진단해줍니다
  • 배포 전에 반드시 한 번 확인하는 것을 추천합니다

5-5. 프로젝트 파일 관리

  • 다운로드: 마음에 드는 버전은 바로 소스 코드 다운로드
  • 이어서 작업: 다운로드한 파일은 다른 도구(커서, VS Code, 안티그래비티 등)에서 이어서 작업 가능
  • 버전 관리: 주요 변경 전후로 다운로드해두면 롤백이 가능
실전 팁: "작업하시던 파일 다운받아서 다른 데서(안티그래비티) 이어하세요. 코드 수정이 조금 필요할 수도 있지만 가능합니다" — 실사용자 조언

5-6. UI/UX 흐름 고려하기

프로젝트가 기능적으로 잘 동작해도, 사용자 흐름(UX)을 잘 설계하지 않으면 결과물의 완성도가 떨어집니다.

  • 사용자 시나리오 먼저 그리기: 사용자가 사이트에 들어와서 어떤 순서로 행동하는지 정리
  • 참고 디자인 수집: 다른 앱이나 사이트에서 마음에 드는 디자인을 캡처해두기
  • 페이지 흐름 요청: 러버블에 "사용자가 홈 → 강좌 목록 → 강좌 상세 → 수강 신청 순서로 이동합니다"처럼 구체적인 흐름을 제시
실전 팁: "사용자 흐름을 잘 고민해주는 것도 좋아요, UI/UX" — 실사용자 조언

Step 1: PRD 준비

# PRD: 코딩 클래스 안내 사이트

## 프로젝트 개요
바이브코딩 강좌를 안내하고 수강 신청을 받는 사이트

## 핵심 기능
1. 강좌 목록 보기 (카드 그리드 레이아웃)
2. 강좌별 상세 정보 (커리큘럼, 일정, 강사 소개)
3. 수강 후기 섹션

7. 웹사이트 공개하기 (배포)

방법 1: 러버블 자체 배포 (Publish)

가장 빠른 방법입니다:

  1. Publish 버튼 클릭
  2. 자동 생성된 URL로 바로 접속 가능
  3. 수정하면 다시 Publish하여 업데이트

방법 2: GitHub + Vercel 배포 (추천)

더 안정적이고 나만의 URL을 가질 수 있습니다.

GitHub 연동:

  1. 프로젝트 설정에서 GitHub 연동 선택
  2. 저장소(Repository) 생성 또는 기존 저장소 선택
  3. Push 클릭하면 코드가 자동으로 GitHub에 업로드

Vercel 배포:

  1. Vercel 대시보드 → Add New Project
  2. GitHub 저장소 Import
  3. Framework 자동 감지 → Deploy 클릭
  4. 약 1~2분 후 배포 완료

자동 배포의 장점:

Vercel은 GitHub과 연동되면 코드 변경 시 자동 재배포됩니다:

러버블에서 수정 → GitHub Push → Vercel 자동 재배포 → 사이트 업데이트

10. 문제가 생겼을 때 (트러블슈팅)

"Chat 모드에서 크레딧이 너무 빨리 줄어요"

  • 첫 프롬프트에 PRD를 충분히 상세하게 작성하세요
  • 수정 요청은 한 번에 하나씩만
  • 이벤트 기간에는 무제한이니 마음껏 실험하세요

"러버블이 의도와 다른 결과를 만들었어요"

  1. 되돌리기(Undo) 기능으로 이전 상태로 복원
  2. 더 구체적으로 재요청 (색상, 위치, 크기 등 명시)
  3. "현재 상태를 유지하면서 [X] 부분만 변경해줘"로 범위 제한

"코드가 깨져서 아무것도 안 보여요"

  1. 당황하지 말고 되돌리기(Undo) 클릭
  2. 여러 번 되돌려도 안 되면 마지막으로 다운로드한 파일에서 재시작
  3. 새 프로젝트를 만들어서 PRD를 다시 투입하는 것도 방법

"DB 연결 후 에러가 발생해요"

  • 처음부터 DB를 연결하지 말고, UI(화면)를 먼저 완성하세요
  • UI가 마음에 든 후에 DB 연결을 시도하세요
  • Supabase 환경변수(URL, API키)가 정확한지 확인하세요

"프로젝트가 커지니까 수정이 점점 어려워요"

이는 매우 흔한 문제입니다. 러버블 같은 도구로 만들고 나중에 구조를 이해하지 못하면 수정이 어렵고, 크레딧도 많이 소모됩니다.

  • 예방법: PRD를 충분히 상세하게 작성하여 처음부터 구조를 잡기
  • 대처법: 프로젝트 파일을 다운로드하여 직접 코드 구조를 살펴보기
  • 장기 전략: 개발자편 교안을 참고하여 코드 구조를 이해하고, 커서/VS Code에서 직접 수정하는 방법 학습
실사용자 경고: "러버블 같은 걸로 사이트 만들고 나중에 구조를 이해 못하면 나중에 더 힘들어집니다... 커지면 수정하는 데 크레딧도 쭉쭉 나가요" — 실사용자 경고

"카카오톡에 공유했는데 썸네일이 안 나와요"

  • 카카오톡은 한번 읽어간 썸네일(OG 이미지)을 캐시합니다
  • 사이트를 수정해도 카톡 미리보기가 바로 바뀌지 않습니다
  • 해결 방법: 카카오 OG 캐시 초기화 페이지에서 URL을 입력하고 "캐시 초기화" 클릭
  • OG 이미지를 직접 만들고 싶다면, 러버블에 "OG 메타 태그와 1200x630 썸네일 이미지를 추가해줘"라고 요청하세요

11. 자주 묻는 질문 (FAQ)

Q: 러버블에 자체 DB가 있나요?

아니요. 러버블 자체에는 데이터베이스가 없습니다. 정적 소개 사이트는 DB 없이 가능하지만, 데이터를 저장·관리하려면 Supabase 같은 외부 서비스를 연결해야 합니다.

Q: 러버블에서 만든 사이트를 다른 도구에서 이어서 작업할 수 있나요?

네! 프로젝트 파일을 다운로드하면 VS Code, Cursor, 클로드 코드 등에서 이어서 수정할 수 있습니다. GitHub에 올려놓으면 어디서든 접근 가능합니다.

Q: 러버블과 안티그래비티, 뭐가 더 좋나요?

정답은 없습니다. 러버블은 웹(React) 기반으로 빠른 프로토타입에 강하고, 안티그래비티는 Flutter 기반으로 앱 제작에 강합니다. 오늘 이벤트로 러버블을 체험해보고, 취향에 맞는 도구를 선택하세요.

Q: 한 번 만든 프로젝트를 팀원과 공유할 수 있나요?

GitHub에 올려두면 팀원들과 함께 수정할 수 있습니다. Vercel에 배포하면 URL로 결과물도 바로 공유 가능합니다.

Q: 이벤트가 끝나면 만든 사이트는 어떻게 되나요?
  • 러버블 자체 배포(Publish): 러버블 계정이 유지되는 한 접속 가능
  • GitHub + Vercel 배포: 이벤트와 무관하게 계속 유지 (무료)
  • 파일 다운로드해두면 어디서든 다시 사용 가능
Q: 러버블 크레딧과 $100 API 크레딧은 같은 건가요?

아닙니다. 러버블 크레딧은 러버블에서 채팅하며 사이트를 만들 때 사용되는 것이고(이벤트 중 무제한), $100 API 크레딧은 외부에서 Claude API를 연동할 때 쓰는 별도 크레딧입니다. 러버블에서 사이트 만드는 것만으로는 API 크레딧이 소모되지 않습니다.

Q: 크레딧이 너무 빨리 줄어드는데 어떻게 해야 하나요?

크레딧은 "뻑하면 끝난다"는 경험담이 많습니다. 핵심은 러버블에 들어가기 전에 기획을 충분히 하는 것입니다. 다른 무료 AI(제미나이, ChatGPT)에서 PRD를 완성한 후 러버블에 한 번에 투입하면 크레딧을 크게 절약할 수 있습니다. 이벤트 기간(25시간 무료)을 최대한 활용하세요.

Q: AI Studio 빌더와 러버블, 어떤 순서로 배우는 게 좋을까요?

바이브코딩 자체가 처음이라면 AI Studio 빌더에서 "아~ 이런 거구나" 느낌을 먼저 잡고, 그 다음에 러버블로 넘어오면 훨씬 수월합니다. 하지만 AI 도구 사용에 이미 익숙하다면 바로 러버블을 시작해도 좋습니다.

증강(Augmentation) 경험의 가치

바이브코딩에서 가장 중요한 것은 AI를 통한 증강 경험입니다:

  • AI가 만든 결과물을 직접 확인하고 수정 방향을 잡는 과정 자체가 학습입니다
  • 증강(AI의 도움을 받아 내 역량을 확장하는 것)이 익숙해지면, 자동화(반복 작업을 AI에게 맡기는 것)도 자연스럽게 됩니다
  • "하라고 하면 그대로 하면 됩니다. 자꾸 창의력 발휘하면 안 됩니다" — 지시를 정확히 따르는 것이 바이브코딩의 핵심입니다
