## 1. 바이브코딩이란?

### 학습 목표

- 바이브코딩의 개념을 설명할 수 있다
- 전통적인 개발 방식과 바이브코딩의 차이를 이해한다
- 주요 바이브코딩 도구 7가지를 비교할 수 있다
- 나에게 맞는 학습 경로를 선택할 수 있다

### 상세 설명

**바이브코딩(Vibe Coding)**은 코딩을 직접 하지 않고, AI에게 일상 언어로 원하는 것을 설명하면 AI가 코드를 작성해주는 새로운 개발 방식입니다.

- 프로그래밍 언어를 몰라도 됩니다
- "이런 웹사이트를 만들어줘"라고 말하면 AI가 만들어줍니다
- 마치 전문가에게 의뢰하듯이 대화로 진행합니다

### 전통 개발 vs 바이브코딩

| 항목 | 전통 개발 방식 | 바이브코딩 |
|------|---------------|-----------|
| 사전 지식 | 프로그래밍 언어 학습 필수 | 자연어(한국어)로 지시 |
| 코드 작성 | 수백~수천 줄 직접 작성 | AI가 코드를 자동 생성 |
| 디버깅 | 오류를 직접 찾고 수정 | AI가 오류를 자동 수정 |
| 시작 장벽 | 수개월의 학습 필요 | 아이디어만 있으면 바로 시작 |
| 결과물 속도 | 수주~수개월 | 수분~수시간 |
| 대상 | 전문 개발자 | 누구나 가능 |

쉽게 말해, 전통 개발이 "외국어를 배워서 직접 대화하는 것"이라면, 바이브코딩은 "통역사(AI)에게 한국어로 말하면 대신 소통해주는 것"과 같습니다.

### 왜 지금 주목받고 있을까?

- 2024~2026년 AI 코딩 도구가 급격히 발전했습니다
- 아이디어만 있으면 누구나 웹사이트/앱을 만들 수 있는 시대가 열렸습니다
- 교육, 소상공인, 프리랜서, 동호회 등 다양한 분야에서 활용 가능합니다
- ChatGPT는 전 세계에서 가장 많이 사용되는 AI 도구이므로 진입 장벽이 낮습니다

## 2. ChatGPT 코딩 기능 소개

### 학습 목표

- ChatGPT의 코딩 관련 핵심 기능을 이해한다
- GPT-4o와 GPT-4.5 모델의 차이를 안다
- Canvas와 Codex의 역할을 구분할 수 있다
- 무료 플랜과 유료 플랜의 차이를 파악한다

### 상세 설명

#### ChatGPT란?

**ChatGPT**는 OpenAI가 만든 대화형 AI 서비스입니다. 텍스트, 이미지, 코드 등 다양한 작업을 대화 형식으로 처리할 수 있으며, 전 세계에서 가장 널리 사용되는 AI 도구입니다.

코딩 관점에서 ChatGPT의 강점은:

- **이미 익숙합니다**: 대부분의 사람이 ChatGPT를 사용해본 경험이 있습니다
- **별도 설치가 필요 없습니다**: 웹 브라우저에서 바로 사용 가능합니다
- **한국어를 잘 이해합니다**: 한국어로 코딩 요청을 해도 정확히 이해합니다
- **범용적입니다**: 웹, 앱, 데이터 분석, 자동화 등 모든 분야의 코드를 작성합니다

#### GPT 모델 비교

| 항목 | GPT-4o | GPT-4.5 | o3 |
|------|--------|---------|-----|
| **속도** | 매우 빠름 | 보통 | 느림 (추론 중) |
| **코딩 능력** | 우수 | 매우 우수 | 최상급 |
| **멀티모달** | 텍스트+이미지+음성 | 텍스트+이미지 | 텍스트+이미지 |
| **추론 능력** | 우수 | 매우 우수 | 최상급 (심층 추론) |
| **무료 사용** | 가능 (제한적) | 불가 (Plus 이상) | 불가 (Plus 이상) |
| **추천 용도** | 일반 코딩, 빠른 답변 | 복잡한 코딩, 대규모 프로젝트 | 어려운 로직, 알고리즘 |

> **초보자 추천**: GPT-4o로 시작하세요. 무료 플랜에서도 사용 가능하고, 대부분의 코딩 작업에 충분합니다.

#### ChatGPT의 코딩 관련 핵심 기능

| 기능 | 설명 | 비유 |
|------|------|------|
| **대화형 코드 생성** | 채팅으로 코드를 요청하면 즉시 생성 | "이런 것 만들어줘"라고 말하면 만들어주는 전문가 |
| **Canvas** | 코드를 시각적으로 편집하고 수정하는 별도 화면 | 코드 전용 작업 공간 (칠판처럼 같이 보며 수정) |
| **Codex** | 자율적으로 코딩 작업을 수행하는 에이전트 | 혼자 알아서 일하는 주니어 개발자 |
| **Data Analysis** | 파일 업로드 후 데이터를 분석하고 시각화 | 엑셀 전문가에게 "이거 분석해줘" 하는 것 |
| **이미지 인식** | 스크린샷을 보여주고 "이런 디자인으로 만들어줘" 가능 | 사진을 보여주고 "이렇게 해줘" 하는 것 |

#### Canvas(코드 편집 모드) 상세

Canvas는 ChatGPT 안에 내장된 코드 편집 도구입니다. 일반 채팅과 다른 점은:

- 코드가 **별도 패널**에 표시되어 더 넓게 볼 수 있습니다
- 코드의 **특정 부분을 선택**하여 수정 요청할 수 있습니다
- **실시간으로 코드를 편집**할 수 있습니다 (직접 타이핑 가능)
- **코드 리뷰, 버그 수정, 주석 추가** 등을 한 번 클릭으로 요청할 수 있습니다
- 완성된 코드를 **복사/다운로드**하여 바로 사용할 수 있습니다

#### Codex(자율 코딩 에이전트) 상세

Codex는 ChatGPT Plus/Pro 사용자를 위한 자율 코딩 에이전트입니다:

- **작업을 지시하면 알아서 실행합니다**: "이런 앱을 만들어줘"라고 하면 처음부터 끝까지 자율적으로 코드를 작성합니다
- **GitHub 연동**: 작성한 코드를 GitHub 저장소에 자동으로 저장합니다
- **샌드박스 환경**: 안전한 격리 환경에서 코드를 실행하고 테스트합니다
- **비동기 실행**: 작업이 완료될 때까지 기다리는 동안 다른 작업을 할 수 있습니다

#### 무료 vs Plus vs Pro 비교

| 항목 | 무료 (Free) | Plus ($20/월) | Pro ($200/월) |
|------|-------------|---------------|---------------|
| **GPT-4o** | 제한적 사용 | 충분한 사용량 | 무제한 |
| **GPT-4.5** | 불가 | 가능 | 무제한 |
| **o3** | 불가 | 가능 | 무제한 |
| **Canvas** | 가능 (제한적) | 전체 기능 | 전체 기능 |
| **Codex** | 불가 | 가능 | 전체 기능 |
| **Data Analysis** | 제한적 | 전체 기능 | 전체 기능 |
| **이미지 생성** | 제한적 | 가능 | 무제한 |
| **파일 업로드** | 가능 | 가능 | 가능 |
| **추천 대상** | 가볍게 시작 | 본격적인 코딩 활용 | 전문적 업무 |

> **초보자 팁**: 무료 플랜으로도 충분히 시작할 수 있습니다. Canvas와 기본 코드 생성 기능을 체험한 후, 필요하다면 Plus로 업그레이드하세요.

## 3. 용어 해설

### 학습 목표

- 바이브코딩과 ChatGPT 관련 핵심 용어 10가지를 이해한다
- 각 용어를 쉬운 비유로 설명할 수 있다

### 상세 설명

교안에서 자주 나오는 용어들을 미리 알아두세요. 외울 필요는 없습니다. 나중에 다시 찾아보면 됩니다.

| 용어 | 한 줄 정의 | 쉬운 비유 |
|------|-----------|-----------|
| **프롬프트 (Prompt)** | AI에게 보내는 요청 메시지 | 식당에서 주문하는 메뉴판 같은 것 — 명확하게 주문할수록 원하는 요리가 나옵니다 |
| **토큰 (Token)** | AI가 텍스트를 처리하는 최소 단위 (한글 1글자 ≈ 2~3토큰) | 글자를 세는 단위 — 사용량을 측정하는 기준이 됩니다 |
| **컨텍스트 (Context)** | AI가 한 번에 기억할 수 있는 대화 범위 | 화이트보드 크기 — 너무 많이 쓰면 앞부분이 지워집니다 |
| **Canvas** | ChatGPT 내장 코드 편집 도구 (코드를 보며 수정하는 별도 화면) | 선생님과 함께 칠판에서 코드를 보며 고치는 작업 공간 |
| **Codex** | OpenAI의 자율 코딩 에이전트 (알아서 코드 작성) | 지시만 하면 알아서 일하는 주니어 개발자 |
| **API (Application Programming Interface)** | 프로그램들이 서로 소통하는 규약 | 식당 주방과 손님을 연결하는 웨이터 — 요청을 전달하고 결과를 가져옵니다 |
| **GPT (Generative Pre-trained Transformer)** | OpenAI가 만든 대규모 언어 모델 | ChatGPT의 "두뇌" — 사전에 방대한 지식을 학습한 AI 엔진 |
| **모델 (Model)** | AI의 두뇌 역할을 하는 핵심 프로그램 (GPT-4o, o3 등) | 자동차의 엔진 — 같은 차체라도 엔진이 다르면 성능이 다릅니다 |
| **플러그인/GPTs** | ChatGPT에 추가 기능을 붙이는 확장 도구 | 스마트폰의 앱 — 필요한 기능을 골라서 설치합니다 |
| **아티팩트 (Artifact)** | AI가 생성한 결과물 (코드, 문서, 이미지 등) | AI가 만들어 준 작업 결과물 (보고서, 코드 파일 등) |

### 추가 알아두면 좋은 용어

| 용어 | 설명 |
|------|------|

| **CSS** | 웹 페이지의 디자인을 꾸미는 언어 (옷 역할) |
| **JavaScript** | 웹 페이지에 동작을 추가하는 언어 (근육 역할) |
| **Python** | 범용 프로그래밍 언어 (데이터 분석, 자동화에 많이 사용) |
| **IDE** | 코드를 작성하는 전문 도구 (Integrated Development Environment) |
| **디버깅 (Debugging)** | 코드의 오류를 찾아서 수정하는 과정 |

## 5. 프롬프트 작성법

### 학습 목표

- 좋은 프롬프트의 5가지 원칙을 이해한다
- 코딩 전용 프롬프트 템플릿 5가지를 활용할 수 있다
- GPTs(커스텀 GPT)가 무엇인지 안다

### 상세 설명

ChatGPT로 코드를 잘 만들어 받으려면 **프롬프트 작성법**이 가장 중요합니다. 프롬프트는 "AI에게 보내는 주문서"입니다. 주문서를 명확하게 작성할수록 원하는 결과를 받을 수 있습니다.

#### 좋은 프롬프트 5원칙

| 원칙 | 설명 | 나쁜 예 | 좋은 예 |
|------|------|---------|---------|
| **1. 구체적으로** | 원하는 것을 세부적으로 설명 | "웹사이트 만들어줘" | "학교 동아리 소개 웹페이지를 만들어줘. 동아리 이름, 활동 사진 3장, 가입 신청 버튼이 있어야 해" |
| **2. 역할 부여** | AI에게 전문가 역할을 부여 | "코드 써줘" | "너는 10년 경력의 웹 개발자야. 초보자도 이해할 수 있게 코드를 작성하고 각 줄마다 한국어 주석을 달아줘" |
| **3. 예시 제공** | 참고할 예시나 형식을 제시 | "예쁘게 만들어줘" | "네이버 블로그 스타일로 만들어줘. 상단에 큰 배너, 좌측에 카테고리 메뉴, 우측에 글 목록이 있는 레이아웃" |
| **4. 단계적으로** | 한 번에 다 요청하지 않고 나누어 요청 | "완전한 쇼핑몰 만들어줘" | "먼저 메인 페이지만 만들어줘. 상품 카드 4개가 그리드로 배치된 형태로" |
| **5. 제약 조건** | 사용할 기술, 스타일 등 조건 명시 | "아무거나 괜찮아" | "HTML, CSS, JavaScript만 사용해줘. 외부 라이브러리는 쓰지 마. 반응형으로 만들어줘" |

#### 코딩 프롬프트 템플릿 5가지

**템플릿 1: 웹 페이지 만들기**

```
너는 웹 개발 전문가야.
아래 요구사항에 맞게 HTML/CSS/JavaScript 코드를 작성해줘.

[만들고 싶은 페이지 설명]
- 페이지 목적: _______
- 포함할 요소: _______
- 디자인 스타일: _______
- 색상 테마: _______

조건:
- HTML, CSS, JS를 하나의 파일로 작성해줘
- 모바일에서도 잘 보이게 반응형으로 만들어줘
- 코드마다 한국어 주석을 달아줘
```

**템플릿 2: 기존 코드 수정하기**

```
아래 코드를 수정해줘.

[기존 코드를 붙여넣기]

수정 요청:
1. _______ 부분을 _______ 으로 변경
2. _______ 기능을 추가
3. 디자인을 _______ 스타일로 변경

기존 구조는 유지하면서 수정해줘.
```

**템플릿 3: 에러 해결하기**

```
아래 코드에서 에러가 발생하고 있어.

[에러가 있는 코드 붙여넣기]

에러 메시지:
[에러 메시지 붙여넣기]

1. 에러 원인을 설명해줘
2. 수정된 전체 코드를 보여줘
3. 왜 그렇게 수정했는지 설명해줘
```

**템플릿 4: 코드 설명 요청하기**

```
아래 코드를 초보자도 이해할 수 있게 설명해줘.

[이해하고 싶은 코드 붙여넣기]

- 각 줄마다 한국어로 주석을 달아줘
- 핵심 개념을 쉬운 비유로 설명해줘
- 이 코드를 커스터마이징하려면 어떤 부분을 바꾸면 되는지 알려줘
```

**템플릿 5: 프로젝트 기획 단계**

```
나는 코딩을 모르는 초보자야.
아래 아이디어를 웹사이트로 만들고 싶어.

아이디어: _______
대상 사용자: _______
핵심 기능: _______

1. 이 프로젝트를 만들기 위해 필요한 파일 구조를 알려줘
2. 어떤 순서로 만들면 좋을지 단계를 나눠줘
3. 첫 번째 단계의 코드를 작성해줘
```

#### GPTs(커스텀 GPT) 소개

GPTs는 특정 목적에 맞게 미리 설정된 ChatGPT입니다:

- **무엇인가요?**: 특정 역할, 지식, 행동 패턴이 미리 설정된 커스텀 ChatGPT입니다
- **어떻게 찾나요?**: ChatGPT 왼쪽 사이드바에서 "GPT 탐색" 또는 [GPT Store](https://chat.openai.com/gpts)에서 검색합니다
- **코딩에 유용한 GPTs**: "Code Copilot", "Grimoire", "Website Generator" 등을 검색해보세요
- **직접 만들 수도 있습니다**: Plus 이상 사용자는 자신만의 GPT를 만들 수 있습니다

> **실전 팁**: GPTs는 매번 프롬프트를 길게 쓰지 않아도 됩니다. 한 번 설정해두면 해당 분야에 특화된 AI로 바로 활용할 수 있어 편리합니다.

### 실습 과제

1. **템플릿 1**을 사용하여 자신만의 자기소개 웹 페이지를 요청해보세요
2. 결과가 마음에 안 들면 **추가 프롬프트**를 작성하여 수정을 요청해보세요

## 6. Canvas 활용하기

### 학습 목표

- Canvas 모드를 실행하고 사용할 수 있다
- Canvas에서 코드를 직접 편집할 수 있다
- Canvas의 핵심 기능 6가지를 활용할 수 있다
- 완성된 코드를 내보낼 수 있다

### 상세 설명

#### Canvas 모드란?

Canvas는 ChatGPT에 내장된 **코드 전용 작업 공간**입니다. 일반 채팅에서는 코드가 대화 흐름 속에 표시되지만, Canvas에서는 별도 패널에서 코드를 넓게 보고 편집할 수 있습니다.

**Canvas가 열리는 조건:**
- 코드를 포함한 응답이 생성될 때 자동으로 열립니다
- 또는 프롬프트에 "Canvas에서 보여줘"라고 명시합니다
- 채팅 입력창 왼쪽의 도구 아이콘에서 Canvas를 선택할 수도 있습니다

**일반 채팅 vs Canvas 비교:**

| 항목 | 일반 채팅 | Canvas |
|------|----------|--------|
| 코드 표시 | 대화 흐름 안에 코드 블록 | 별도 넓은 패널 |
| 편집 방식 | 전체를 다시 요청 | 특정 부분만 선택하여 수정 |
| 직접 수정 | 불가 | 코드를 직접 타이핑하여 수정 가능 |
| 버전 관리 | 대화 히스토리에서 찾기 | 수정 이력 확인 가능 |
| 내보내기 | 코드 블록에서 복사 | 전체 코드 복사/다운로드 |

#### Canvas 핵심 기능 6가지

**기능 1: 코드 직접 편집**

Canvas에서는 코드를 직접 클릭하여 수정할 수 있습니다:

1. Canvas 패널에서 수정하고 싶은 부분을 클릭합니다
2. 직접 타이핑하여 코드를 수정합니다
3. 수정 후 ChatGPT에게 "이 수정이 맞는지 확인해줘"라고 요청합니다

**기능 2: 부분 선택 후 수정 요청**

코드의 특정 부분만 선택하여 수정을 요청할 수 있습니다:

1. 수정하고 싶은 코드 부분을 드래그하여 **선택(하이라이트)**합니다
2. 선택 영역 옆에 나타나는 **입력창에 수정 요청**을 작성합니다
3. 예: "이 부분의 색상을 파란색으로 바꿔줘"

**기능 3: 코드 리뷰 요청**

Canvas 하단 도구 모음에서 사용할 수 있는 기능입니다:

```
프롬프트 예시:
"이 코드를 리뷰해줘. 개선할 점과 잠재적인 문제를 알려줘."
```

**기능 4: 디버깅 (버그 수정)**

코드에 오류가 있을 때:

```
프롬프트 예시:
"이 코드를 실행하면 [에러 메시지]가 나와. 원인을 찾아서 수정해줘."
```

**기능 5: 주석 추가**

Canvas 하단의 도구 모음에서 "주석 추가" 버튼을 클릭하면:
- AI가 코드의 각 섹션에 한국어 주석을 자동으로 추가합니다
- 코드를 이해하는 데 큰 도움이 됩니다

**기능 6: 코드 내보내기**

완성된 코드를 가져오는 방법:
1. Canvas 우측 상단의 **복사 버튼**을 클릭합니다
2. 메모장이나 VS Code에 **붙여넣기**합니다
3. 적절한 파일명으로 저장합니다 (예: `index.html`)

## 7. Codex(자율 코딩) 체험

### 학습 목표

- Codex가 무엇인지 이해한다
- Codex에 작업을 요청하는 방법을 안다
- Codex의 결과물을 확인하고 다운로드할 수 있다
- Canvas와 Codex의 차이를 구분할 수 있다

### 상세 설명

#### Codex란?

**Codex**는 OpenAI가 제공하는 **자율 코딩 에이전트**입니다. Canvas가 "같이 보면서 수정하는 작업 공간"이라면, Codex는 "작업 지시만 내리면 알아서 완성해주는 주니어 개발자"입니다.

| 항목 | Canvas | Codex |
|------|--------|-------|
| **작업 방식** | 대화하며 실시간 수정 | 작업 지시 후 자율 실행 |
| **사용자 관여** | 높음 (같이 작업) | 낮음 (지시만 하면 됨) |
| **결과 형태** | 채팅 내 코드 | GitHub PR, 완성된 프로젝트 |
| **실행 환경** | 채팅 내 | 클라우드 샌드박스 |
| **비유** | 과외 선생님과 함께 공부 | 숙제를 대신 해주는 도우미 |
| **사용 조건** | 무료/유료 모두 가능 | Plus 이상 필요 |

> **참고**: Codex는 2025년 하반기에 출시된 비교적 새로운 기능입니다. Plus 이상 요금제에서 사용 가능합니다. 무료 사용자라면 이 섹션은 개념만 이해하고 넘어가도 됩니다.

#### Codex 작업 요청 방법

1. ChatGPT 왼쪽 사이드바에서 **Codex** 메뉴를 클릭합니다
2. **New Task** (새 작업)를 클릭합니다
3. 작업 내용을 자연어로 설명합니다

**작업 요청 예시:**

```
할 일 목록(To-Do List) 웹 앱을 만들어줘.

기능:
- 할 일 추가, 완료 체크, 삭제
- 완료된 항목은 취소선 표시
- 로컬 스토리지에 저장 (새로고침해도 유지)
- 깔끔한 미니멀 디자인
- 다크모드 지원

기술 스택: HTML, CSS, JavaScript (라이브러리 없이)
```

4. **Start** 버튼을 클릭하면 Codex가 자율적으로 작업을 시작합니다
5. 작업 진행 상황을 실시간으로 확인할 수 있습니다

#### Codex 결과 확인

작업이 완료되면:
1. **생성된 파일 목록**을 확인합니다
2. 각 파일의 **코드를 미리보기**합니다
3. **전체 프로젝트를 다운로드**하거나 **GitHub에 PR(Pull Request)**을 생성합니다

#### Codex 제한사항

| 제한 | 설명 |
|------|------|
| **요금제** | Plus 이상 필요 (무료 사용자는 사용 불가) |
| **처리 시간** | 복잡한 작업은 수 분이 걸릴 수 있음 |
| **실행 환경** | 클라우드 샌드박스에서 실행 (로컬 파일 접근 불가) |
| **외부 서비스 연동** | 외부 API 호출이 제한될 수 있음 |
| **프로젝트 규모** | 매우 큰 프로젝트는 여러 태스크로 나누어야 함 |

### Canvas vs Codex: 언제 무엇을 쓸까?

| 상황 | 추천 도구 |
|------|-----------|
| 간단한 코드 한 개 파일 만들기 | **Canvas** |
| 코드를 같이 보며 수정하고 싶을 때 | **Canvas** |
| 코드를 배우면서 만들고 싶을 때 | **Canvas** |
| 여러 파일로 구성된 프로젝트 만들기 | **Codex** |
| 작업 지시만 하고 결과만 받고 싶을 때 | **Codex** |
| GitHub에 바로 올리고 싶을 때 | **Codex** |

### 실습 과제

1. (Plus 사용자) Codex에서 **간단한 계산기 웹 앱**을 만들어달라고 요청해보세요
2. (무료 사용자) Canvas에서 같은 계산기를 만들어보고 Codex와의 **차이점을 이해**해보세요
3. Codex가 만든 결과물의 **파일 구조**를 살펴보세요

## 8. 실전: 첫 프로젝트 만들기

### 학습 목표

- ChatGPT와 대화하며 프로젝트 아이디어를 구체화할 수 있다
- Canvas에서 코드를 작성하고 수정할 수 있다
- 완성된 코드를 로컬에서 실행할 수 있다
- 반복적인 수정(이터레이션)을 통해 결과물을 개선할 수 있다

### 상세 설명

이제 실제로 프로젝트를 만들어봅시다! 아래 4단계를 순서대로 따라하세요.

#### Step 1: 아이디어 구상 (ChatGPT와 대화)

무엇을 만들지 아직 정하지 못했다면, ChatGPT에게 도움을 요청하세요:

```
나는 코딩 초보자야.
ChatGPT를 활용해서 간단한 웹 페이지를 만들어보고 싶어.
30분 안에 완성할 수 있는 재미있는 미니 프로젝트 5개를 추천해줘.
각 프로젝트마다 필요한 기술과 난이도(1~5)를 알려줘.
```

**추천 첫 프로젝트 (난이도순):**

| 프로젝트 | 난이도 | 배우는 것 | 소요 시간 |
|----------|--------|-----------|-----------|
| 자기소개 페이지 | 1 | HTML 기본 구조, CSS 스타일링 | 15분 |
| 디지털 시계 | 2 | JavaScript 기초, 실시간 업데이트 | 20분 |
| 할 일 목록 (To-Do) | 3 | 이벤트 처리, 데이터 저장 | 30분 |
| 퀴즈 게임 | 3 | 조건문, 점수 계산 | 30분 |
| 날씨 정보 대시보드 | 4 | API 연동, 비동기 처리 | 45분 |

> **추천**: 첫 프로젝트로 **자기소개 페이지**를 만들어보세요. 가장 쉬우면서도 HTML/CSS의 기본을 배울 수 있습니다!

#### Step 2: Canvas에서 코드 작성

아이디어가 정해졌다면 ChatGPT에게 코드를 요청합니다. 여기서는 **자기소개 페이지**를 예시로 진행합니다.

**프롬프트 예시:**

```
Canvas에서 나만의 자기소개 웹 페이지를 만들어줘.

요구사항:
- HTML, CSS, JavaScript를 하나의 HTML 파일로 작성
- 상단: 프로필 이미지 자리(원형), 이름, 한 줄 소개
- 중간: 자기소개 텍스트, 관심사 태그(배지 형태)
- 하단: 연락처(이메일, SNS 링크 버튼)
- 전체적으로 깔끔하고 모던한 디자인
- 다크모드 전환 버튼 포함
- 반응형 (모바일 대응)
- 부드러운 스크롤 애니메이션
- 각 코드에 한국어 주석 추가

색상 테마:
- 메인 색상: #6366f1 (보라색 계열)
- 배경: 밝은 그레이 (#f8fafc)
```

ChatGPT가 Canvas에서 코드를 생성합니다. 생성된 코드를 살펴보세요.

#### Step 3: 로컬에서 실행

Canvas에서 완성된 코드를 로컬 컴퓨터에서 실행해봅시다:

1. Canvas 우측 상단의 **복사 버튼**을 클릭합니다
2. **메모장**(또는 VS Code)을 엽니다
3. 복사한 코드를 **붙여넣기** 합니다
4. **다른 이름으로 저장**을 선택합니다
5. 파일 형식을 **모든 파일**로 변경합니다
6. 파일명을 `index.html` 로 입력하고 저장합니다
7. 저장된 파일을 **더블 클릭**하여 브라우저에서 엽니다

> **VS Code 사용자라면**: 파일을 저장한 후 우측 하단의 "Go Live" 버튼을 클릭하면 자동으로 브라우저에서 열립니다 (Live Server 확장 필요).

```
로컬 실행 순서:
ChatGPT Canvas → 코드 복사 → 메모장에 붙여넣기 → index.html로 저장 → 브라우저에서 열기
```

#### Step 4: 수정 반복 (이터레이션)

결과물을 확인한 후 마음에 들지 않는 부분을 수정합니다:

**수정 요청 예시들:**

```
수정 1: "프로필 이미지를 더 크게 만들어줘. 150px에서 200px로"
수정 2: "관심사 태그를 클릭하면 색이 바뀌게 해줘"
수정 3: "하단에 방문자 카운터를 추가해줘"
수정 4: "배경에 은은한 그라데이션 효과를 넣어줘"
수정 5: "전체적인 폰트를 'Noto Sans KR'로 바꿔줘"
```

> **핵심 팁**: 한 번에 완벽한 결과를 기대하지 마세요. 수정을 반복하면서 점점 완성도를 높여가는 것이 바이브코딩의 핵심입니다. 보통 3~5회 수정이면 만족스러운 결과물이 나옵니다.

#### 실전 코드 예시: 간단한 자기소개 페이지

아래는 ChatGPT가 생성할 수 있는 자기소개 페이지의 기본 구조입니다:

```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>나의 자기소개 페이지</title>
    <style>
        /* 전체 페이지 기본 스타일 */
        * { margin: 0; padding: 0; box-sizing: border-box; }
        body {
            font-family: 'Noto Sans KR', sans-serif;
            background: #f8fafc;
            color: #333;
            min-height: 100vh;
            display: flex;
            justify-content: center;
            align-items: center;
        }
        /* 프로필 카드 컨테이너 */
        .card {
            background: white;
            border-radius: 20px;
            padding: 40px;
            max-width: 500px;
            width: 90%;
            box-shadow: 0 10px 40px rgba(0,0,0,0.1);
            text-align: center;
        }
        /* 프로필 이미지 (원형) */
        .profile-img {
            width: 150px;
            height: 150px;
            border-radius: 50%;
            background: #6366f1;
            margin: 0 auto 20px;
            display: flex;
            align-items: center;
            justify-content: center;
            color: white;
            font-size: 48px;
        }
        /* 이름 스타일 */
        .name { font-size: 28px; font-weight: 700; margin-bottom: 8px; }
        /* 한 줄 소개 */
        .tagline { color: #6366f1; font-size: 16px; margin-bottom: 24px; }
        /* 관심사 태그 */
        .tags { display: flex; flex-wrap: wrap; gap: 8px; justify-content: center; margin: 20px 0; }
        .tag {
            background: #eef2ff;
            color: #6366f1;
            padding: 6px 16px;
            border-radius: 20px;
            font-size: 14px;
        }
    </style>
</head>
<body>
    <div class="card">
        <!-- 프로필 이미지 영역 -->
        <div class="profile-img">👤</div>
        <!-- 이름과 소개 -->
        <h1 class="name">홍길동</h1>
```

## 9. 코드 가져오기와 실행

### 학습 목표

- ChatGPT에서 생성된 코드를 로컬 파일로 저장할 수 있다
- VS Code에서 코드를 열고 편집할 수 있다
- 브라우저에서 HTML 파일을 실행할 수 있다
- 기본적인 파일 구조를 이해한다

### 상세 설명

#### 9-1. ChatGPT 코드를 로컬에 저장하기

**방법 1: Canvas에서 복사하기 (추천)**

1. Canvas에서 코드가 표시된 상태에서 **우측 상단 복사 아이콘**을 클릭합니다
2. 텍스트 편집기(메모장, VS Code 등)에 **붙여넣기** 합니다
3. 파일명을 지정하여 저장합니다

**방법 2: 일반 채팅에서 복사하기**

1. 코드 블록 우측 상단의 **Copy code** 버튼을 클릭합니다
2. 텍스트 편집기에 붙여넣기합니다

**방법 3: 여러 파일이 있는 경우**

ChatGPT가 여러 파일(HTML, CSS, JS)을 생성했다면:

1. 프로젝트 폴더를 만듭니다 (예: `my-project`)
2. 각 코드 블록을 해당 파일명으로 따로 저장합니다

```
my-project/
├── index.html    ← HTML 코드
├── style.css     ← CSS 코드
└── script.js     ← JavaScript 코드
```

#### 9-2. VS Code에서 열기

VS Code를 사용하면 코드를 더 편하게 관리할 수 있습니다:

1. VS Code를 실행합니다
2. **File → Open Folder** 를 클릭합니다
3. 프로젝트 폴더를 선택합니다
4. 좌측 파일 탐색기에서 파일을 클릭하여 열 수 있습니다

**VS Code에서 유용한 단축키:**

| 단축키 | 기능 |
|--------|------|
| `Ctrl + S` | 파일 저장 |
| `Ctrl + Z` | 실행 취소 |
| `Ctrl + C / V` | 복사 / 붙여넣기 |
| `Ctrl + F` | 검색 |
| `Ctrl + Shift + P` | 명령 팔레트 열기 |

#### 9-3. 브라우저에서 실행

**방법 1: 파일 더블 클릭 (가장 간단)**
- `index.html` 파일을 더블 클릭하면 기본 브라우저에서 열립니다

**방법 2: Live Server 사용 (추천)**
- VS Code에서 `index.html`을 연 상태에서
- 우측 하단의 **"Go Live"** 버튼을 클릭합니다
- 자동으로 브라우저에서 열리고, 코드 수정 시 **실시간으로 반영**됩니다

**방법 3: 브라우저에 직접 드래그**
- 파일 탐색기에서 `index.html`을 브라우저 창에 끌어다 놓습니다

#### 9-4. 파일 구조 이해하기

웹 페이지는 보통 3가지 파일로 구성됩니다:

| 파일 | 역할 | 비유 |
|------|------|------|
| **index.html** | 웹 페이지의 구조(뼈대) | 건물의 설계도 — 어떤 요소가 어디에 있는지 |
| **style.css** | 웹 페이지의 디자인(꾸미기) | 건물의 인테리어 — 색상, 크기, 배치 |
| **script.js** | 웹 페이지의 동작(기능) | 건물의 전기 시설 — 버튼 클릭, 애니메이션 |

> **초보자 팁**: ChatGPT에게 "하나의 HTML 파일로 만들어줘"라고 요청하면 HTML, CSS, JS가 모두 합쳐진 하나의 파일을 만들어줍니다. 처음에는 이 방법이 관리하기 쉽습니다.

#### Python 파일 실행하기 (선택)

ChatGPT가 Python 코드를 생성했다면:

```python
# hello.py - 간단한 인사 프로그램
name = input("이름을 입력하세요: ")
print(f"안녕하세요, {name}님! 바이브코딩의 세계에 오신 것을 환영합니다!")
```

**실행 방법:**
1. 코드를 `hello.py` 파일로 저장합니다
2. 명령 프롬프트(터미널)를 엽니다
3. `python hello.py` 를 입력하고 Enter를 누릅니다

> **참고**: Python을 실행하려면 먼저 [python.org](https://www.python.org)에서 Python을 설치해야 합니다. 웹 페이지 만들기만 할 거라면 Python 설치는 필요 없습니다.

### 실습 과제

1. ChatGPT에서 만든 코드를 **VS Code에서 열어보세요**
2. VS Code에서 코드의 **텍스트를 직접 수정**하고 저장한 후 브라우저에서 변화를 확인하세요
3. Live Server를 설치하고 **실시간 미리보기**를 활용해보세요

### 체크포인트

- [ ] Canvas에서 코드를 복사하여 파일로 저장할 수 있다
- [ ] VS Code에서 프로젝트를 열 수 있다
- [ ] 브라우저에서 HTML 파일을 실행할 수 있다
- [ ] HTML, CSS, JS 파일의 역할 차이를 안다

## 10. 문제가 생겼을 때 (트러블슈팅)

### 학습 목표

- 자주 발생하는 5가지 문제를 스스로 해결할 수 있다
- ChatGPT에게 효과적으로 오류 해결을 요청할 수 있다

### 상세 설명

ChatGPT로 코딩하다가 문제가 생기면 당황하지 마세요! 아래 5가지가 가장 흔한 문제입니다.

#### 문제 1: 코드 에러가 발생한다

**증상**: 브라우저에서 페이지가 제대로 표시되지 않거나, 콘솔에 빨간 에러 메시지가 보인다.

**해결 방법:**

1. 브라우저에서 **F12** 키를 눌러 개발자 도구를 엽니다
2. **Console** 탭에서 빨간색 에러 메시지를 확인합니다
3. 에러 메시지를 복사합니다
4. ChatGPT에게 아래와 같이 요청합니다:

```
아래 코드를 실행했는데 에러가 발생했어.

[코드 붙여넣기]

에러 메시지:
[에러 메시지 붙여넣기]

원인을 알려주고 수정된 코드를 보여줘.
```

> **핵심**: 에러 메시지를 그대로 복사하여 ChatGPT에게 보여주는 것이 가장 빠른 해결 방법입니다.

#### 문제 2: 토큰 한도 초과

**증상**: "You've reached the current usage cap" 또는 메시지 전송이 안 된다.

**해결 방법:**

| 상황 | 해결책 |
|------|--------|
| 무료 사용자 | 몇 시간 기다린 후 다시 시도하세요 |
| 대화가 너무 길어진 경우 | **새 채팅**을 시작하세요 (긴 대화는 토큰을 많이 소모합니다) |
| 자주 발생하는 경우 | Plus 요금제($20/월)로 업그레이드를 고려하세요 |
| 코드가 너무 긴 경우 | 코드를 나누어서 요청하세요 |

**예방 팁:**
- 하나의 대화에서 너무 많은 요청을 하지 마세요
- 새로운 주제나 프로젝트는 **새 채팅**에서 시작하세요
- 긴 코드를 반복해서 붙여넣지 않도록 "위 코드를 기반으로..."라고 참조하세요

#### 문제 3: Canvas가 열리지 않는다

**증상**: 코드를 요청했는데 Canvas 패널이 나타나지 않고 일반 채팅에 코드가 표시된다.

**해결 방법:**

1. 채팅 입력창 왼쪽의 **도구 아이콘**(+)을 클릭합니다
2. **Canvas** 옵션이 있는지 확인합니다
3. 프롬프트에 명시적으로 "**Canvas에서 보여줘**"를 추가합니다
4. 브라우저를 **새로고침**합니다
5. 다른 브라우저(Chrome, Edge 등)로 시도합니다

> **참고**: Canvas는 일부 짧은 코드 스니펫에는 자동으로 열리지 않을 수 있습니다. 일정 길이 이상의 코드에서 활성화됩니다.

#### 문제 4: 코드가 잘린다 (불완전한 코드)

**증상**: ChatGPT의 응답이 도중에 끊기고 코드가 완성되지 않았다.

**해결 방법:**

```
프롬프트: "이어서 계속 작성해줘"
```

또는:

```
프롬프트: "위 코드의 나머지 부분을 완성해줘. [마지막으로 표시된 줄] 이후부터"
```

**예방 팁:**
- 처음부터 너무 복잡한 것을 요청하지 마세요
- "단계별로 나눠서 만들어줘"라고 요청하세요
- 큰 프로젝트는 여러 개의 작은 파일로 나누어 요청하세요

#### 문제 5: 원하는 결과가 안 나온다

**증상**: 코드는 동작하지만 내가 원하는 것과 다르다.

**해결 방법:**

| 문제 유형 | 개선된 프롬프트 |
|-----------|---------------|
| 디자인이 다름 | "배경색을 #1a1a2e로, 제목 글꼴 크기를 32px로 바꿔줘" (구체적 수치 제시) |
| 기능이 빠짐 | "○○ 버튼을 클릭하면 △△가 되는 기능을 추가해줘" (동작을 명확히) |
| 레이아웃이 다름 | 원하는 레이아웃의 **스크린샷**을 첨부하고 "이런 레이아웃으로 만들어줘" |
| 전체적으로 다름 | 처음부터 **더 구체적인 프롬프트**로 새로 요청하기 |

> **황금 규칙**: "구체적일수록 결과가 좋다." 색상은 색상 코드(#FF5733)로, 크기는 픽셀(200px)로, 동작은 단계별로 설명하세요.

### 실습 과제

1. 일부러 **에러가 있는 코드**를 ChatGPT에게 보여주고 수정을 요청해보세요
2. 브라우저 **개발자 도구(F12)**를 열어 Console 탭을 확인해보세요
3. "이어서 작성해줘" 프롬프트를 사용하여 잘린 코드를 완성해보세요

### 체크포인트

- [ ] 코드 에러 발생 시 ChatGPT에게 효과적으로 도움을 요청할 수 있다
- [ ] 토큰 한도 초과 시 대처 방법을 안다
- [ ] 브라우저 개발자 도구에서 에러 메시지를 찾을 수 있다
- [ ] 원하는 결과가 안 나올 때 프롬프트를 개선하는 방법을 안다

## 11. 자주 묻는 질문 (FAQ)

### 학습 목표

- ChatGPT 코딩 활용에 대한 주요 궁금증을 해소한다
- 자주 하는 오해를 바로잡는다

### FAQ

#### Q1. 정말 무료로도 코딩할 수 있나요?

**A:** 네, 가능합니다! ChatGPT 무료 플랜에서도 코드 생성, Canvas 기본 기능을 사용할 수 있습니다. 다만, 사용량에 제한이 있어 하루에 보낼 수 있는 메시지 수가 한정됩니다. 가볍게 웹 페이지를 만드는 정도는 무료로도 충분합니다. Codex(자율 코딩 에이전트)를 사용하려면 Plus($20/월) 이상이 필요합니다.

#### Q2. ChatGPT와 Claude(클로드)의 차이는 뭔가요?

**A:**

| 항목 | ChatGPT (OpenAI) | Claude (Anthropic) |
|------|-------------------|-------------------|
| **강점** | 범용성, Canvas/Codex 통합, GPTs 생태계 | 긴 문맥 이해, 안전성, 정밀한 코드 생성 |
| **코딩 도구** | Canvas, Codex | Artifacts, Claude Code |
| **접근성** | 전 세계 1위 사용자 수 | 빠르게 성장 중 |
| **무료 사용** | GPT-4o 제한적 무료 | Claude 3.5 Sonnet 제한적 무료 |
| **생태계** | GPTs, 플러그인, API | MCP, 프로젝트 |

> 어떤 도구가 더 좋다기보다, **용도에 따라 적합한 도구**가 다릅니다. ChatGPT는 범용성과 접근성에서, Claude는 정밀함과 긴 대화에서 강합니다.

#### Q3. Canvas와 Codex의 차이는 뭔가요?

**A:** 간단히 비유하면, Canvas는 "선생님과 함께 칠판에서 코드를 보며 수정하는 것"이고, Codex는 "전문가에게 일을 맡기고 결과만 받는 것"입니다.

- **Canvas**: 실시간 대화하며 코드를 편집. 학습에 좋고, 간단한 코드에 적합
- **Codex**: 작업 지시 후 자율 실행. 복잡한 프로젝트, 여러 파일 프로젝트에 적합

#### Q4. ChatGPT가 만든 코드의 저작권은 누구에게 있나요?

**A:** OpenAI의 이용약관에 따르면, ChatGPT가 생성한 결과물(코드 포함)의 권리는 **사용자에게** 있습니다. 다만 아래 사항에 유의하세요:

- 생성된 코드가 기존 오픈소스 라이브러리와 유사할 수 있으므로, 상용 프로젝트에서는 **라이선스를 확인**하세요
- AI 생성물의 저작권에 대한 법적 판단은 **나라마다, 상황마다 다를 수 있습니다**
- 중요한 프로젝트에서는 **법률 전문가에게 상담**하는 것을 권장합니다

#### Q5. 한국어로 코딩 요청해도 잘 되나요?

**A:** 네, 매우 잘 됩니다! ChatGPT는 한국어를 훌륭하게 이해합니다. 코딩 요청, 설명 요청, 에러 해결 모두 한국어로 해도 정확한 결과를 받을 수 있습니다. 다만:

- **코드 자체**(변수명, 함수명 등)는 영어로 작성하는 것이 관례입니다
- **주석과 설명**은 한국어로 요청하면 한국어로 달아줍니다
- 한국어 프롬프트로 영어 코드를 생성하는 것이 가장 일반적인 패턴입니다

#### Q6. 코딩을 전혀 몰라도 괜찮나요?

**A:** 네, 괜찮습니다! 이 교안의 대상이 바로 "코딩을 모르는 분"입니다. ChatGPT가 코드를 대신 작성해주고, Canvas에서 시각적으로 코드를 편집할 수 있으므로 코딩 지식이 없어도 시작할 수 있습니다.

다만 **약간의 코딩 지식이 있으면 훨씬 유리합니다**:
- 어떤 부분을 수정해야 하는지 빠르게 파악할 수 있습니다
- 에러가 발생했을 때 원인을 추측할 수 있습니다
- 더 구체적인 프롬프트를 작성할 수 있습니다

> ChatGPT와 함께 코딩하다 보면 자연스럽게 코딩 지식이 쌓입니다. 어렵게 생각하지 마세요!

#### Q7. 결과물을 상용화(돈 벌기)해도 되나요?

**A:** 기본적으로 **가능합니다**. OpenAI 이용약관에 따르면 생성된 결과물을 상업적으로 사용할 수 있습니다. 다만 주의할 점이 있습니다:

- 생성된 코드의 **품질과 보안을 반드시 검증**해야 합니다
- 민감한 데이터(결제, 개인정보 등)를 다루는 코드는 **전문가 검수**를 받으세요
- ChatGPT가 사용한 코드 패턴이 특정 라이선스에 해당할 수 있으므로 확인이 필요합니다
- AI 생성 코드임을 명시해야 하는 경우가 있을 수 있습니다 (계약 등에 따라)

#### Q8. 보안 문제는 없나요?

**A:** 아래 사항만 주의하면 안전하게 사용할 수 있습니다:

| 주의 사항 | 설명 |
|-----------|------|
| **비밀번호 입력 금지** | ChatGPT에 비밀번호, API 키 등 민감 정보를 입력하지 마세요 |
| **개인정보 주의** | 주민번호, 카드번호 등 개인정보를 포함하지 마세요 |
| **회사 기밀 주의** | 회사 내부 코드나 기밀 정보를 ChatGPT에 입력하지 마세요 |
| **코드 검증 필수** | AI가 만든 코드를 그대로 실제 서비스에 적용하지 말고 검증하세요 |
| **히스토리 관리** | 민감한 대화는 삭제하거나 "Temporary Chat" 모드를 활용하세요 |

### 실습 과제

1. 위 FAQ 중 **궁금한 것 3가지**를 골라 ChatGPT에게 직접 물어보세요
2. ChatGPT의 답변과 이 FAQ의 내용을 **비교**해보세요

### 체크포인트

- [ ] 보안 주의사항 5가지를 기억한다

## 12. 주의사항 및 팁

### 학습 목표

- ChatGPT로 코딩할 때 반드시 지켜야 할 주의사항을 안다
- 생산성을 높이는 실전 팁 5가지를 활용할 수 있다

### 상세 설명

#### 주의사항

**1. API 키 보안**

```
절대 하지 말 것:
❌ ChatGPT에 API 키, 비밀번호를 입력하는 것
❌ 코드에 API 키를 하드코딩(직접 입력)하는 것
❌ GitHub에 API 키가 포함된 코드를 올리는 것

반드시 할 것:
✅ API 키는 환경 변수(.env 파일)에 저장하기
✅ .gitignore에 .env 파일을 등록하기
✅ API 키가 노출되었다면 즉시 재발급받기
```

> **비유**: API 키는 집 열쇠와 같습니다. 아무 데나 두면 안 됩니다!

**2. 비용 관리**

| 플랜 | 월 비용 | 관리 포인트 |
|------|---------|-------------|
| Free | $0 | 사용량 제한을 넘으면 잠시 대기 필요 |
| Plus | $20 | 매월 자동 결제 — 불필요 시 해지 확인 |
| Pro | $200 | 높은 비용 — 정말 필요한지 신중히 판단 |
| API 사용 | 종량제 | 사용량에 따라 과금 — 반드시 한도 설정 |

> **팁**: API를 직접 사용하는 경우, OpenAI 대시보드에서 **사용량 한도(Usage Limit)**를 반드시 설정하세요. 예상치 못한 과금을 방지할 수 있습니다.

**3. 코드 검증의 필요성**

ChatGPT가 만든 코드는 항상 올바르지는 않습니다:

- **문법 오류**: 드물지만 잘못된 문법의 코드가 생성될 수 있습니다
- **논리 오류**: 코드가 동작하지만 의도한 결과가 아닐 수 있습니다
- **보안 취약점**: 보안을 고려하지 않은 코드가 생성될 수 있습니다
- **최신 정보 미반영**: 최신 API나 라이브러리 변경 사항을 반영하지 못할 수 있습니다

> **황금 규칙**: "AI가 만든 코드를 무조건 신뢰하지 말고, 실행해보고 확인하세요."

**4. 개인정보 주의**

ChatGPT에 입력하는 모든 내용은 AI 학습에 사용될 수 있습니다:

- 실제 사용자 데이터를 입력하지 마세요
- 테스트할 때는 **가짜(더미) 데이터**를 사용하세요
- 민감한 프로젝트는 **Temporary Chat** 모드를 활용하세요
- Settings에서 **"Improve the model for everyone"** 옵션을 끌 수 있습니다

#### 유용한 팁 5가지

**팁 1: 새 채팅으로 시작하기**

대화가 길어지면 ChatGPT의 응답 품질이 떨어질 수 있습니다. 새로운 기능을 추가하거나 다른 주제로 넘어갈 때는 **새 채팅을 시작**하세요.

```
새 채팅 시작 시 컨텍스트 제공:
"이전에 만든 프로젝트를 이어서 작업하려고 해. 아래가 현재 코드야:
[코드 붙여넣기]
여기에 ○○ 기능을 추가해줘."
```

**팁 2: 스크린샷 활용하기**

원하는 디자인의 웹사이트 스크린샷을 ChatGPT에게 보여주면 비슷하게 만들어줍니다:

```
프롬프트: "첨부한 스크린샷과 비슷한 디자인으로 웹 페이지를 만들어줘.
같은 레이아웃, 비슷한 색상 톤으로 만들되,
내용은 [내 프로젝트 주제]에 맞게 바꿔줘."
```

**팁 3: "주석 달아줘" 활용하기**

코드를 이해하고 싶을 때:

```
프롬프트: "이 코드의 모든 줄에 한국어 주석을 달아줘.
초보자도 이해할 수 있게 쉬운 말로 설명해줘."
```

이렇게 하면 코드를 읽으며 자연스럽게 프로그래밍을 배울 수 있습니다.

**팁 4: 단계별 요청하기**

복잡한 프로젝트는 한 번에 만들지 말고 나눠서 요청하세요:

```
1차 요청: "먼저 기본 레이아웃(HTML 구조)만 만들어줘"
2차 요청: "이제 CSS로 예쁘게 디자인해줘"
3차 요청: "JavaScript로 ○○ 기능을 추가해줘"
4차 요청: "모바일에서도 잘 보이게 반응형으로 수정해줘"
```

**팁 5: 코드 백업 습관 들이기**

마음에 드는 코드가 나왔을 때:

1. **즉시 파일로 저장**하세요 (ChatGPT 대화가 사라질 수 있습니다)
2. **버전별로 저장**하세요 (v1, v2, v3... 또는 날짜별)
3. 가능하다면 **GitHub에 업로드**하세요 (영구 보관)

```
예시 파일 구조:
my-project/
├── v1_index.html     ← 첫 번째 버전
├── v2_index.html     ← 두 번째 버전 (디자인 수정)
├── v3_index.html     ← 세 번째 버전 (기능 추가)
└── index.html        ← 최종 버전
```

### 실습 과제

1. ChatGPT **Settings**에서 데이터 학습 관련 설정을 확인하세요
2. 이전에 만든 코드를 **버전별로 저장**해보세요 (v1, v2...)
3. ChatGPT에게 기존 코드의 **모든 줄에 한국어 주석을 추가**해달라고 요청하세요

### 체크포인트

- [ ] API 키를 안전하게 관리하는 방법을 안다
- [ ] ChatGPT에 개인정보를 입력하면 안 된다는 것을 인지했다
- [ ] 코드 검증의 중요성을 이해했다
- [ ] 실전 팁 5가지 중 2가지 이상을 실제로 활용했다
