02 MCP 프로토콜 구조

MCP 서버를 만들기 전에 서버가 Claude에게 무엇을 제공할 수 있는지 이해해야 합니다. MCP 서버는 3가지를 제공합니다.

MCP 서버가 제공하는 3가지 능력 ├── Tools → Claude가 직접 실행하는 함수 │ ├── 파일 읽기, API 호출, DB 쿼리... │ └── Claude가 "이 함수 실행해줘"라고 요청 → 결과 반환 │ ├── Resources → Claude가 읽을 수 있는 데이터 소스 │ ├── file://, db://, api:// 형태 URI │ └── 파일, DB 레코드, API 응답 등 │ └── Prompts → 재사용 가능한 프롬프트 템플릿 ├── 매개변수를 받아 동적으로 프롬프트 생성 └── 예: "이 코드를 리뷰해줘" 템플릿

통신 방식 (Transport)

방식사용 상황특징
stdio로컬 실행 (가장 일반적)빠름, 설정 간단, 로컬 전용
SSE원격 HTTP 서버팀 공유 가능, 인터넷 연결 필요
WebSocket실시간 양방향빠른 실시간 통신, 복잡도 높음
대부분의 경우 stdio면 충분합니다. 혼자 또는 소규모 팀이 로컬에서 쓴다면 stdio로 시작하세요. SSE는 서버를 배포해 여러 사람이 공유할 때 사용합니다.

05 Resource & Prompt

Resource — 데이터 소스 제공

Tool은 Claude가 "실행"하는 함수라면, Resource는 Claude가 "읽을 수 있는 데이터"를 제공합니다.

# 정적 리소스 — 항상 같은 내용 @mcp.resource("school://학생목록") def 전체_학생_목록() -> str: """전체 학생 명단을 제공합니다""" students = db.query("SELECT 학번, 이름, 학년, 반 FROM 학생") return "\n".join([f"{s.학번}: {s.이름} ({s.학년}학년 {s.반}반)" for s in students]) # 동적 리소스 — URI 매개변수로 변화 @mcp.resource("school://학생/{학번}/성적") def 학생_성적_리소스(학번: str) -> str: """특정 학생의 전체 성적 이력을 제공합니다""" records = db.query( "SELECT * FROM 성적이력 WHERE 학번=? ORDER BY 날짜", 학번 ) return format_records(records)

# school_server.py import sqlite3 import os from mcp.server.fastmcp import FastMCP mcp = FastMCP("학교 성적 관리 서버") DB_PATH = os.getenv("DB_PATH", "school.db") def get_db(): conn = sqlite3.connect(DB_PATH) conn.row_factory = sqlite3.Row return conn @mcp.tool() def 반별_성적_조회(학년: int, 반: int) -> str: """특정 학년·반의 전체 학생 성적을 조회합니다""" conn = get_db() rows = conn.execute( "SELECT 이름, 국어, 수학, 영어, 과학, (국어+수학+영어+과학)/4.0 AS 평균 " "FROM 성적 WHERE 학년=?
